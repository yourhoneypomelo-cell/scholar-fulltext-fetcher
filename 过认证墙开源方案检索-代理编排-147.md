## 一、把问题拆成两层（这是选型的前提）

RSC 两道门（见 -160/-165）：门① Cloudflare Turnstile（nodriver 真 Chrome 能过）；门② `crawlprevention/governor` 的**坏 reCAPTCHA**（`Invalid domain for site key`，**不可解**，按速率/行为软封）。代理只影响"**被判定为可疑的概率 + IP 是否被 pin**"，因此把"代理"拆成正交两层：

- **L1 IP 供给（supply）**：出口 IP 的**类型与信誉**。RSC governor 的判据里，**数据中心 ASN = 高可疑**、**`cf_clearance` 绑 IP**（换 IP 即失效）。→ 决定"能不能不被 pin"。
- **L2 编排（orchestration）**：给定一批 IP，如何**轮换 / 黏住(sticky) / 按host限速 / 探封后换出口**。→ 决定"多篇批量时会不会踩速率门"。

**开源在 L2 有价值，在 L1 对 RSC 无价值**（住宅供给天然是付费的）。下面据此评估。

## 三、Top 推荐（分层落地）

### L1 供给：**商业住宅 / 静态 ISP 代理**（RSC 唯一有效供给）
- **优先 budget 档**：Webshare($3/GB)、IPRoyal($7)、Decodo/Smartproxy($8.5)、SOAX($5.5) —— 选**支持 sticky 10–30min + 城市/ASN 定向 + 合规供应商(KYC/伦理采集)**者。
- **静态住宅(ISP) $2–8/IP/月** 更适合"长会话复用 cf_clearance"（IP 不变→cf_clearance 稳）。
- **成本估算**：RSC 金 OA ~8 篇 × 每篇几 MB × 几次重试 ≈ **<1GB → 全程几美元**。**成本非瓶颈**。
- **排除**：所有免费开源池（数据中心/公共 IP + 蜜罐风险）；AWS-API-Gateway 系（数据中心 + 可识别头）。

### L2 编排：**自建轻量 `ProxyLease`（~100 行）**，不整包引入
- 复用**已有** -165 `_host_capture_gate` / `_host_register_governor`（per-host 30–90s 限速 + governor 冷却），**加两件事**：① **sticky 出口分配**（一个 worker 绑一个住宅 session id）；② **按 IP 请求预算 + 探封轮换**（命中 governor / K 次后换 fresh session、冷却旧 IP）。
- **借鉴不引入**：mubeng 的 `--rotate-on-error/--remove-on-error`、scrapy-rotating-proxies 的 ban-detection → 我们只借"探封即换出口"的状态机。
- **不采纳 Scrapoxy**（EOL）。

### L3 HTTP-JA3：**保留 curl_cffi `impersonate` + `proxies`**
- 参考 `tls-impersonate-proxy`：`403/429/challenge → 换 fresh exit 重试`，与 L2 状态机合并。

### ★ 最关键接线：**IP-会话亲和（cf_clearance 绑 IP）**
> nodriver 过 CF + 铸 `cf_clearance` 的**出口 IP**，与随后 curl_cffi JA3 回放的**出口 IP 必须是同一个住宅 IP**，且 sticky 覆盖整个会话寿命。否则 cf_clearance 在异 IP 上**立即失效** → 依旧 403。这是本项目 N.3「浏览器过门 / curl 下 PDF 分离」在**代理层**的延伸约束。

## 四、给现有栈（nodriver + curl_cffi + FS shim）的集成示例（建议·未落地）

### 4.1 一个 sticky 住宅出口，喂给"浏览器 + HTTP"两端

```python
# 建议新增 fulltext_fetcher/proxy_lease.py（纯编排,~100 行；不接商业密钥则整体退化为直连）
import itertools, time, os

class ProxyLease:
    """住宅 sticky 出口租约:同一 lease 的 exit_url 在浏览器铸 cf_clearance 与 curl_cffi 回放两端一致。
    provider 端点形如 http://user-session-<sid>:pass@gate.provider.com:7777 (sid 决定 sticky IP)。"""
    def __init__(self, template=None, budget=2, cooldown=1800):
        self.template = template or os.environ.get("RES_PROXY_TEMPLATE")  # 含 {sid}
        self.budget, self.cooldown = budget, cooldown
        self._sid = itertools.count(1); self._n = 0
        self._cur = None; self._burned = {}  # exit_url -> until_ts

    def _new(self):
        sid = f"rsc{next(self._sid)}-{int(time.time())}"
        self._cur = self.template.format(sid=sid) if self.template else None
        self._n = 0
        return self._cur

    def acquire(self):                       # 供浏览器与 curl 共用同一出口
        if self._cur is None or self._n >= self.budget:
            return self._new()
        return self._cur

    def note_request(self): self._n += 1
    def note_governor(self):                 # 命中 governor→烧掉当前 IP、冷却、下次换新
        if self._cur: self._burned[self._cur] = time.time() + self.cooldown
        self._cur = None
```

### 4.2 浏览器端：nodriver 启同一出口（过 CF、铸 cf_clearance）

```python
lease = ProxyLease()
exit_url = lease.acquire()                   # 例: http://user-session-rscX:pass@gate:7777
browser_args = []
if exit_url:
    browser_args.append(f"--proxy-server={exit_url}")   # ← 与 curl 同一住宅 IP
# 注意(-165 P4): 绝不再加 --disable-blink-features=AutomationControlled
# 过 CF 后从 tab 取 cf_clearance + cookies；命中 governor 页 → lease.note_governor() 后换出口重试
```

### 4.3 HTTP 端：curl_cffi JA3 回放走同一出口 + 探封重试

```python
from curl_cffi import requests as cffi
def fetch_pdf(url, cookies, exit_url):
    r = cffi.get(url, impersonate="chrome",           # JA3/HTTP2(3) 对齐真 Chrome
                 cookies=cookies,                     # 复用浏览器铸的 cf_clearance
                 proxies={"http": exit_url, "https": exit_url} if exit_url else None,
                 timeout=45)
    if r.status_code in (403, 429, 503) or b"crawlprevention/governor" in r.content[:4000]:
        return None                                    # 交 L2:note_governor()→换 fresh 住宅出口重试
    return r.content if r.content[:5].lstrip().startswith(b"%PDF") else None
```

### 4.4 与 -165 补丁的关系（编排叠加，不替代）
`ProxyLease` 只补 **P6（住宅IP/会话）**；**P1 governor检测 / P2 landing预热 / P3 per-host限速冷却** 仍是前提。落地顺序建议：**先 -165 P1–P3（本机直连，零成本验证补丁）→ 再挂 `ProxyLease` 住宅出口做小样本 A/B**。

## 五、成本 / 合规

### 5.1 成本（2026 web 实测 $/GB）
| 类型 | $/GB | CF通过率 | 说明 |
|---|---|---|---|
| 数据中心(共享) | $0.5–2 | ~5% | 对RSC无效 |
| **住宅(轮换)** | **$1.75–15**(budget ~$3–8.5) | **~93–97%** | **RSC推荐** |
| 静态住宅/ISP | $2–8/IP/月 | 高 | 长会话复用cf_clearance最佳 |
| 移动4G/5G | $3.5–40 | ~95–99% | 最高信誉;但CGNAT致会话中途换IP、破sticky |
- **RSC 金 OA 全量 <1GB → 几美元**；订阅 ~59 篇属付费墙、**唯 A5 机构**，代理无关（见 -160 ROI）。

### 5.2 合规（硬约束）
- **只用于 OA / 已授权内容**：RSC 金 OA 为开放许可、下载正当；代理仅为**规避 IP 限速 pin**，**不用于绕过付费墙/鉴权**（订阅内容不走此路）。
- **供应商合规**：选 **KYC + 伦理采集（用户同意/ISP 协议）** 的商业住宅（Bright Data/Oxylabs/IPRoyal 等）；**严禁免费代理池**（多为蜜罐/劫持设备 → 法律+安全风险）。
- **操作合规**：尊重 robots/ToS、限速拟人、数据最小化、留存请求日志备审；遵守 GDPR/CCPA。

## 六、结论（可直接引用）

1. **对 RSC governor：开源代理池帮不上**（免费=数据中心/蜜罐，被 governor 盯死 + 合规风险）；**Scrapoxy 已 EOL**、不采纳；**AWS-IP-rotator 系**（数据中心+可识别头）RSC 无效。
2. **RSC 唯一有效供给 = 商业住宅/静态ISP 代理 + sticky（10–30min）**，成本仅几美元（金 OA <1GB）。
3. **编排层自建轻量 `ProxyLease`**（sticky + 按IP预算 + 探封轮换），复用 -165 per-host gate，借鉴 mubeng/scrapy-rotating-proxies 模式，**不整包引入**。
4. **最关键接线 = IP-会话亲和**：cf_clearance 绑 IP，浏览器铸盾与 curl_cffi 回放**同一住宅出口**。
5. **住宅 IP 必要非充分**：仍需 **-165 补丁①②③** 压制 governor 速率/行为软封。落地先做零成本的 -165 补丁验证，再挂住宅出口小样本 A/B（≥3 篇），**别一上来买代理**。
