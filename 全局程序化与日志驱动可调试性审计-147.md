## 二、② 日志结构化 / 逐 DOI 可读 / 一页总结充分性

### 2.1 强项（已达标）

| 能力 | 产物 | 说明 |
|---|---|---|
| 结构化事件流 | `fetch/attempts.jsonl` | `EventLog` 线程安全逐行 JSON：`input/resolved/source/located/download/result` 全程可回放 |
| 逐条总账 | `fetch/metadata.jsonl` | 每输入一行：`success/error/pdf_path/pdf_bytes/source_used/attempts[]` |
| 逐条人读 | `fetch/run.log` | 每 DOI 一行 `[OK]/[MISS] raw -> doi (source, Nms)` |
| **逐条可 grep** | `run_all_detail.tsv` | 每 DOI 一行 `status/doi/source/reason/pdf_filename/error/qc/title/pdf_path`；MISS 排前、`grep '^MISS'` 看全败 |
| 一页总结 | `run_all.log`(=屏幕) | `_render_page_lines` 单一真源双写；含 去重/续跑/本次/净覆盖/**失败原因分桶**/逐条样例(前50)/QC 剔除账 |
| 表格/网页 | `results.csv`+`report.html` | Excel 友好 + 浏览器总览/各源贡献/失败原因/明细 |
| 净口径 | `coverage.json.summary.qc` | 原始成功→剔→净，三层口径清晰（Runbook §5） |
| 编码健壮 | UTF-8 强制 | `run_all._force_utf8_console` + logsetup `errors=backslashreplace`，防中文乱码伤"读日志判断" |

### 2.2 缺口

- **G1〔P0〕`attempts.jsonl` 无逐源尝试链的人类可读渲染**：`report.html` 仅到"最终结果 + 各源贡献 + 失败原因分布"，逐 DOI 的**各源依次为何失败**只留一句"更细见 attempts.jsonl"，需手读 JSONL。用户"想知道这条为什么没下来"的**最后一公里未闭合**——**这正是 157 波曾派『日志可读性增强』被取消待重启的点**。
- **G2〔P0〕虚高口径告警不够硬**：`--no-qc`/黑名单缺失时净口径退化为盲口径（可能把 websearch 抓错论文当成功）。现状仅"[缺失]"字样 + "未启用QC,可能虚高"内嵌行，**易被忽略**；建议顶部 WARN 横幅。
- **G8〔P3〕无多轮趋势聚合**：每次 run 的 `coverage.json` 独立，缺"扫 `out/*/coverage.json` 出净成功率时间序列"的视图，跨波对比靠人工。

## 三、③ 调试闭环（改码 → 跑 → 读日志，无需手动介入）

### 3.1 强项（已达标）

- **全离线自检闭环**：`python run_all.py --selftest → RUN_ALL_OK`、`tools/build_coverage.py --selftest → COVERAGE_OK`、`run_all_selftests.py`（全包 compileall + 各模块 `*_OK`）——改码后**一条命令**判回归，不联网。
- **隔离调试开关**：`--no-download`（只定位不下载，秒验源命中）、`--sources`（隔离单源）、`--no-qc`（对比盲/净口径）。
- **单条复现已支持**：`python run_all.py "10.x/y" -o out/debug`（位置参数直传）即可复跑单条——**无需改数据**。
- **无人值守韧性**：`pipeline` straggler 看门狗——尾部卡死 N 秒无进展即判 `straggler-timeout` 失败并停等，**收尾必落 `summary/metadata/results.csv/report.html`**（回归已锁定），不因单条挂起整批。

### 3.2 缺口

- **G6〔P2〕route-B 调试断裂**：route-B 命中判定（CF vs governor、`blocked:*` 归类）目前需**真机有头 + 真 CF 站**才能验，"改码→跑→读日志"在这段不闭环。165 §四已设计**离线 governor fixture selftest**（注入 HTML fixture 断言 `_looks_governor`/冷却），**尚未落地**——落地后 route-B 判定逻辑即可离线回归，真机只验最后取字节一环。
- **G7〔P2〕QC 判定调试含**有意的**人工开卷环**：监管者169 SOP「回写必开卷」——QC 误杀/漏杀（假阴/假阳）的裁决需人工开 PDF 首页核对。这是**数据质量守门的有意设计**（websearch 假阳实锤在案），非缺陷；但违背"纯程序化"理想。建议**工具化开卷**（给 DOI → 自动抽 PDF 首页文本 + 期望 title 并排输出）降低人工成本，裁决仍留人。
- **G5〔已满足，仅需文档化〕**：单条复现无需专用 `--replay`，位置参数已够；Runbook 可显式点明"复现某条 = 直接把该 DOI 当参数跑"。
