## 〇、核心思路（代理为什么是自建抓 Scholar 栈最决定性、又最被高估的资源）

角度3 的「三层 + 两翼」检测模型里，**翼A（IP 信誉）是抓 Scholar 被封的首要触发点**：数据中心整段 ASN 信誉差、极易被封；短时多次 `/scholar?q=` 即触发 429 与 reCAPTCHA。所以**代理质量直接决定自建路线的天花板**。

但角度3 已给出**关键实测警示，本角度必须再次强调**：

> **代理 ≠ 万能。** 代理只改了**翼A（IP）**，而 **L1（TLS/HTTP2 指纹）、L2（JS/浏览器指纹）仍来自你的真机/容器**。实测中「未代理的 Mac」反而通过了「被住宅代理的 Linux 服务器」被拦死的站点——因为指纹露馅，换再多 IP 也被 JA4/JS 一眼识破。

**结论**：代理是自建路线的**必要非充分条件**，必须与角度3 的 `curl_cffi`（L1）/ `nodriver`、`Camoufox`、`Patchright`（L2/L3）**组合使用**，单独堆 IP 是浪费钱。

**三条路线的资源对照**：

| 维度 | 自建代理（角度6） | 商业整包服务（角度4） | 开放 API（角度2） |
| --- | --- | --- | --- |
| 你买的是什么 | **纯 IP 资源**（$/GB 或 $/IP） | 整包：IP+指纹+打码+解析（$/1k 结果） | 官方数据访问（多免费） |
| 谁扛过人机 | **你自己**（IP 这层外包，其余自理） | **服务商全扛** | 根本无验证码 |
| 持续成本 | 代理费 + **自己的工程/运维人力** | 按量/按月付费 | 多数免费/低价 |
| 灵活度 | 最高（想怎么抓怎么抓） | 中（受服务商接口约束） | 受 API 字段约束 |
| 适用 | 已有反爬栈、要可控、量大摊薄 | 不想养栈、要省心 | 能用官方数据就最优 |

## 一、代理类型选型（抓 Scholar 场景，呼应角度3）

| 类型 | IP 信誉 / 抗封 | 速度 | 成本 | 抓 Scholar 适用性 |
| --- | --- | --- | --- | --- |
| **数据中心（Datacenter）** | 低（整段 ASN 易被封） | 最快 | 最低（$0.6/GB 或 $1–3/IP/mo） | ❌ **不推荐**——抓 Scholar 极易整段封 |
| **住宅（Residential，轮换）** | 高（真实家庭 IP） | 中（~0.x–1s） | 中（$2–8/GB） | ✅ **主力**——每 5–15 次请求换 IP |
| **ISP / 静态住宅（Static Residential）** | 高 + **持久不轮换** | 快（机房速度 + 住宅信誉） | $2–3/IP/mo 起 | ✅ **养号/持久会话**（需保持 cookie 的场景） |
| **移动（Mobile 4G/5G）** | **最高**（多设备共享 NAT IP，几乎不敢封） | 中-慢 | 最高（$5+/GB 或 $100+/IP/mo） | ✅ **最难目标兜底**——被封最严重时用 |

> 一句话：**住宅轮换是抓 Scholar 的默认主力；要养号/保持会话用 ISP 静态；被封到怀疑人生时上移动；数据中心只配抓无防护页面。**（与角度3「翼A」结论一致）

## 三、逐个详解（按定位分档）

### 第一梯队：企业级、最稳最全（也最贵）
- **Bright Data**：市场池最大，但**住宅池规模营销口径混乱**〔152复核 2026-07〕——官网现称 **400M+ 住宅**（按月活 opt-in SDK 网络口径），第三方审计/评测多记 **72M+ 住宅、总池 150–175M+**；选型别迷信单一数字，以实测成功率为准。**geo 可精确到 ZIP/城市/ASN**，Proxy Manager 支持复杂轮换规则；合规最齐（SOC2 Type II / ISO27001 / GDPR / CCPA / KYC 门槛）。住宅定价（官网 2026-06）：PAYG 标 $8 / **券后 $4**，月付 $3.5(141GB)/$3.0(332GB)/$2.5(798GB)，1TB+ 企业议价更低（第三方记入门 ~$5.04/GB）；首充 1:1 匹配（≤$500）。**适合**：已有反爬栈、要最硬 IP + 合规背书的团队。注意：产品/SKU 极多（住宅/DC/ISP/移动各有 PAYG 与流量档），上量前务必用后台计算器按你的 zone 组合估算。
- **Oxylabs**：175M+ 住宅、成功率 99.95%、无限并发、**成功计费**（系统错误不收费）、专属客户经理。住宅 $6→$2.50/GB（1TB），PAYG $8/GB。**适合**：高强度、要 SLA 与人工支持的企业。

### 性价比自助派（认真抓 Scholar 的甜点区）
- **Decodo（原 Smartproxy）**：**透明线性定价** $3.75→$2/GB、无承诺、3 天试用、支持 rotating / sticky 会话。125M+ 池含住宅+移动+ISP。**适合**：中小团队认真抓 Scholar 的首选性价比。
- **SOAX**：**统一 credit 跨住宅/移动/ISP/DC** 灵活调配，**ASN 级 geo**，合规优先，成功率 99.5%+。住宅 PAYG $4→$2.00/GB（800GB）、企业 $0.32 起；$1.99/400MB 试用。**适合**：要混合代理类型、按目标域名调最优 IP 的研究项目。

### 持久会话 / 养号
- **NetNut**：one-hop 直连架构、速度快，**1M+ 静态 ISP**（always-on、强 US）适合**保持登录/cookie 的持久会话**。from $3.53/GB，年付企业 $1.59/GB；**无 PAYG**、需月承诺。
- **IPRoyal ISP**（$2.40/IP/mo）/ **Bright Data ISP**：同样用于需要固定出口 IP 的养号场景。

### 预算 / 灵活 / 试水
- **IPRoyal**：**流量永不过期**（不定期项目极省）、无最低消费，住宅 $7→bulk $1.75/GB；DC $1.39/IP（无限带宽）。池偏小、US 城市级有 ~6% 空池率，需注意 geo 精度。
- **Webshare**：**免费 10 代理 + 1GB（无需信用卡）**最适合零成本跑通 PoC；住宅促销低至 $1.40/GB、DC 订阅 $2.99/mo。但 **DC 质量偏低、无合规认证**，仅 PoC / 轻量。
- **DataImpulse**：住宅 **$1/GB** 价格地板，适合极度预算敏感的大批量。

## 四、轮换与会话策略（抓 Scholar 实操，与角度3 一致）

- **轮换频率**：每 **5–15 次请求换一个 IP**，配 **45–90s 抖动延迟**；触发 429 / 验证页立刻**指数退避**并换会话/IP，**别硬刚**（硬刚只会让该 IP 更快进黑名单）。
- **rotating vs sticky**：
  - **rotating（每请求/每隔几次换 IP）**：用于大面积「搜索结果页」抓取，分散指纹。
  - **sticky session（粘性会话，保持同一 IP 几分钟~1 小时）**：用于需要**保持 cookie / 翻页连续性 / 养号**的链路（Decodo/SOAX/Bright Data 均支持指定 session 时长）。
- **分散维度**：按 **ASN / 地域**分散出口，避免同一 ASN 段集中触发；UA / 指纹 / IP / cookie 组成的「身份」要**保持一致、别中途串味**。
- **再次强调**：以上都属翼A；**必须叠加角度3 的 L1/L2/L3 指纹对抗**，否则换 IP 无效。

## 七、与其他角度的关系（衔接点）

1. **→ 角度3（反爬技术备线）**：本角度是角度3「**翼A：IP 信誉/代理**」的**供应商级 + 自建级展开**。**强绑定**：代理必须与角度3 的 L1（`curl_cffi`）、L2/L3（`nodriver`/`Camoufox`/`Patchright`）指纹对抗**组合**，否则「换 IP 无效」（角度3 的核心实测）。
2. **→ 角度4（商业整包服务）**：**互补且边界清晰**——角度4 买「IP+指纹+打码+解析」整包（$/1k 结果），角度6 只买「IP 这一层」（$/GB），其余自理。**同一批厂商**（Bright Data / Oxylabs / SOAX / Decodo）**两边都出现**：在角度4 是其 SERP/Web Unlocker/Scraper API 产品，在角度6 是其裸代理产品。决策轴 = **「要不要自己维护反爬栈」**。
3. **→ 角度1（GitHub 项目）**：角度1 的 `scholarly`（`ProxyGenerator` 原生支持 Bright Data/ScraperAPI）、`PyPaperBot`（`--proxy`）、`scrapy-spider`（ScraperAPI）等正是代理的**消费方**；自建 `proxy_pool` 可作为它们的代理后端。
4. **→ 角度2（开放 API 主线）**：**能走官方 API 就别折腾代理**。代理基础设施是「确需直接抓 Scholar」时才投入的资源；角度2（无验证码、无需代理）始终是更优主线。

## 八、对工作组目标的结论

1. **代理是自建抓 Scholar 的决定性资源，但只是「必要非充分」**：决定天花板的是「住宅/移动 IP 质量 + 指纹对抗」的组合，**单买 IP 不配指纹对抗等于白花钱**（角度3 实测）。
2. **认真做就买住宅**：中小量首选 **Decodo / SOAX（$2–3.6/GB 起）**；企业级 **Bright Data / Oxylabs**；养号用 **ISP 静态（NetNut/IPRoyal）**；最难目标上**移动代理**。
3. **免费/自建代理池抓 Scholar 基本不可行**：`jhao104/proxy_pool` 等只适合学机制/兜底；其「轮换框架」可复用，但必须喂商业住宅 IP 才有产能。
4. **成本权衡（三选一）**：自建代理（买 IP + 自养反爬栈，角度6+3）／买整包服务（角度4）／走免费开放 API（角度2）。**除非必须拿 Scholar 原生被引/版本数据，否则优先角度2**；要自研可控、且量大能摊薄运维，再选角度6。
5. **落地建议**：免费/试用跑通 PoC → Decodo/SOAX 住宅主力 + 每 5–15 次换 IP + 45–90s 抖动 + 指数退避 → 叠加角度3 指纹对抗 → 难目标上移动 / 养号上 ISP 静态。
