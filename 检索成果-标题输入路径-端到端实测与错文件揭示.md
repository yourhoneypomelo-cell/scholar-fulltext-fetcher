## 一句话结论

**标题→DOI 解析 20/20 全对(100%),resolve 路径无短板;短板全在下载侧的"内容对不对":
内容级复核后诚实端到端正确率 16/20(80%),2 条实锤错文件(openaire 仓库错挂)+
2 条 websearch 兜底存疑——且那 2 条错文件同样污染着 batch6,现有 QC(只扫 websearch)全部漏过。**

## 样本与做法

- 20 篇标题:混 13 家出版社(elsevier/acs/wiley/rsc/nature/mdpi/science/springer/frontiers/aps/pnas/bmc/other),
  含付费墙(ACS×2、APS)与新旧文献。真值取自历史成功记录中 DOI-keyed 的 (标题,DOI) 权威对。
- 输入为**纯标题**(非 DOI),`python -m fulltext_fetcher -f titles_probe.txt -o out/title_probe`,
  并发 4,用时 219.7s(平均 11s/篇)。

## 指标(三层口径,由松到严)

| 口径 | 结果 | 说明 |
|---|---|---|
| 标题→DOI 严格解析命中(解析DOI==真值DOI) | **20/20 = 100%** | 无解析错、无解析不到 |
| 端到端(title_probe 口径:严格命中 AND 落盘) | 20/20 = 100% | 但此口径不看 PDF 内容 |
| **内容级诚实口径(命中 AND 内容 match AND 非重复文件)** | **16/20 = 80%** | 复用 qc_content_match.classify |

内容级 verdict 分布:match 16 / mismatch 2 / uncertain 1 / scanned 1。

## 失败模式(全部在下载侧,resolve 零失误)

1. **openaire 仓库错挂(实锤错文件,2 条)**:`10.1016/j.susc.2014.02.019` 与
   `10.1016/j.apcata.2013.07.028` 两个不同的 Elsevier DOI,被 openaire 指到**同一个**
   筑波大学仓库文件 `tsukuba.repo.nii.ac.jp/record/38855/...CPL_655/656.pdf`(实际内容是
   Chemical Physics Letters 的"水渗透醋酸纤维素膜"论文,与两篇均无关)。两条 PDF MD5 相同。
   - **强错误信号:同一 run 内同一 pdf_url / 同 MD5 服务多个 DOI。**
2. **websearch 兜底质量差(存疑 2 条,均为付费 ACS)**:
   - `10.1021/acsami.2c01959` → 拿到 arXiv 相关他文(Distinguishing Dynamic Phase Catalysis...),
     题目重叠 63.4 分,疑似错论文;pipeline 已自标 `qc_uncertain`(存疑放行)✓
   - `10.1021/nl200722z` → 拿到 `tile.loc.gov`(美国国会图书馆 FRD 档案)扫描件,0 分无正文,
     基本可断定错文件;pipeline 已自标 `qc_uncertain` ✓
3. 解析错 / 解析不到 / 下载失败(无文件):均为 0。

## 连带发现(影响存量批次,已实锤)

- **batch6 同两条 DOI 也是同一错文件**(`out/batch6/metadata.jsonl` 中 pdf_url 完全相同,success=true)。
- 现有内容 QC(`tools/qc_content_match.py`)**只扫 source_used 含 websearch 的记录**,
  openaire 等 DOI-keyed 来源的错文件完全在盲区;URL域×DOI前缀审计法也拦不住(tsukuba 是仓库域,判不了对错)。

## resolve 路径短板与建议

- resolve 本身:本轮无短板(20/20)。运行中 openalex 503 连续退避、pmc idconv 429、
  ri.conicet.gov.ar 熔断均按设计工作,靠 crossref/unpaywall 等冗余源补齐,未影响命中。
- 建议(按优先级):
  1. **内容级 QC 扩展到所有来源**(至少 repository 域),别只扫 websearch——本轮 2 条实锤错文件全是 DOI-keyed 来源。
  2. **download 层加"重复 URL/重复 MD5 跨 DOI"守卫**:同 run 内同一 pdf_url 或同 MD5 被第二个 DOI 复用时降级/告警。
  3. websearch 对付费文献的兜底维持"存疑放行+qc_uncertain 标记"设计即可,审计时按标记复核。
  4. batch6 的 2 条错文件建议走 `tools/qc_move_rejected.py` 隔离并进回收清单。
