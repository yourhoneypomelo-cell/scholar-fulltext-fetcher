## 一、项目定位与要解决的问题

**工作组课题**:研究"绕过 Google Scholar(谷歌学术)的人机检测认证、抓取文献元数据并下载 PDF"的全部可行路线。

项目从 **7 个角度**系统检索了所有路线(GitHub 开源爬虫 / 官方开放 API / 反爬反 reCAPTCHA / 商业解锁服务 / 即用型工具 / 代理基础设施 / 中文社区镜像站),聚合成一份**结论先行的选型决策书**,并给出**可运行的落地实现**。

**核心结论(一句话)**:**能用「官方开放 API」(OpenAlex / Semantic Scholar / Crossref / Unpaywall)就别去硬抓 Scholar**——开放 API 没有验证码、合规、免费/低价、稳定,且能拿到同样的元数据与开放获取(OA)PDF。只有非要 Scholar 原生口径(被引数/版本)时才考虑付费买断或自建反爬;**付费墙内的全文任何免费路线都拿不到**,属合理边界(靠机构订阅 / 馆际互借解决)。

因此本仓库有两层产物:

1. **研究层**:七角度检索报告 + 聚合选型决策书(`检索成果-*.md`)。
2. **工程层(★核心交付物)**:`fulltext_fetcher/` —— 一个**生产级多源开放获取全文抓取包**。它要解决的具体问题是:

> **输入 DOI / 标题 / arXiv id → 全自动榨干全网可及的免费全文 → 校验并下载入库**,追求高成功率 + 高速率 + 全程程序化(只靠"运行 + 读日志"即可迭代,无需人工盯)。

## 三、核心功能与数据处理流程(全链路 pipeline)

`fulltext_fetcher` 的主链路:**输入 → 解析富化 → 多源回退定位 → 下载校验 → 落盘 + 结构化日志**。

```
输入(DOI / 标题 / arXiv id / 各种 URL 形态)
   │  resolve.classify_input:正则判类型(arxiv: / arxiv.org / doi.org / 10.x / 旧式 arXiv id → 否则当标题)
   │  resolve.resolve_to_paper:富化元数据
   │    · DOI   → OpenAlex 单条(免费)富化,失败回退 Crossref
   │    · arXiv → 构造 DOI 10.48550/arxiv.{id} 再走 OpenAlex
   │    · 标题  → Crossref bibliographic 反查 DOI(词集 Jaccard 相似度≥0.6,否则取第一条)再富化
   ▼
Paper{doi, title, year, authors, arxiv_id, pmid, pmcid, is_oa, oa_status, journal, resolved_via}
   │  pipeline._gather_and_download:按 config.DEFAULT_SOURCE_ORDER 逐源遍历
   │    · 每源先 applicable(paper) 过滤(如 biorxiv 仅 10.1101/;arxiv 仅 arXiv/无 DOI 标题)
   │    · find_candidates 内部吞掉自身异常(单源失败不拖垮全局),编排器独立计时并记 Attempt
   │    · 每源产出 PdfCandidate[](url + kind + confidence 0–100)
   ▼
   │  直链候选(kind=pdf/render)按 confidence 降序「即时下载」
   │    → download.download_pdf 校验:HTTP 200 → 非 text/html → 前 1KB 含 %PDF 魔数 → 大小≥1KB 且≤80MB
   │    → 若返回 HTML 落地页:landing.extract_pdf_links 抠内嵌 PDF 直链再下一层(防递归)
   │  ✔ 第一个成功下载即「短路」return(省掉后续源请求 = 提速)
   │  落地页候选(kind=landing/file)累积,全部直链走完后才兜底尝试(--oa-only 则跳过)
   ▼
落盘 out/pdfs/<DOI 或 arxiv_id 净化>.pdf + 三类结构化产物:
   attempts.jsonl(每事件一行:input/resolved/source/download/result/located/resolve_error)
   metadata.jsonl(每条最终 FetchResult,断点续跑依据)
   summary.json(本次运行 processed/success/success_rate/by_source/elapsed_sec)
   run.log(人类可读日志,UTF-8)
```

**关键设计决策(为什么这么做):**

1. **源优先级 + 首个直链成功即短路**:源按"直链命中率/速度/稳定性"综合排序;命中靠前源的一条 DOI 仅需数秒,既保证回退覆盖(高成功率)又避免无谓请求(高速率)。
2. **候选打分(confidence 0–100)**:直链(pdf/render)高分优先、落地页低分仅兜底;本地快照命中给最高分(96)。pipeline 按 `(is_direct, confidence)` 排序决定尝试顺序。
3. **PDF 校验防"落地页冒充 PDF"**:必须 HTTP 200 + 非 HTML + 前 1KB 含 `%PDF` 魔数 + 大小达标,任一不过即判失败并记原因(`http-403`/`landing-page`/`not-pdf`/`too-small`/`too-large`/`no-response`…),绝不把 HTML 当 PDF 落盘。
4. **落地页内 PDF 二次解析**:对"拿到 HTML 落地页而非直链"的情况,`landing.py` 从 `<meta name=citation_pdf_url>`(最高置信)与 `<a>/<embed>/<iframe>` 的 `.pdf`/`/pdf` 链接中提取真实直链,回收一部分 bronze/绿色 OA;`download.py` 收到 HTML 时会自动调用它再下一层(`allow_landing=False` 防递归)。
5. **按域限速 + 重试退避 + 熔断器**:每 host 最小请求间隔(默认 0.34s,arXiv 强制 3s);429/5xx 指数退避并尊重 `Retry-After`(上限 30s);某 host 连续 **3** 次(非 SSL)连接异常即本次运行内**熔断跳过**(把环境内连不通的 host 从反复超时降到近 0 开销;阈值原为 2、后提至 3 以减少高峰误熔,SSL 瞬时抖动不计入)。多线程共享同一 client 与限速状态。
6. **元数据富化减少重复定位**:resolve 阶段用 OpenAlex 一次性拿 pmcid/pmid/arxiv_id/is_oa 等,下游 Europe PMC / PMC / arXiv 直接命中,省去各源自己再定位。
7. **全程程序化 + 断点续跑**:所有状态进结构化日志;对同一 `--out` 重复运行会读 `metadata.jsonl` 自动跳过已成功项、只重试失败(`--no-resume` 关闭),支撑"改程序 → 重跑 → 读日志"的闭环。
8. **本地快照零联网线**:配置 `--snapshot-db` 后,DOI 先查本地 SQLite 快照(由官方 OpenAlex CC0 / Unpaywall Data Feed 转储灌入),命中即省去全部在线 API 调用——彻底绕开免费 API 的额度/限速且完全合规。
9. **只定位不下载模式**:`--no-download` 时每源照常产出候选并记录,`success` 表示"是否定位到候选",用于快速验证各源命中率(诊断神器)。

## 五、当前实现状态与验证结果

**实现状态**:`fulltext_fetcher` 已是可用的生产级包(零第三方依赖,仅 `requests`),具备多源回退、并发、按域限速、熔断、断点续跑、CLI + 编程 API + 子进程 JSON 三种接入、本地快照零联网线、落地页二次解析。`landing.py` 自带 selftest(`python -m ...` 直接跑断言,输出 `SELFTEST_OK`)。

**规模化验证(同一 500 DOI 语料 · RWGS/CO₂ 加氢催化方向 · 订阅型期刊为主 · 数据集 `title_doi_500_batch_6.xlsx`)——两种口径:**

| 口径 | 成功 / 处理 | 成功率 | 主力贡献源 |
|---|---|---|---|
| **现行:全部 22 源(含 websearch/publisher_oa)** | **410 / 500**(`out/batch6`) | **~82%** | websearch ≈72% + publisher_oa(ACS 等)+ 聚合 OA-API |
| 早期基线:仅聚合 OA-API 源(`val500_out`) | 54 / 498 | 10.8% | unpaywall 23、semantic_scholar 20、openaire 5、crossref 3、europe_pmc 2、hal 1 |

> `out/batch7` 对 213 条子集重跑得 108/213(50.7%,websearch 99);与 batch6 去重合并 **412/500 = 82.4%**。**10.8% → ~82% 的跃升几乎全部来自 `websearch`(搜索引擎找作者自存稿/机构库副本)与 `publisher_oa`**——"免费公开边界"的真实天花板远高于"仅聚合 OA-API"口径,后者是早期未启用 websearch/publisher_oa 时的基线。

**早期 OA-API-only 基线细分(`val500_out`,供对照):**
- **各源候选命中率**:crossref 47.9%(217/453,最高但成功仅 3——候选多为付费链 403)、unpaywall 23.7%、semantic_scholar 20.7%、openalex 20.0%、openaire 7.6%、europe_pmc 4.6%、doaj 3.1%、hal 0.2%,pmc/core/zenodo 本语料 0%。
- **按出版商成功率**:Springer Nature 37.0%、MDPI 13.3%、Wiley 9.5%、ACS 8.6%、RSC 7.7%、Elsevier 7.5%(纯订阅刊偏低;开放/混合 OA 占比高者更高)。
- **下载失败原因 Top**:crossref http-403(191)、unpaywall http-403(78)、europe_pmc no-response(34)、unpaywall no-response(33)、各源 landing-page 若干。总耗时 14.6 分钟(并发 12、限速 0.2s)。

**结论与边界:**
- **免费公开边界内实测 ~82%**(全部 22 源,同一语料);**websearch(搜索引擎发现作者自存稿/机构库 PDF)是主力(约 72%)**,publisher_oa 次之;均经 `%PDF` 魔数与标题校验过滤伪装/错配。
- **10.8% 是早期"仅聚合 OA-API 源"口径**(未启用 websearch/publisher_oa),两者口径不同、勿直接相比;当时对纯订阅论文返回"无免费版"是该口径下的正确边界。
- **付费墙内全文仍是物理边界**:连 websearch 也找不到自存副本的纯订阅全文,根本解法是机构订阅(见 `检索成果-00`)或商业解锁(角度4)。
- **历史增益记录**:集成 `landing.py`(落地页 `citation_pdf_url`/内嵌 PDF 解析)后,同一 80 条样本 11.2% → 13.8%(OA-API 口径下的回收)。

**验证产物留存说明**:`val500_out/` 完整留存 **证据链 + 正文**——`pdfs/`(本次成功下载的 **54 篇 PDF 正文**,按 DOI 净化命名)、`summary.json`、`metadata.jsonl`、`attempts.jsonl`、`references.bib`、`zotero.csl.json`、`run.log`,证据链与 54 篇文献清单(可直接导入 Zotero)完整可查、正文可直接取用(已成功项均为 OA,可凭 DOI 稳定复现)。

### 1) 命令行(最常用)


## 七、谷歌学术爬虫子系统(fulltext_fetcher/scholar/)

> 在"绕开 Scholar、走开放 API"主线之外,本子系统**正面实现**:**输入标题/DOI →(避免或自动过人机验证)→ 抓 Google Scholar 元数据 + Scholar 可及 PDF → 文件名标准化 → 一键批量**,全程结构化日志闭环。架构与接口契约见 `谷歌学术爬虫-架构与选型.md`,详细用法见 `谷歌学术爬虫-使用说明.md`,踩坑见 `经验记录-踩坑与发现.md` F 节。

### 7.1 定位与"和父包 OA"的关系
- **目标**:拿 Scholar 原生口径(被引/版本/作者/年)+ Scholar 页可及的 PDF 直链,全程程序化、结构化日志。
- **与父包关系**:父包 OA 多源(Unpaywall/OpenAlex/…)在本子系统里作为"**其它渠道兜底**",**默认关**(`--oa-fallback` 开启才用);Scholar 无直链或付费墙时才回退,复用父包 `download/landing/sources/resolve/report`,不重造轮子。

### 7.2 架构:三层 + 两翼反爬(默认最安全)
- **分层取回降级**:L1 `curl_cffi`(TLS 指纹静态抓)→ L3 `nodriver`(直连 CDP 浏览器)→ `patchright`(备选);命中验证码/风控即升级,并按开关换代理 + 退避 + 冷却。
- **两翼(默认全关)**:翼A 住宅代理池(`proxy.py`,轮换/拉黑)、翼B reCAPTCHA 打码(`captcha.py`,2captcha/capsolver 适配)。**避免触发优先,不硬刚**。
- **模式 `--mode`(默认 `auto`)**:有 SerpApi key → `serpapi`(商业合规、无需自过验证码);否则 `self`(自建反爬栈)。
- **可选依赖全部函数内延迟导入**(curl_cffi/nodriver/patchright/2captcha-python/python3-capsolver),缺失即优雅降级、**不进强制依赖**;限速/退避/冷却默认从严(页间 45–90s 抖动、退避 base2/cap60s、命中风控冷却 900s、默认串行)。

### 7.3 模块清单(`fulltext_fetcher/scholar/`,共 14 个 .py + `__init__`)
`models`(数据类 ScholarResult/ScholarQuery/SerpPage/FetchOutcome/ScholarFetchResult)· `config`(ScholarConfig 全字段,代理/打码/scihub 默认关)· `logsetup`(复用父包 + 事件名常量)· `query`(build_query/title_match_score)· `serp`(build_scholar_url/parse_serp/parse_serpapi/detect_captcha,纯 html.parser)· `fetcher`(分层引擎 + fetch_serp/fetch_html)· `proxy` · `captcha` · `download`(download_result_pdf + oa_fallback,复用父包)· `naming`(build_filename `{year}_{author}_{title}` + 去重)· `pipeline`(ScholarPipeline.run 编排)· `cli`/`__main__`(`python -m fulltext_fetcher.scholar`)· `selftest_e2e`(不联网端到端)。

### 7.4 如何运行

```bash
# 单条(标题或 DOI)/ 批量 -f(.txt/.csv/.xlsx,复用父包读取器)
python -m fulltext_fetcher.scholar "Attention is all you need"
python -m fulltext_fetcher.scholar "10.1038/nature14539" --num 5
python -m fulltext_fetcher.scholar -f titles.txt --oa-fallback

# 真实抓取(二选一,见 7.5 边界):SerpApi key 或 住宅代理
python -m fulltext_fetcher.scholar "…" --serpapi-key <KEY>     # 或环境变量 SERPAPI_KEY
python -m fulltext_fetcher.scholar "…" --proxy                 # 代理池经 SCHOLAR_PROXIES

# 不联网端到端自检
python -m fulltext_fetcher.scholar --selftest                  # → SCHOLAR_E2E_OK
```

产物落 `out_scholar/`:`pdfs/`(标准化命名)+ `serp/`(原始 SERP 快照)+ `metadata.jsonl` + `serp.jsonl`(抓取事件)+ `attempts.jsonl`(下载事件)+ `summary.json`(含 by_source/by_engine/blocked)+ `results.csv` + `report.html` + `run.log`。

### 7.5 现状与边界(如实)
- **代码全绿**:各模块内置离线 selftest + 端到端 `SCHOLAR_E2E_OK` 全通过;一键回归 `python run_all_selftests.py` → **PASS=36 / FAIL=0 / SKIP=0(total=36)**;无新增强制依赖。
- **真实抓取的物理边界**:**数据中心 / 机房 IP 直连 Scholar 几乎必被 HTTP 403**(实测即便 `curl_cffi` 伪装 Chrome TLS 指纹也即刻 403)——这是 **IP 信誉**边界,非程序缺陷。要真实抓到结果,需 **SerpApi key** 或 **住宅代理**;被拦时程序不硬刚,记 `blocked` 事件后按开关冷却/换代理/升级浏览器。
- **付费墙是物理边界**:付费墙内全文任何免费手段都拿不到,靠机构订阅 / 其它渠道。
- **可选异步**:父包新增 `fulltext_fetcher/aio.py`(C4 异步并发原型,httpx/aiohttp 可选、同步核心不变),用于大批量"定位/元数据"阶段提吞吐。

### 7.6 合规
优先正门(开放 API / 商业合规 SerpApi);直抓 Scholar 属灰色(违反 ToS 与 robots.txt),**默认最安全**(代理/打码/Sci-Hub/OA 兜底全关),使用者自负;注意 `nodriver`=AGPL 等许可证传染。详见各模块头部与 `谷歌学术爬虫-架构与选型.md` §8。
