## 一、样本、口径与两个分母(务必先看)

分析涉及两个不同分母,全文分别标注,不可混用:

| 分母 | 含义 | 规模 |
|---|---|---|
| **按条(record)** | `metadata.jsonl` 每条输入一条终态记录 | 498 条(成功 54 / 失败 444) |
| **按下载尝试(attempt)** | `attempts.jsonl` 中 `event=download` 事件;一条记录可能触发多次下载尝试(多源、多候选、重试) | 541 次(成功 54 / 失败 **487**) |

**关键前置口径 —— "定位到候选" ≠ "存在免费全文":**
`candidates>0`(定位到候选)= 至少有一个源返回了候选 URL,**其中包含 Crossref/Unpaywall 回传的出版商 TDM / 付费链接**(如 `api.wiley.com/...`、ACS/RSC 出版商页)。这些链接一律计入"定位到候选",但下载时多半 403。因此"定位到却下不动"里既有**可回收的软失败**(落地页/超时/软封),也有**真付费墙**,下文会拆开。

## 二、失败的宏观结构(按条,444 条失败)

| 失败大类 | 条数 | 占失败 | 可回收性 |
|---|---|---|---|
| **定位到却下不动**(candidates>0 且未成功) | **248** | 55.9% | 部分可回收(A2/A3/基础设施) |
| **免费源无候选**(candidates=0) | **196** | 44.1% | 免费源天花板,需机构/商业(角度4) |

> 校验:248 + 196 = 444 = 总失败;302 定位到候选 − 54 成功 = 248 下不动。与《规模化验证报告》"定位到候选 302(其中定位到却下不动 248)"逐字吻合。

**结论 1:** 免费源 10.8% 的天花板里,**约 44% 的失败是"根本没有免费候选"**——这是订阅型语料(Elsevier/ACS/RSC 为主)的正确边界,不是程序缺陷。真正值得工程优化的是另外 **248 条"定位到却下不动"**。

## 三、下载失败原因分布(按下载尝试,487 次失败)

| 原因 | 次数 | 占失败尝试 | 含义 / 归因 |
|---|---|---|---|
| **http-403** | **300** | 61.6% | 出版商付费墙 / 反爬拒绝(付费链或被封) |
| **no-response(retries-exhausted)** | **85** | 17.5% | 连接超时、重试耗尽(网络/限速/源慢) |
| **landing-page(HTML,无内嵌 PDF)** | **79** | 16.2% | 拿到 HTML 落地页而非 PDF——**A2 landing.py 的靶心** |
| http-400 | 11 | 2.3% | 请求被拒(Wiley TDM API 缺 token,见下) |
| not-pdf(响应是图片/JFIF) | 6 | 1.2% | 下到的是图片(封面/验证图)而非 PDF |
| http-202 | 3 | 0.6% | 服务端异步占位(未直接给文件) |
| http-405 / 404 / 401 | 各 1 | 各 0.2% | 方法不允许 / 不存在 / 需鉴权 |

> `landing-page` 79 = 各 charset 变体合并(`; charset=utf-8` 38 +`;charset=utf-8` 31 +`; charset="utf-8"` 6 +无 charset 3 +`iso-8859-1` 1)。

**结论 2:** 失败尝试的三大主因是 **403(61.6%)、超时(17.5%)、HTML 落地页(16.2%)**,合计 95%。403 里含大量真付费墙;超时靠重试/代理;**落地页是纯工程可回收项**。

## 七、"最值得回收"清单:按回收路径分组(248 条下不动的记录级归因)

对 **248 条"下不动"记录**做**互斥优先级归因**(一条记录取其最可回收的失败原因):

| 记录级归因 | 条数 | 占下不动 | 回收路径 |
|---|---|---|---|
| **仅 http-403** | **149** | 60.1% | 多为真付费墙 / 硬封 → 机构订阅 / 商业(角度4) |
| **landing-page(HTML)** | **54** | 21.8% | **A2 landing.py**(解析 `citation_pdf_url`/内嵌 PDF) |
| no-response(超时) | 29 | 11.7% | 基础设施:重试/超时/代理 |
| **http-400** | **11** | 4.4% | **A3 Wiley 适配器**(TDM API 需 token) |
| not-pdf / 其它 http | ~5 | ~2% | A3 适配器:识别图片/异步响应 |

### 7.1 A2 靶心 —— HTML 落地页(54 条可回收)按出版商

| 出版商 | 可回收(landing-page)条数 |
|---|---|
| **Elsevier** | **23** |
| Springer | 9 |
| Springer Nature | 6 |
| Springer/Kluwer(10.1023) | 4 |
| ACS | 4 |
| RSC | 3 |
| Wiley / IEEE / IOP / Pleiades / 其它 | 各 1 |

> **Elsevier + Springer 系(Springer+Nature+Kluwer=19)合计 42 条,占 landing-page 可回收的 78%。** 优先实现 Elsevier(ScienceDirect `pdf.sciencedirect.com` / `citation_pdf_url`)与 Springer 系落地页解析,即可吃下 A2 的大头。与《规模化验证报告》"集成 landing.py 后 80 样本 11.2%→13.8%"的实测增益方向一致。

### 7.2 A3 靶心 —— 出版商适配器

- **Wiley `http-400`(11 次尝试 / 11 条)**:候选是 Crossref 回传的 `api.wiley.com/onlinelibrary/tdm/...`,需 `Wiley-TDM-Client-Token` 头,否则 400。**加 Wiley TDM 适配器**即可回收。
- **MDPI 反爬软封(9 条 only-403)**:MDPI 是**金色 OA**,正常应近 100% 可下;却出现 9 条纯 403 → **是反爬软封而非付费墙**。补**规范 UA/Referer/`Accept` 头 + 代理**即可回收,性价比最高(内容本就免费)。
- **not-pdf(图片/JFIF)**:识别响应为图片时不落盘、改走下一候选。

### 7.3 真付费墙(免费源难回收,需角度4/机构)

`仅 http-403` 149 条中,**ACS 80 + RSC 41 = 121 条(81%)** 为订阅刊真付费墙;另有 Elsevier 5、Wiley 4、AIP 4、AAAS 2 等。此部分**不应再投入免费源工程**,应交由**机构订阅集成 / 商业 ScholarAPI(角度4)**。

## 八、结论与优化优先级(可执行排序)

**免费源天花板 10.8% 的成因结构(444 失败):**
无免费候选 196 条(44.1%,硬边界)+ 仅 403 真付费墙 149 条(33.6%)= 345 条(77.7%)→ **约 3/4 的失败需要机构/商业才能突破**;剩余约 1/4(≈103 条)才是纯工程可回收。

**可回收部分(约 103 条 / 占失败 23%)的落地优先级:**

| 优先级 | 动作 | 可回收(条,记录级上限) | 成本 |
|---|---|---|---|
| **P0** | A2 landing.py 落地页解析(先 Elsevier + Springer 系) | 54(其中 42 集中在两家) | 低,已验证 +2.6pp |
| **P1** | A3 MDPI/通用反爬头 + 代理(软封回收) | ~9(MDPI)+ 部分 403 软封 | 低 |
| **P2** | A3 Wiley TDM token 适配器 | 11 | 低-中(需申请 token) |
| **P3** | 基础设施:重试/超时/代理治理 | ~29(no-response) | 中 |
| **P4** | 选源打分:下调 Crossref 候选权重、上调 SS/OpenAIRE | 减少无效下载/限速 | 低 |

**不建议免费源硬刚的部分:** ACS/RSC/Elsevier 的真 403 付费墙(≈121+ 条)→ 走**机构订阅集成**(见《机构订阅集成设计》)或**角度4 商业服务**(见《检索成果-00 决策》)。

**乐观上限测算(仅供排序参考,非承诺):** 若 P0–P3 全部落地并全部命中,理论上再补 ~103 条,成功率上限约 (54+103)/498 ≈ **31%**;现实中落地页/重试不会 100% 命中,故合理预期是在 landing.py 已验证的 **13.8%** 基础上,叠加 MDPI/Wiley/重试再抬升数个百分点。

## 附:复算方法(可由原始数据复现)

1. **样本**:逐行 `json.loads` 读取 `val500_out/metadata.jsonl`(498 条)与 `attempts.jsonl`(7128 事件)。
2. **宏观结构(第二节)**:`success = metadata.success==true`;`located = candidates>0`;`stuck = candidates>0 且 success==false`;`no-candidate = candidates==0`。
3. **下载原因(第三/四节)**:取 `attempts.jsonl` 中 `event=="download" 且 ok==false` 的 `error` 字段;`landing-page(...)` 按前缀归并、`no-response(...)` 归并、`not-pdf(...)` 归并。
4. **各源转化率(第五节)**:同一批 `event=="download"` 事件按 `source` 分组,`ok==true` 计成功。
5. **出版商分档(第六节)**:DOI 前缀(`doi.split('/')[0]`)映射出版商(10.1016→Elsevier、10.1021→ACS、10.1039→RSC、10.1002→Wiley、10.1038→Springer Nature、10.1007→Springer、10.3390→MDPI …)。
6. **记录级归因(第七节)**:对每条 stuck 记录聚合其全部失败下载 `error` 集合,按优先级 `landing-page > http-400 > not-pdf > no-response > 仅 http-403` 归入互斥桶。
7. 全部数字与 `summary.json`(processed 498 / success 54 / rate 0.10843)及《项目总报告》《规模化验证报告》自洽。
