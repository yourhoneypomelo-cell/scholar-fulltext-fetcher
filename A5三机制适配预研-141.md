## 〇、TL;DR（核心洞察）

- **三机制的差异只在「登录取 cookie」这一步的 `login_url` 构造 + detect 启发式**；一旦 cookie 落进 `CookieStore`，**注入 → 下载 → 内容 QC 全部共用同一条 route-B 链**（`plan_route_b_injection` → `inject_institutional_session` → B1 页内 `fetch` / B2 `viewerfetch`）。故适配三机制 = 在**已就绪的共享层**上补三个 `run_<mechanism>_login` 变体。
- **EZproxy 已基本就绪**（`ezproxy.py` 双模式改写 + `run_ezproxy_login` 范式），只差联网实测；**Shibboleth/OpenAthens 与 CARSI/WebVPN 待补 `run_saml_login` / `run_carsi_login`**（-153），二者与 EZproxy 共用 `login_browser.py` 鸭子接口。
- **CARSI = Shibboleth+SAML 的中国联邦特例**：协议同 SAML 2.0 Web Browser SSO，唯一增量是 **IdP 发现走 CARSI DS（`ds.carsi.edu.cn`）+ 用户选中国高校**。因此 `run_carsi_login` 可复用 `run_saml_login`，仅 `login_url` 默认指向 CARSI DS、detect 需把 `carsi.edu.cn` 也算「未完成」。
- **开源参考（2026-07-03 复核）**：**instsci**（MIT、star 96、覆盖 CARSI/WebVPN/EZproxy、出版商工作流最全）**最值得借鉴设计**；**scansci-pdf v1.6.1**（Apache-2.0、CARSI/WebVPN 强项）次之；**SciTeX-Scholar v1.4.1**（**AGPL-3.0 传染性许可**、provider 抽象 + `verify_live` 设计优秀，但**只可借思路、不可引依赖/抄码**）。本仓延续 168 结论：**借设计不借依赖、执行底座坚持 nodriver + route-B**。

## 二、机制一：EZproxy（URL 重写代理）— 已就绪，缺联网实测

### 2.1 协议 / 流程
- **本质**：中间件，动态改写网页内 URL 主机名，使后续所有请求都回到代理服务器。两种模式：
  - **proxy-by-hostname（现代默认）**：`www.sciencedirect.com` → `www-sciencedirect-com.ezproxy.uni.edu`（`.`→`-`，本仓 `ezproxy.py` 已实现此改写）。
  - port-mapping（旧、已弃）：靠端口区分，cookie 兼容差。
- **认证流程**：校外 IP 访问 → EZproxy 拦截 → 302 到登录（library barcode / CGI / 校 SSO）→ 验证 → 签发 session cookie → 302 回**改写后的**资源 URL。
- **CGI/Ticket 认证**：可把认证委托给外部脚本，认证后用带签名的 Ticket URL 回 EZproxy 建会话。

### 2.2 cookie / session 获取点
- 登录成功后 EZproxy 在**代理服务器域**（`login.ezproxy.YOUR-UNI.edu`）种 cookie：
  - `ezproxy`（classic，无 SameSite/Secure/HttpOnly，兼容老浏览器）
  - `ezproxyN`（`SameSite=None; Secure; HttpOnly`，支持第三方 iframe/https）
  - `ezproxyL`（`SameSite=Lax; HttpOnly`）
  - 三者存同一 session 信息，指令不同。**抓取时三者都取**（`get_valid_cookies` 全存）。
- 注入时 domain 用**代理服务器域**（非出版商域），因为访问的是改写后的 `*.ezproxy.uni.edu`。

### 2.3 与 route-B 的对接点
- **URL 改写**：走 `RouteBInjectionPlan.ezproxy_prefix` / `rewrite_target_host`，由 render_fetch 的 `rewrite_url_for_injection_plan` 把 `target_pdf_url` 改写成代理 URL 后再导航。
- **注入**：`ezproxy`/`ezproxyN`/`ezproxyL` 三 cookie 注入 nodriver context（domain=代理域）。
- **下载**：改写后 URL 落到 `*-sciencedirect-com.ezproxy.uni.edu/.../pdfft` → B1 页内 `fetch` 直下。

### 2.4 凭据字段需求
| 字段 | 来源 | 说明 |
|---|---|---|
| `provider="ezproxy"` | 用户配置 | `AuthProvider.EZPROXY` |
| `ezproxy_prefix` | 用户填 | 形如 `https://login.ezproxy.uni.edu/login?url=`（登录入口 + `{url}` 占位） |
| `institution_cookie` 或 CookieStore | 登录后自动 | `ezproxy=...; ezproxyN=...`（`run_ezproxy_login` 抓取回填） |
| `institution_domains` | 用户填 | 注入白名单（如 `sciencedirect.com,pubs.acs.org`） |

### 2.5 待补（-153）
- `run_ezproxy_login` **已实现**，仅缺：nodriver 联网 E2E 实测（`cookies.get_all`/`stop` 签名以实测为准，见 `login_browser.py` 注释）；`verify_live(client)` 联网探针（打改写后订阅页判 200 vs 登录墙）。

## 三、机制二：Shibboleth / SAML（+ OpenAthens）— 待补 `run_saml_login`

### 3.1 协议 / 流程（SAML 2.0 Web Browser SSO，SP-initiated）
1. 用户访问 **SP（出版商）** 受保护资源 → 无 SP session。
2. SP 发起 **AuthnRequest**（302 重定向到 IdP 的 SSO endpoint）；若 IdP 未知，先经 **Discovery Service（SAMLDS）/ WAYF** 让用户选 IdP（回带 `entityID`）。
3. 用户在 **IdP** 登录（+2FA/CAPTCHA）。
4. IdP 生成 **SAML Response（断言）**，经浏览器 **HTTP POST** 到 SP 的 **ACS（Assertion Consumer Service，通常 `/Shibboleth.sso/SAML2/POST`）**。
5. SP 验证断言签名（IdP 证书）→ 建立 **SP 本地 session** → 302 回原资源。
- 手动入口：`https://<publisher>/Shibboleth.sso/Login?entityID=<IdP entityID>&target=<资源URL>`。

### 3.2 cookie / session 获取点
- **SP 域**：`_shibsession_<hash>` cookie（HttpOnly；SP session 主键）——**这是下载正文的关键 cookie**。
- **IdP 域**：IdP 自身 session cookie（用于 SSO 复用，跨 SP 免重登）。
- **OpenAthens**：机制类似（OpenAthens 是托管式 IdP/联邦），SP 域同样落 session cookie；本仓 `AuthProvider.OPENATHENS` 单列，与 Shibboleth 共用 `run_saml_login`。

### 3.3 与 route-B 的对接点
- **无 URL 改写**（不同于 EZproxy）：直接访问出版商原域，靠 `_shibsession_*` cookie 授权。故 `RouteBInjectionPlan.ezproxy_prefix=None`、`rewrite_target_host=出版商域`。
- **注入**：`_shibsession_*`（+ 必要的出版商 session cookie）注入 nodriver context，domain=出版商域。
- **detect**：登录链跨 `idp/sso/wayf/saml/shibboleth/openathens` 多个中转域，`url_looks_logged_in` 需在**回到出版商域**时才判成功（现有 `_SSO_DONE_HINTS` 已含这些关键词 ✓）。

### 3.4 凭据字段需求
| 字段 | 来源 | 说明 |
|---|---|---|
| `provider="shibboleth"`（或 `"openathens"`） | 用户配置 | `AuthProvider.SHIBBOLETH/OPENATHENS` |
| `login_url` | 用户填 / 推导 | 出版商 Shibboleth 入口，如 `https://www.sciencedirect.com/...`→跳 SSO，或 `.../Shibboleth.sso/Login?entityID=...` |
| IdP `entityID`（可选） | 用户填 | 跳过 WAYF 直选本校 IdP（如 `https://idp.uni.edu/idp/shibboleth`） |
| `institution_cookie` 或 CookieStore | 登录后自动 | `_shibsession_...`（`run_saml_login` 抓取） |
| `institution_domains` | 用户填 | 注入白名单（出版商域） |

### 3.5 待补（-153）
- **`run_saml_login(session, browser, login_url, entity_id=None)`**：结构 = `run_ezproxy_login` 的翻版（打开 `login_url` →（可选拼 `entityID`）→ 轮询 `url_looks_logged_in` → 抓 `_shibsession_*` → `save_cookies`）。
- **login_url 推导表**（per 出版商 Shibboleth 入口）：ScienceDirect / ACS / RSC / Springer / Wiley / IEEE 各家 Shibboleth 登录路径不同，建议 -153 建 `{prefix→shibboleth_login_url}` 小表（借 instsci 出版商工作流表）。

## 四、机制三：CARSI（中国教育网联邦认证）— 待补 `run_carsi_login`（复用 SAML）

### 4.1 协议 / 流程
- **本质**：**基于 Shibboleth 中间件 + SAML 2.0** 的中国教育科研网联邦认证（北大计算中心研发，CERNET 管理），**2019 年入 eduGAIN 全资格会员**。**协议与 §三 Shibboleth 完全同源**，差异只在 **IdP 发现**。
- **架构**：高校 **IdP**（对接校统一身份认证）+ 资源 **SP**（出版商）+ **CARSI DS/WAYF**（`ds.carsi.edu.cn` 选高校）。
- **流程**：用户访问出版商(SP) → 重定向 **CARSI DS（`ds.carsi.edu.cn`）** 选所属高校 → 跳转该校 **IdP（如 `idp.pku.edu.cn/idp/shibboleth`）** 登录 → SAML 断言回 SP 的 ACS → 建 SP session（`_shibsession_*`）→ 回资源。
- **SP 三种对接方式**（出版商侧，非本仓关心，仅背景）：① eduGAIN 直连；② 自建 Shibboleth SP（SAML）；③ CARSI SP+OAuth 网关（应用侧只需支持 OAuth 2.0，回调可能落 `connect/authorize`）。

### 4.2 cookie / session 获取点
- 与 Shibboleth 相同：**出版商 SP 域的 `_shibsession_*`**（+ 高校 IdP 域 session）。
- **WebVPN 变体**（`AuthProvider.WEBVPN`）：部分高校用 WebVPN 门户改写 URL（类 EZproxy 的主机名改写 + 校 CAS 登录），cookie 落 WebVPN 门户域——此时更接近 EZproxy 模型（URL 改写 + 门户 cookie）。

### 4.3 与 route-B 的对接点
- 与 Shibboleth 一致（无 URL 改写，靠 `_shibsession_*`）；**WebVPN 子情形**走 EZproxy 式改写（登记为 `provider=webvpn` 时启用 host 改写）。
- **detect 增强点**：登录链经 `ds.carsi.edu.cn`、`idp.<校>.edu.cn`、（OAuth 网关时）`connect/authorize`，需确保这些都算「未完成」，回到出版商域才判成功。

### 4.4 凭据字段需求
| 字段 | 来源 | 说明 |
|---|---|---|
| `provider="carsi"`（或 `"webvpn"`） | 用户配置 | `AuthProvider.CARSI/WEBVPN` |
| `login_url` | 用户填 / 推导 | 出版商 CARSI 入口，或直接 `https://ds.carsi.edu.cn/...`（带出版商 SP entityID + target） |
| 高校 IdP `entityID` | 用户填 | 如 `https://idp.pku.edu.cn/idp/shibboleth`（跳过 DS 选校，直连本校） |
| `institution_cookie` 或 CookieStore | 登录后自动 | `_shibsession_...`（`run_carsi_login` 抓取） |
| `institution_domains` | 用户填 | 注入白名单（出版商域）；WebVPN 时含门户域 |

### 4.5 待补（-153）
- **`run_carsi_login`**：可**直接复用 `run_saml_login`**，仅 `login_url` 默认走 CARSI DS + detect 加 `carsi`/`webvpn`（§六-2）。
- **高校选择**：DS 页选校是人工步骤（可见浏览器完成）；若用户填了 IdP `entityID` 可拼 URL 跳过 DS。

## 六、三机制共性适配层 + 两处可落地增强（-153）

### 6.1 统一登录内核（建议：把 `run_ezproxy_login` 泛化）
三个 `run_*_login` 仅 `login_url` 构造 + detect 不同，主体（打开→轮询→抓 cookie→save）一致。建议 -153 抽一个 `run_browser_login(session, browser, login_url, done_when=...)`，三机制传不同 `login_url`/`done_when`，避免复制粘贴（EZproxy 版已是模板）。

### 6.2 【本文新增发现】detect 启发式增强 `_SSO_DONE_HINTS`
现值：`("login","cas","sso","wayf","saml","idp","shibboleth","openathens")`。
- **缺口**：CARSI OAuth 网关回调（`connect/authorize`、`oauth`）、CARSI DS（`carsi`）、WebVPN 门户（`webvpn`）未覆盖 → 可能在中转页**误判已登录**。
- **建议补**：`+ ("oauth","authorize","carsi","webvpn","openid")`；并对 WebVPN 采「落在出版商域**且**不含门户域」的双条件（借 scansci「落出版商域且不在 login/wayf/saml/cas/idp」启发式）。
- 可离线加 selftest 断言（如 `not url_looks_logged_in("https://ds.carsi.edu.cn/...")`），无凭据即可验证。

### 6.3 verify_live 联网探针（G3，-153）
现 `verify_live(client=None)` 仅离线查 cookie 未过期；-153 补：对 `institution_domains` 首个订阅页发 HEAD/GET，命中登录墙（302→idp/paywall、401、登录页 HTML）判 False → 触发重登。

### 6.4 按出版商批级一次登录（借 scansci Phase2）
同一出版商多 DOI 复用一次 SP session（`_shibsession_*` 有效期内），减少重复 SSO；按 `institution_domains` 分组。
