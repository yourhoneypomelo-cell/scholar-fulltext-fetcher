# 聚合源"缺 DOI 归属校验"缺陷清单(openaire 已修,其余待办)

- 出品:148(openaire 假阳修复的附带盘点,按监管者-150 建议产出)
- 日期:2026-07-02
- 背景:openaire 假阳根因 = **搜索/递归抽取出的 URL 不校验"所属记录 == 查询 DOI"**。同类缺陷在其它聚合源也可能存在。本清单逐源核查"URL 归属是否有记录级保证",供总指挥决定 QC 复扫与修复排期。

## 结论一览

| 源 | 定位方式 | URL 归属保证 | 风险等级 | 建议 |
|---|---|---|---|---|
| openaire | `/search/publications?doi=` 相似检索 | **已修**(148 本轮):记录级 pid==DOI 才收;rels(引用/相关文献)整棵排除 | ~~高~~ → 已闭环 | 回归测例已入 selftest |
| base | `dcdoi:"{doi}"` 全文检索 | **无**:取 docs[:5] 的 dclink,不核 `dcdoi==查询DOI` | **高** | 与 openaire 同修:仅收 `_first(d.get("dcdoi"))` 归一后 == 查询 DOI 的 doc |
| core | `q=doi:"{doi}"` 检索 | **无**:取 results[] 的 downloadUrl,不核 `results[].doi` | **高** | CORE v3 结果有 `doi` 字段,应逐条核对后再收 |
| zenodo | `q=doi:"{doi}"` 检索 | **弱**:取 hits[:1] 全部 files,不核记录 doi/metadata.doi;且 Zenodo 常一 DOI 多附件 | 中高 | 核 `hits[].doi` 或 `metadata.doi`(注意 Zenodo 自身 DOI 与论文 DOI 是两个概念,需比对 `metadata.related_identifiers` 或直接 doi 字段) |
| europe_pmc | `DOI:"{doi}"` 检索(pageSize=1) | **弱**:取 result[0],不核 `r0.get("doi")==查询DOI`;标题模式下更松 | 中 | 加一行:`if r0.get("doi") and _norm(r0["doi"]) != _norm(paper.doi): return []`(标题输入路径保留模糊) |
| hal | `doiId_s:"{doi}"` 精确字段查询 | **中**:Solr 字段精确匹配,但未回读核验;docs[:1] | 低中 | 低成本加固:fl 加 doiId_s 并核对 |
| doaj | 路径式 `articles/doi:{doi}` 精确定位 | API 端点按 DOI 定位(非全文检索) | 低 | 可不动 |
| semantic_scholar | 路径式 `paper/DOI:{doi}` 精确定位 | 端点按 DOI 定位,openAccessPdf 属于该 paper 对象 | 低 | 可不动(S2 偶发 openAccessPdf 指向近似版本属数据质量问题,由下载后标题门兜底) |
| unpaywall / openalex / crossref | 路径式按 DOI 定位 | 记录级(单对象响应) | 低 | 可不动 |
| osf | `filter[doi]={doi}` 精确过滤 | JSON:API 服务端过滤 | 低 | 可不动 |
| biorxiv / pmc(idconv) | 路径式按 DOI | 记录级 | 低 | 可不动 |

## 高风险两例的实锤依据(代码位置)

1. `fulltext_fetcher/sources/green_oa.py` Base.find_candidates:`docs[:5]` 循环里只看 `dcoa`/`dclink`,**从不比对 `dcdoi`**。BASE 的 fcgi `query=dcdoi:"…"` 是检索而非精确定位,返回近似命中时与 openaire 假阳同构。
2. `fulltext_fetcher/sources/aggregators.py` Core.find_candidates:`q=doi:"{doi}"` 检索 limit=3,`results[]` 逐条收 downloadUrl,**从不比对 `results[].doi`**。CORE 检索对带标点 DOI 有分词近似命中风险。

## 与既有防线的关系

- 下载后内容 QC 门(config.content_qc,双门 union)只覆盖 **非 DOI-keyed 来源**(websearch/wayback/browser_search 及 landing 解析);上表聚合源都算 DOI-keyed、**被豁免**——所以源头必须自己把归属关校严,QC 门兜不住这类假阳(监管者-150 的两例实证即从 openaire 溜进 success)。
- 建议(采纳监管者-150):QC 复扫范围从"仅 websearch"扩到 **openaire/base/core/zenodo/europe_pmc 五源历史产物**,按本清单风险等级排序。

## 修复模式(可复用 148 本轮的做法)

1. 响应按**记录级**遍历,取每条记录自带的 DOI 字段归一后与查询 DOI 比对,不等则整条跳过;
2. 记录内"关系/引用"子树(如 openaire 的 rels)整棵排除;
3. 无法确证归属的 URL 不收(或降级低分 landing 留给下载后标题门);
4. 每源补"邻近记录误收"回归测例进模块 selftest。
