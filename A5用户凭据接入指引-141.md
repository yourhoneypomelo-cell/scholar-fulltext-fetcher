# A5 机构订阅 · 用户凭据接入指引（-141）

> **面向对象**：拥有**合法机构订阅**、对目标文献有访问权的使用者。
> **目标**：你**提供一次凭据**，即可让全文抓取一键走机构通道（HTTP 直链 + route-B 浏览器内认证）。
> **边界**：本文只讲「如何提供凭据、如何启用」，不含任何真实凭据。凭据严禁贴 chat / 日志 / git。
> **配套**：`.ftf_institutional.local.json.example`（模板）、`A5接入框架就绪-176.md`（框架 prep）、`机构订阅集成设计.md`、`路线A-机构订阅实测Runbook-凭据到手3步.md`。

---

## 0. TL;DR（三步启动）

1. **浏览器登录**你的机构入口（EZproxy / SSO），用 Cookie-Editor 导出登录后的 Cookie 串。
2. **写入凭据**：复制 `.ftf_institutional.local.json.example` → `.ftf_institutional.local.json`，填 `enabled / provider / ezproxy_prefix / institution_cookie / institution_domains`。
3. **一键运行**：

```powershell
# 核心 CLI：显式启用机构 + route-B
python -m fulltext_fetcher -f still_missing.txt --institutional --route-b cf-only --email you@uni.edu

# 或一键路径：读到 enabled 的本地 JSON / FTF_INSTITUTIONAL=1 会“自动”进机构模式，无需 flag
python run_all.py -f still_missing.txt --route-b cf-only
```

离线验证（**不需凭据**）：`python -m fulltext_fetcher.institutional.selftest_a5_framework` → 打印 `A5_FRAMEWORK_OK`。

## 1. 第一步：选择你的机构接入方式

| 机制 | 你需要准备 | 现在能用吗 |
|---|---|---|
| **EZproxy（前缀式）** | 登录后 Cookie 串 + 前缀 `https://login.ezproxy.你的学校.edu/login?url=` | ✅ 立即可用 |
| **EZproxy（主机名改写式）** | 登录后 Cookie 串 + 代理裸域名 `ezproxy.你的学校.edu` | ✅ 立即可用 |
| **Shibboleth / SSO** | 浏览器完成 SSO 后，导出目标出版商域的 Cookie 串 | ✅ 手动导 Cookie 可用；自动登录待 -153 |
| **CARSI** | 经 CARSI 登录出版商后导 Cookie | ✅ 手动导 Cookie 可用；自动登录待 -153 |
| **WebVPN / OpenAthens** | 同上 | ✅ 手动导 Cookie 可用；自动登录待 -153 |

> **关键结论**：无论哪种机制，只要你能在浏览器里登录到「能看全文」的状态并导出 Cookie，**现在就能用**。差别仅在于「自动可见浏览器 SSO 登录」尚在 -153 实现，不影响手动 Cookie 通路。

### 如何导出 Cookie（所有机制通用）

1. 浏览器登录到能正常看全文的状态（地址栏已**脱离** `login/cas/sso/idp/wayf` 等中转页）。
2. 装 **Cookie-Editor** 扩展 → 打开目标出版商页面 → **Export → Header String**（形如 `k1=v1; k2=v2`）。
3. 该串即 `institution_cookie`。**只放本地文件或环境变量，绝不贴聊天 / 日志。**

## 2. 第二步：把凭据交给工具（两条通路，二选一）

### 通路 A · 本地 JSON 文件（推荐：批量、持久）

```powershell
Copy-Item .ftf_institutional.local.json.example .ftf_institutional.local.json
```

字段说明：

| 字段 | 含义 | 示例 |
|---|---|---|
| `enabled` | 机构总开关，**必须 `true` 才生效** | `true` |
| `provider` | 机制类型 | `ezproxy` / `shibboleth` / `carsi` / `webvpn` / `openathens` / `manual` |
| `ezproxy_prefix` | EZproxy 前缀或代理裸域（非 EZproxy 留 `null`） | `"https://login.ezproxy.YOUR-UNI.edu/login?url="` |
| `institution_cookie` | 登录后 Cookie 串 | `"ezproxy=xxx; sess=yyy"` |
| `institution_domains` | 只对这些出版商域启用（空=不主动改写任何域） | `["sciencedirect.com","pubs.acs.org"]` |
| `cookie_store_path` | 可选，Cookie 仓路径 | `".ftf_cookies.local.json"` |

> 该文件已加入 `.gitignore`（本波补：`.ftf_*.local.json`），不会被 `git add`；模板 `.example` 不受影响、可入库。仍请你 `git status` 自查。
> 想放到别处：设 `FTF_INSTITUTION_CONFIG_PATH` 指向你的 JSON 绝对路径。

### 通路 B · 环境变量（临时 / CI / 不落文件）

**优先用框架层 `FTF_*`**（同时被核心 CLI 与 `run_all.py` 一键路径识别）：

```powershell
$env:FTF_INSTITUTIONAL        = "1"
$env:FTF_INSTITUTION_PROVIDER = "ezproxy"
$env:FTF_EZPROXY_PREFIX       = "https://login.ezproxy.YOUR-UNI.edu/login?url="
$env:FTF_INSTITUTION_COOKIE   = "ezproxy=...; sess=..."     # 从浏览器导出，勿贴 chat
$env:FTF_INSTITUTION_DOMAINS  = "sciencedirect.com,pubs.acs.org,pubs.rsc.org"
```

> 也存在旧的「CLI 直传默认值」变量 `EZPROXY_PREFIX` / `INSTITUTION_COOKIE`（仅作命令行参数缺省），但 `FTF_*` 覆盖更全（含总开关 / provider / domains / 本地 JSON 合并），**建议统一用 `FTF_*`**。

**凭据优先级**：命令行参数 `>` `FTF_*` 环境变量 / 本地 JSON。即你在命令行显式给了 `--ezproxy-prefix`，不会被文件里的值覆盖（`bootstrap` 只补 CLI 未填的字段）。

## 3. 第三步：一键启用（提供凭据后即可跑）

### 核心 CLI

```powershell
python -m fulltext_fetcher -f still_missing.txt --institutional --route-b cf-only --email you@uni.edu
```

- `--institutional`：启用机构直链源 `publisher_direct`（HTTP 层：EZproxy 改写 + Cookie 注入）。若已在 JSON/env 里 `enabled`，运行时也会自动置为机构模式。
- `--route-b cf-only`：对 RSC / ACS / Wiley / ScienceDirect 等 **JA3 绑定型强 CF 站**，在浏览器内（nodriver 真 Chrome）**注入机构 Cookie 后页内抓字节**（过 CF 的同时带机构身份）。

命令行也可直接传（不落文件、但会进 shell 历史，慎用）：

```powershell
python -m fulltext_fetcher -f still_missing.txt --institutional `
  --ezproxy-prefix "https://login.ezproxy.YOUR-UNI.edu/login?url=" `
  --institution-cookie "ezproxy=...; sess=..." `
  --institution-domain sciencedirect.com,pubs.acs.org `
  --route-b cf-only --email you@uni.edu
```

### run_all 一键路径

```powershell
python run_all.py -f still_missing.txt --route-b cf-only
```

> `run_all.py` 启动即 `bootstrap_institutional_config`：**读到 `enabled` 的本地 JSON 或 `FTF_INSTITUTIONAL=1` 就自动进机构模式**，无需额外 flag —— 这正是「一提供凭据即可启动」。

### 两条通道各治什么

| 通道 | 触发 | 适用站点 |
|---|---|---|
| HTTP 直链（`publisher_direct`） | `--institutional` | 弱防护 / EZproxy 前缀可达的订阅站 |
| route-B 浏览器内注入 | `--route-b cf-only` + 机构凭据 | 强 CF / JA3 绑定站（RSC/ACS/Wiley/SD） |

两者可**同时开**。route-B 注入 hook（`inject_institutional_session` + `rewrite_url_for_injection_plan`）已在 `render_fetch.py` 接线，等你的凭据即可联调。

## 4. 安全纪律（务必遵守）

- ❌ 严禁把 Cookie / 前缀**硬编码**进任何 `.py`。
- ❌ 严禁把凭据贴进 chat、`run.log`、`metadata.jsonl` 或任何产物。
- ❌ 严禁 `git add` 本地凭据文件（已 gitignore，仍需 `git status` 自查）。
- ✅ 框架已对凭据自动脱敏：`InstitutionalCredentials.__repr__` 只显示 `abcd…yz` 形式，日志里看不到明文。
- ✅ 仅对**你有合法访问权**的内容使用；无订阅时直链会 401/403 并被 `%PDF` 魔数校验过滤（不会污染产物）。

## 5. 验证与排障

### 离线自检（不需凭据，任何时候可跑）

```powershell
python -m fulltext_fetcher.institutional.selftest_a5_framework   # → A5_FRAMEWORK_OK
python run_all_selftests.py                                       # 全量回归，a5_framework 应为 PASS
```

### 常见问题

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 大面积 401/403 | Cookie 过期 / 未覆盖该域 | 重新登录导 Cookie；确认 `institution_domains` 含该站 |
| 强 CF 站仍 miss | 没开 route-B | 加 `--route-b cf-only`（需 nodriver + 有头显示） |
| 机构模式没生效 | `enabled` 非 true / 变量没设 | 查 JSON `enabled`、`FTF_INSTITUTIONAL=1`；运行日志会打印 `机构凭据自 FTF 加载(source=..., provider=...)` |
| route-B 静默 no-op | 缺 nodriver / 无显示环境 | 装 nodriver，或用有头 / xvfb 虚拟显示 |
