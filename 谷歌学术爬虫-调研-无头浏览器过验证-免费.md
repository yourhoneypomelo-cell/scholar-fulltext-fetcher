## 〇、一页速览（TL;DR）

- **反爬的胜负手在 L3"怎么驱动浏览器"，不在 L1"浏览器声称什么"。** 2026 benchmark（7 工具 / 31 CF 目标 / 651 判定）里 **`nodriver`（直连 CDP、无 Playwright shim）唯一做到 31 目标零封锁**；`curl_cffi`（21 行 TLS 包装）竟与整只带 49 处 C++ 补丁的 Chromium fork 打平（26 OK）。
- **免费栈最佳两件套（与本仓一致）**：`curl_cffi`（L1，TLS/JA3/JA4 指纹，静态抓）＋ `nodriver`（L3，真 Chrome 兜底）。锁死 Playwright 栈时用 `patchright`（`channel=chrome`）。
- **免费打码只有 reCAPTCHA v2「音频挑战 + 本地 Whisper」这一条真正可用**（多个 MIT OSS，标准挑战号称近 100%）；**hCaptcha / reCAPTCHA v3 无成熟免费方案**（超出本文范围，且属高风险）。
- **无代理是本课题的最强约束**：不能轮换 IP ⇒ 一旦触发"Your computer or network may be sending automated queries"就**没有退路**。故第一目标是 **"不触发"** 而非 "过验证"。好消息：**家用/本地 IP 的信誉远高于数据中心 IP**，本地 IP 是相对优势——**关键是别烧掉这唯一的干净 IP**。
- **落地纪律**：真 Chrome（nodriver / patchright `channel=chrome`）＋ headed/Xvfb ＋ 串行 ＋ 45–90s 抖动限速 ＋ 命中风控即冷却；**优先走合规正门**（开放 API / SerpApi），自建直抓仅兜底；免费打码仅作"极低频最后手段"。这与本仓 `ScholarConfig` 默认值（`concurrency=1`、`page_interval 45–90`、`captcha_enabled=False`）逐条吻合。

### 1.2 逐库要点（免费视角）

- **nodriver — 首选**。undetected-chromedriver 的**官方继任者**（同作者），纯异步、直连 Chrome DevTools Protocol，`navigator.webdriver`、`cdc_` 变量、`--enable-automation` 全无，"启动方式就像人双击图标"。**代价**：① **AGPL-3.0** 有网络传染性（对外提供服务须开源修改，内部/个人研究无碍）；② 让你"不被识别"但**不自动解** Turnstile/reCAPTCHA token；③ 官方也承认**代理鉴权（user:pass）支持尚不成熟**、**headless 仍更易被检测**（无代理场景本就不轮换、正好用 headed）。
- **patchright — Playwright 生态的最佳退路**。在协议握手层打掉 Playwright 的 CDP 泄漏，`channel=chrome` 驱动系统真 Chrome（真 TLS + 版本戳）。若代码库已锁死 Playwright（如本仓 `PatchrightEngine`），它是零改动升级；且 **Apache-2.0** 商用友好。
- **undetected-chromedriver / rebrowser-playwright — 不用**。前者作者已把精力转向 nodriver、对现代防护渐失效；后者仅 50 星、2024-09 后基本停更，benchmark 垫底——"只打补丁、不换真 Chrome"不足以过关。
- **botright — 谨慎借鉴**。它是**唯一自带"免费 AI 打码"**的框架（reCognizer/hcaptcha_challenger 做 CV/CLIP 识图），但**维护放缓**：依赖 `hcaptcha_challenger`、`onnxruntime` 版本冲突，需旧 Python + 手动改，多个验证类型标注"当前不可用"。其"本地 ML 免费解图形验证"的思路值得参考，但**不宜作生产主力**。
- **通用铁律（所有工具）**：**headless 是共同的检测风险，headed 或 Xvfb 虚拟屏显著提升过关率**；且**没有银弹**——IP 信誉与行为分析独立于浏览器指纹运作（直接引出 §4 的无代理约束）。

## 二、Cloudflare / 反爬免费方案对比（含适用场景）

> 说明：Google Scholar 本身不走 Cloudflare，但**下载 PDF 时的出版商/镜像站**常有 CF；本节方案对"过 CF 型落地页"通用。

| 方案 | 仓库/形态 | 许可证 | 处理层 | 能力与局限 | 适用场景（2026） |
| --- | --- | --- | --- | --- | --- |
| **curl_cffi** | lexiforest/curl_cffi | **MIT** | L1 TLS/HTTP | 复刻 Chrome 的 JA3/JA4 + HTTP2 帧序（BoringSSL=Chrome 的 TLS 库）；`impersonate="chrome"` 自动跟最新；极轻快。**无 JS 引擎 → 过不了 Turnstile/JS 挑战** | **首选**：仅 TLS 指纹拦截的静态页（Scholar 结果页大多可静态解析） |
| **FlareSolverr** | FlareSolverr/FlareSolverr | MIT | L3 代理服务 | headless Chrome + UC 藏在 `/v1` API 后，Docker 易部署；**对现代 CF（Turnstile / Managed Challenge）失效**（无自有 solver）；会 hang/freeze；**不支持 per-request 鉴权代理** | 仅"轻 JS 挑战 / 低量 / 老站" |
| **cloudscraper** | VeNoMouS/cloudscraper | MIT | L1+轻 JS | 处理**旧式** CF IUAM JS 挑战；对现代 Turnstile 基本失效 | 渐少的老式 CF 站，聊胜于无 |
| （2026 后继）**Byparr** | FlareSolverr 的 drop-in 替代 | 开源 | L3 代理服务 | 一行替换 FlareSolverr，**能更好处理 Turnstile** | 需要 FlareSolverr 式服务但要过 Turnstile 时 |
| （2026 后继）**Scrapling** | D4Vinci/Scrapling | 开源 | 一体化框架 | `Fetcher`（静态）/ `StealthyFetcher(solve_cloudflare=True)` / `DynamicFetcher`（真 DOM）分层，v0.4.8（2026-05） | 想要"一个库按目标难度自动分层"时最干净 |
| （备选）tls-client | bogdanfinn/tls-client | BSD | L1 TLS | Go 核心 + Python 绑定，可**自定义** JA3/HTTP2 profile | 需要 curl_cffi 预置之外的指纹时 |

**场景决策树（免费）**：
1. 页面纯静态 / 仅 TLS 指纹拦 → **curl_cffi**（`impersonate="chrome"`）。够用就不上浏览器。
2. 触发 JS 挑战（Turnstile）/ 行为评分 → 上**真 Chrome 浏览器**（nodriver / patchright `channel=chrome`，headed），或 Byparr。
3. 数据中心 IP 被判低信誉 → 常规解法是**住宅代理**——**但本课题无代理**，见 §4（改为"低频养住本地 IP"）。

## 三、reCAPTCHA / hCaptcha 免费应对（OSS，不含付费打码）

### 3.1 reCAPTCHA v2：音频挑战 + 本地 Whisper —— 唯一真正可用的免费路

**原理**：reCAPTCHA v2 的无障碍"音频挑战"播放一段**清晰英语语音**（非扭曲噪声），Whisper 可近乎完美转写。流程：点 "I'm not a robot" → 切音频 → 拦截/下载音频 → `faster-whisper` 本地转写 → 填答提交。

| OSS 项目 | 许可证 | 栈 | 说明 |
| --- | --- | --- | --- |
| ibedevesh/capsolver | MIT | Python + Whisper | `RecaptchaV2Solver(model_size="base")`，号称近 100% |
| saifyxpro/recaptcha-v2-audio-solver | MIT | Selenium + UC + faster-whisper | 处理"多次正确"与重试循环 |
| yfe404/recaptcha-audio-solver | （POC） | Playwright + Whisper | 概念验证，tiny/base 即可 |
| k19-sudo/recaptcha-v2-resolver-free | 免费/本地 | TS + Playwright + `@xenova/transformers` | 纯本地、内存内处理、零数据外泄 |

**成功率与硬约束（务必知悉）**：
- ✅ 标准挑战号称"近 100% / 首次即中"（Whisper `small`/`base` 足够，无需 `large`）。
- ⛔ **仅 reCAPTCHA v2（checkbox）**；**不支持 v3（score-based，无挑战可解）**，**不支持 hCaptcha**。
- ⚠️ **重复使用 → 触发限流 / "automated queries" 封禁**；对"疑似 bot"站点**音频选项可能被直接禁用**。
- ⚠️ 官方级经验一致强调：**需住宅代理 + IP 轮换 + 拟人指纹/行为**才能持续——**这正是本课题最缺的**（见 §4，故只能极低频用）。
- 资源：Whisper `small` 约 2GB RAM，需系统装 `ffmpeg`。

### 3.2 行为模拟（免费、辅助）

随机延迟 + 拟人鼠标轨迹 + 真实 UA/指纹可**提升过关率与降低触发**，但**不改变 IP 信誉**——只是"锦上添花"，不是"雪中送炭"。

### 3.3 hCaptcha / reCAPTCHA v3 —— 免费基本无解

- **hCaptcha**：无广泛成功的简单 OSS 绕过；远比 reCaptcha 抗打。`botright` 依赖的 `hcaptcha_challenger`（CV/CLIP 识图）可试，但**维护差、依赖冲突**，不可靠；可靠自动化通常需**付费 AI**（超出本文"免费"范围）。
- **reCAPTCHA v3**：纯分数、无挑战可"解"，只能靠**账号/IP 信誉 + 拟人行为**攒分——本质是"别被扣分"，不是"过关"。

### 3.4 风险总评（无代理单 IP 场景）

免费打码在"**无代理、单一本地 IP**"下**风险最高**：一旦频繁触发，极易被永久标记"automated queries"而**烧掉你唯一的干净 IP**。因此定位为**"偶发人工兜底级"的最后手段**，默认关闭（与本仓 `captcha_enabled=False` 一致）。

## 四、本地 IP、无代理约束下的最实用组合与落地建议 ★

### 4.1 无代理约束的两条核心洞察

1. **不能轮换 IP = 没有退路**：任何触发风控/验证的操作都在消耗**唯一**的 IP 信誉，一旦被拉黑无法逃逸。⇒ **首要目标是"不触发"，其次才是"过验证"。**
2. **本地/家用 IP 是相对优势**：CF/Google 对**住宅 IP 的信誉评分远高于数据中心 IP**（后者无论指纹多完美都被判低信任）。⇒ 本地直连**天然比数据中心代理更容易被放行**——**代价是不可再生，必须"养"住**。

### 4.2 推荐组合（免费 / 无代理，且与本仓 `fetcher.py` 分层逐层对齐）

| 层 | 工具（免费） | 何时用 | 本仓现状 |
| --- | --- | --- | --- |
| **L1 静态** | `curl_cffi`（`impersonate="chrome"`） | 默认首发；Scholar 结果页大多可静态解析 | 已实现 `CurlCffiEngine` |
| **L3 浏览器** | `nodriver`（headed/Xvfb、真 Chrome） | L1 被拦（需 JS / 行为评分）时兜底 | 已实现 `NodriverEngine`；备选 `PatchrightEngine`（`channel=chrome`） |
| **打码（最后手段）** | reCAPTCHA v2 音频 + 本地 Whisper | 仅偶发弹 v2 时、**极低频**；否则宁可放弃该条 | `captcha_enabled` 默认 **False**，与"默认不硬刚"一致 |
| **不做** | hCaptcha / reCAPTCHA v3 硬解 | 免费无解 → 命中即冷却/换策略/走正门 | — |

### 4.3 落地纪律（强合规，逐条对齐本仓 `ScholarConfig` 默认）

- **串行**：`concurrency=1`（不并发抓 Scholar）。
- **强限速抖动**：两次页请求间 `page_interval` **45–90s** 随机抖动；**指数退避**；命中风控**冷却 15min**。
- **真浏览器 + headed**：nodriver/patchright `channel=chrome`；无显示环境用 **Xvfb**（headless 更易被检测）。
- **优先合规正门**：开放 API（角度2）/ SerpApi（免费额度或低价）——**能不自建直抓就不抓**；`mode="auto"` 有 key 即走 SerpApi。
- **PDF 下载**：多为出版商 **OA 落地页，通常无需反爬**，直接复用本仓 `fulltext_fetcher`（`download.py` + `landing.py` + 可选 `render_fetch`）。
- **打码若启用**：Whisper `small` + 本地 + `ffmpeg`；**单次/低频、失败即停**，绝不重试轰炸（否则烧 IP）。

### 4.4 一句话结论

> **在"免费 + 本地 IP + 无代理"下，把唯一的干净 IP 当稀缺资源：用最真实的浏览器（nodriver/真 Chrome）＋ 最慢的节奏（串行 + 45–90s 抖动）＋ 最少的验证触发，优先走合规正门（开放 API / SerpApi），自建直抓与免费音频打码只作兜底。** 反爬没有银弹，"不触发"永远比"过验证"更划算。
