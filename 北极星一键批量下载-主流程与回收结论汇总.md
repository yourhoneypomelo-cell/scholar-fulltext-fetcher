## 一、北极星主流程 `run_all.py`（一键编排器，新文件）

**定位**：直击北极星「输入标题/DOI → 全程程序化 → 标准化文件名 → 一键批量出全文 + 覆盖报告」。**只 import 现有 `fulltext_fetcher.Pipeline` / `cli._read_input_file` 与 `tools/build_coverage.py`，不改它们任何逻辑**。

**流程（6 步）**：
1. 读输入清单（`.txt/.csv/.xlsx`，复用 `cli._read_input_file`；标题与 DOI 可混排）。
2. 输入内去重（规范化 DOI 优先，非 DOI 标题按小写去重，保序）。
3. 跨批续跑（`--resume` 默认开）：扫既有 `out/`（`--coverage-root`）的 coverage，把**已真实成功**（有 PDF 落盘）的 DOI 从本次输入剔除，避免重复下载。
4. 调 `Pipeline` 下载剩余输入到 `RUNROOT/fetch/`（独立子目录，自带 `out_dir` 写锁保护，可与 CF 回收并发）。
5. 末尾调 `build_coverage` 对 `RUNROOT` 生成 `coverage.json` + `still_missing.txt`。
6. 打印一页式总结（输入/去重/跨批跳过/本次成功·miss/覆盖/still_missing/按源）+ 落 `run_all_summary.json`（机器可读）。

**产物布局（`RUNROOT` 由 `-o` 指定，独立目录）**：

| 产物 | 内容 |
|---|---|
| `RUNROOT/fetch/` | `pdfs/` + `metadata.jsonl` + `summary.json` + `results.csv` + `report.html` + `run.log` |
| `RUNROOT/coverage.json` | 去重 coverage 主库（success = metadata.success 且 PDF 落盘） |
| `RUNROOT/still_missing.txt` | 仍缺 DOI 全集（可直接作下一轮 `-f` 输入） |
| `RUNROOT/run_all_summary.json` | 本次编排的机器可读总结 |

**用法**：

```bash
python run_all.py -f inputs.txt --email you@uni.edu -o out/run_all
python run_all.py "10.1371/journal.pone.0000217" "1706.03762" "Attention is all you need" -o out/run_demo
python run_all.py -f inputs.txt --no-resume     # 不做跨批已covered剔除(强制全跑)
python run_all.py --selftest                     # 离线自检(不联网) → RUN_ALL_OK
```

**护栏**：新文件、不改核心码；每次用独立 `-o`（Pipeline 的 `out_dir` 写锁会阻止并发写同一目录）。

## 二、coverage / still_missing 口径 `tools/build_coverage.py`（跨批主库，新文件）

**为什么需要它（而非直接看各 `summary.json`）**：`summary.json` 只统计「本次运行 processed 的条数」，断点续跑 skipped 的历史成功**不计入**，会**低估**真实成功；且回收/probe 分散在多个 `out/` 目录，单目录口径看不到全局。

**权威口径（务必对齐审计）**：
- **dedup key**：规范化 DOI（优先 `doi`，回退 `raw_input`；小写、去 `doi.org/`、`doi:` 前缀）。
- **真实成功（success）**：某 DOI 至少一条 `success==true` **且**其 `pdf_path` 文件名**确实存在**于该批 `pdfs/`（落盘实证）。仅 metadata 声称成功但盘上无文件 → **不计成功**，标 `claimed_success_but_no_pdf`（可复下的软缺口）。
- **跨批合并**：success 取**并集**（任一批真实成功即成功，多处成功取 `pdf_bytes` 最大）；miss 的 `error` **取末次**（按各批 `metadata.jsonl` mtime 升序、行内顺序）。
- **交叉核对**：逐批（非去重）求和 → core 三主语料批复现审计 **866/1213 ≈ 71.4%**；去重主库才是真实「唯一论文」覆盖（差额=跨批重复，如 batch7 多为 batch6 已成功项的重跑，净增仅个位数）。

**产物**：`<out_root>/coverage.json`（每 DOI 规范状态 + 全局汇总）、`<out_root>/still_missing.txt`（仍缺 DOI 全集，每行一条、`#` 头注释会被 `fetcher -f` 跳过，可直接续跑）。

**用法**：

```bash
python tools/build_coverage.py                         # 扫 out/ → out/coverage.json + out/still_missing.txt
python tools/build_coverage.py --out-root out --no-write   # 只打印全局汇总
python tools/build_coverage.py --selftest              # 离线自检 → COVERAGE_OK
```

## 三、回收结论沉淀（权威去重口径）

| 语料批 | 输入 | 真实成功 | 成功率 | 主力成功源 |
|---|---:|---:|---:|---|
| **batch6** | 500 | **410** | **82.0%** | websearch 260(63%) / publisher_oa:acs-authorchoice 52 / openalex 30 / s2 21 / unpaywall 16 |
| **batch4（语料B）** | 500 | **348** | **69.6%** | websearch 263(75.6%) |
| **batch7** | 213 | **108** | 50.7% | （多为 batch6 已成功项的重跑，去重后净增个位数） |
| **core 逐批求和（审计基线）** | 1213 | **866** | **≈71.4%** | — |

**回收关键发现**：
- **batch6 跃升主因**：新管线启用 `websearch`（深度搜索引擎找自存稿 PDF，单独贡献 63% 成功）+ `publisher_oa`（ACS AuthorChoice/GoldOA 直取，几乎清空旧 B 类「仅出版商链接」149→1）；旧 C 类「全网未定位」196→16。
- **batch4 比 batch6 低 12.4pt**：两批输入 DOI 交集为 0（不同论文集），按同规模真实成功率可比；batch4 的 miss 更集中在 Elsevier/Cloudflare。
- **CF 回收（进行中，-145）**：仓内 `tools/flaresolverr_nodriver.py`（免 Docker nodriver shim）已把 **ACS CF403 解成 4.4MB PDF**（`acscatal` recovered）；但 **RSC/ScienceDirect 类的 `cf_clearance` 绑定 JA3 指纹**，交 `curl_cffi` 回放仍 403——**换更强求解器（byparr）无效，真解法是「浏览器内直接下 PDF」**（详见 `选型2026-采纳与淘汰总表` §四 + 新增 P2 项）。
- **still_missing 构成**：主体是**付费墙内纯订阅全文**（物理边界，唯一合法途径=机构订阅 A5）+ **JA3 绑定型强 CF 站**（需浏览器内直下）。`still_missing.txt` 可直接作下一轮 `-f` 输入闭环续跑。
