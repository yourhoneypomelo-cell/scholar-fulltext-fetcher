## 〇、TL;DR

- **A5 是把免费天花板（~38.5–40%）推到 70%+ 的唯一杠杆**：still_missing 中 A5 可救前缀 Top10 ≈ **593/628（94%）**，Elsevier(10.1016)+ACS(10.1021)+RSC(10.1039) 三社 = **536（85%）**（数据源 `A5机构订阅现状与still_missing可救前缀梳理-150.md`）。**当前唯一卡点 = 用户凭据**，不是工程。
- **2026-07 三大可借鉴开源实现（均活跃维护、密码不经手、可见浏览器人工 SSO）**：
  1. **SciTeX-Scholar v1.4.1**（2026-05-27，Playwright，西方 SSO：OpenAthens/EZProxy/Shibboleth/Lean Library）——`-150` 已实读 v1.4.x，本文补 v1.4.1 auth 类结构与 debug 工装。
  2. **scansci-pdf v1.6.1**（2026-06-20）+ **instsci**——CARSI/WebVPN(100+ 高校 CAS) 强项；**已从 CloakBrowser 演进到 CamoFox(Firefox 系) 持久化**（相对仓内 `对141建议` 文档当时的 CloakBrowser 基准的**显著时效变化**）。
  3. **camofox-browser**（jo-inc v1.11.2 / redf0x1 v2.4.6，MIT，Firefox/camoufox）——scansci 现用的持久化底座，`launchPersistentContext` + `~/.camofox/profiles/`（cookies+localStorage+IndexedDB 全持久化）。
- **给本仓的最优路径（本文核心决策）**：**不引入 Playwright / CloakBrowser / CamoFox 任何新浏览器依赖**，而是——
  - **授权会话内下载**：沿用 `-150` delta-2，把持久化 cookie **注入同一个 nodriver 真 Chrome context** → 走**已跑通的 route-B B1 页内 `fetch()` 直下**（A5 与 route-B 焊成同一条链，避免 curl_cffi 裸回放在 JA3 站必 403）。
  - **会话取得/持久化**：**借 scansci 的设计、不借其依赖**——移植 `CookieStore`(零依赖 JSON+过期) + `open_login_browser`(可见浏览器人工登录、URL 脱离 login/cas/sso/wayf/saml/idp 即判成功) 的**思路**，用 nodriver 实现；叠加 SciTeX 的 **`verify_live` 用前活校验**。
  - 理由：route-B 已证明 **nodriver 有头 Chrome 在本机(Windows)能过 CF 并抓字节**（见 `路线B-真机冒烟结果-152.md` / 本会话 ACS b1 复验 13.7MB 通过），再加一层「注入机构 cookie」即得 A5，无需第二套浏览器栈。

## 三、给本仓的最优路径决策（取长补短·借设计不借依赖）

**核心机制次序**（概念骨架，实现归 -153/-141；此处只定机制，不改任何 `.py`）：

```python
async def a5_authorized_download(article_url, doi, cookie_store, provider, cfg):
    host = _host(article_url)
    # ① 用前活校验(SciTeX verify_live):失效即触发人工重登
    cookies = cookie_store.get(host)                       # CookieStore: JSON+过期过滤(移植 scansci)
    if not cookies or (cfg.institutional_verify_live and not await _verify_live(host, cookies)):
        cookies = await open_login_browser(login_url_for(provider, host), cookie_store, host)
        # open_login_browser: nodriver 有头,人工完成 SSO/2FA/CAPTCHA;
        # URL 脱离 {login,cas,sso,wayf,saml,idp} 即判成功(借 scansci detect 启发式),抓 cookie 落 CookieStore
    # ② 注入同一 nodriver context 再导航(-150 delta-2;JA3 站唯一正解)
    browser = await nd.start(headless=False)               # 复用 render_fetch 的 CDP 兼容 patch
    await _inject_cookies(browser, cookies)                # 导航前 Network.setCookies
    tab = await browser.get(article_url)
    await _wait_pass_cf(tab)                               # 过盾期不开 CDP 拦截(route-B N.8 铁律)
    # ③ 同会话内直下:B1 页内 fetch 优先,CSP/跨源挡→B2 过盾后 Fetch.enable(复用 route-B 现成实现)
    pdf = await _b1_inpage_fetch(tab, pdf_url_for(article_url)) or await _b2_nav_capture(tab, ...)
    # ④ 落盘前必过内容 QC 门③:page-1/正文 DOI ≠ 期望 → 拒(复用 tools/qc_content_match)
    return pdf if content_qc_ok(pdf, doi) else None
```

**要点**：
- **A5 = route-B + 注入机构 cookie**，多出的只有「① 会话取得/活校验」「② 注入 cookie」两步；②③④ 全复用 route-B 现成代码与铁律（过盾期不开拦截、B1 优先 B2 兜底、不用 `Page.printToPDF`、CDP 双键 monkeypatch）。
- **绝不 curl_cffi 裸回放**（-177 骨架旧设计那步）——对 JA3 绑定订阅站(RSC/ScienceDirect)必 403。
- **RSC 特例**：本会话 route-B 复验发现 RSC `articlepdf` 的 b2-fetch 过盾后 `get_response_body` 读不到 body（0 字节，已交 -141 修「下载式 body 抓取」）。**A5 落地前该缺口需先修好**，否则 RSC(67 条) 即便有机构会话也抓不到字节——**这是 A5×route-B 合流链上的真实前置依赖，已在协作中**。

## 五、最小落地骨架（对齐 nodriver 栈，实现归 -153）

  __init__.py
  cookie_store.py      # 移植 scansci CookieStore：JSON 持久化+过期+per-社;零依赖;离线 selftest COOKIE_STORE_OK

## 六、合规 / 护栏 / gate（不变）

1. **仅合法权限**：只对用户**确有机构订阅**的刊；A5=以合法身份走正门，**不破付费墙本身**、不引 Sci-Hub/LibGen（scansci 的灰色源本仓一律不采）。
2. **绝不存密码**：SSO/2FA/CAPTCHA 全在可见浏览器由用户完成；cookie 只落本地、限权、可过期 + `verify_live`；进 `.gitignore`，绝不入 git/日志/产物。
3. **OA 豁免**：开放 API/OA 域名永不注入机构 cookie（`_OPEN_ACCESS_HOSTS` 防线保留）。
4. **落盘必过内容 QC 门③**：机构正门下的 PDF 同样核 page-1/正文 DOI＝期望 DOI，防抓错刊/SI/顶包。
5. **礼貌/低并发**：route-B 全局 concurrency=1、按 origin 缓存会话；放量 `-c 2 --per-host-interval 1.0` 防 bulk 风控/AUP 红线。
6. **凭据 gate**：无凭据 → **ROI 严格 = 0、永不联网实测**；代码就绪，凭据到手照 §4.3 + Runbook。
