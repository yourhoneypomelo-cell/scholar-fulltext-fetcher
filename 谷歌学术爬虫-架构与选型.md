## 〇、用户五原则 → 落地映射（本设计据此收敛）

| # | 用户原则 | 在本架构中的落地点 |
|---|---|---|
| ① | **全程程序化、一键批量**：输入 title/DOI，输出文件名标准化的系列原文 | `cli`/`__main__` 批量入口 + `pipeline` 编排 + `naming` 文件名标准化；输入沿用父包 `-f .txt/.csv/.xlsx` |
| ② | **运行 = 跑程序 + 读日志闭环**，结构化日志 | 复用父包 `logsetup`（`run.log` + `EventLog` JSONL）；产物 `attempts.jsonl / serp.jsonl / metadata.jsonl / summary.json / results.csv / report.html` |
| ③ | **反爬以「避免触发」为主**，「自动求解验证码」为**可选、默认关**的最后手段 + 合规限速 | `fetcher` 分层降级（L1 静态→L3 浏览器→翼B 打码）；`captcha` 默认关；`proxy` 默认关；强限速/退避/冷却默认值见 §4 |
| ④ | **PDF 覆盖不求全**，聚焦 Scholar 可及链接；付费墙/闭源由**其它渠道兜底** | `download` 先用结果自带 PDF 直链；未果**复用父包 `fulltext_fetcher` 的 OA 源**（Unpaywall/OpenAlex/…）兜底；Sci‑Hub 默认关 |
| ⑤ | **经验全记录**（踩坑/成功/发现） | 结构化事件 + 运行末尾把 block/captcha/proxy 事件汇总进 `summary.json`；重要发现回写 `经验记录-踩坑与发现.md`（构建阶段） |

**一句话结论（承 00b/R1/R2）**：能走正门（角度2 开放 API）就别翻墙；真要抓 Scholar，**先定位被拦层再加最便宜的对应工具**（curl_cffi → nodriver → 打码），全程住宅代理 + 限速；付费墙全文谁都拿不到，靠既有 OA 兜底，不在本子包强求。

## 一、总体架构（分层 + 数据流）

```
输入(title/DOI/freeform, 批量)
   │  cli/__main__  → 读输入、装配 ScholarConfig
   ▼
query      : 原始输入 → ScholarQuery(q, 年份, 分页, hl)
   ▼
serp.build_scholar_url(ScholarQuery) → Scholar 结果页 URL
   ▼
fetcher    : 取回结果页 HTML —— 分层降级 + 反爬(见 §4)
   │           L1 curl_cffi(静态) ──被拦──► L3 nodriver/patchright(浏览器)
   │                                    ──弹验证码&开关开──► captcha(打码, 默认关)
   │           翼A proxy(住宅代理, 默认关)  翼B 限速/退避/冷却/行为
   │  ┌─ 可选商业路径: serp.parse_serpapi ← scholar_serpapi.search_scholar(SerpApi, 合规, 默认需 key)
   ▼  ▼
serp.parse_serp(HTML) / parse_serpapi(JSON) → List[ScholarResult]
   ▼
（每条结果）
download   : 结果自带 pdf_links → 下载校验(复用父包 download/landing/render_fetch)
   │           未果 → OA 兜底(复用父包 fulltext_fetcher OA 源, 按 DOI/标题)
   ▼
naming     : ScholarResult/Paper → 标准化文件名 → out/pdfs/
   ▼
pipeline   : 汇总 → metadata.jsonl / summary.json / results.csv / report.html
   ▼
logsetup(全程): run.log + attempts.jsonl + serp.jsonl (结构化事件)
```

**与既有 `fulltext_fetcher` 的关系（尽量复用、不重造轮子）**：

| 需求 | 复用既有 | 说明 |
|---|---|---|
| HTTP 客户端(OA 源/限速/退避/熔断/机构代理钩子) | `fulltext_fetcher.http_client.HttpClient` | Scholar 页专用取回走 `fetcher`(反爬引擎)，OA 兜底仍走 `HttpClient` |
| 落地页抽 PDF 直链 | `fulltext_fetcher.landing.extract_pdf_links(html, base_url)` | `serp`/`download` 直接 import 复用（纯标准库、零依赖） |
| PDF 校验/下载/文件名基元 | `fulltext_fetcher.download.{download_pdf, pdf_defect, looks_like_pdf, sanitize_filename, target_name}` | `scholar.download` 薄封装复用 |
| 浏览器渲染 OA 落地页兜底 | `fulltext_fetcher.render_fetch.render_get_pdf_url`（A4 已建） | 作为 `download` 的落地页兜底引擎（合规：仅 OA，不抓 Scholar） |
| SerpApi 合规客户端 | `fulltext_fetcher.scholar_serpapi.{search_scholar, parse_organic_results, ScholarResult}`（B1 已建） | 迁移/收敛为 `scholar.serp.parse_serpapi` 的实现来源；ScholarResult 契约以 §3 为准 |
| DOI/标题解析、OA 富化 | `fulltext_fetcher.resolve.{classify_input, resolve_to_paper, title_to_doi}` | `query`/`download`(OA 兜底) 复用 |
| OA 源(Unpaywall/OpenAlex/S2/…) | `fulltext_fetcher.sources.*` + `build_sources` | 付费墙/无直链时的「其它渠道兜底」 |
| 日志 + 结构化事件 | `fulltext_fetcher.logsetup.{setup_logging, EventLog}` | 直接复用，产物目录结构对齐 |
| CSV/HTML 报表 | `fulltext_fetcher.report.{write_results_csv, write_summary_html}` | 复用；字段做 Scholar 扩展见 §6 |

### 3.1 `models.py` — 核心数据结构（最先冻结，阻塞其它一切）

```python
from __future__ import annotations
from dataclasses import dataclass, field, asdict
from typing import Any, Dict, List, Optional

@dataclass
class ScholarResult:
    """一条 Google Scholar 结果（SERP 解析或 SerpApi 解析后的统一视图；字段尽力填充、可空）。"""
    title: Optional[str] = None
    link: Optional[str] = None                 # 主/落地链接
    result_id: Optional[str] = None            # Scholar result_id / cluster 标识
    snippet: Optional[str] = None
    publication_info: Optional[str] = None     # "作者 - 期刊, 年 - 站点" 摘要串
    authors: List[str] = field(default_factory=list)
    year: Optional[int] = None
    venue: Optional[str] = None
    cited_by: Optional[int] = None             # 被引数
    versions: Optional[int] = None             # 版本数
    cluster_id: Optional[str] = None           # 版本簇 id（取全部版本用）
    pdf_links: List[str] = field(default_factory=list)      # 结果自带 PDF 直链
    resources: List[Dict[str, Any]] = field(default_factory=list)  # 原始 resources(title/format/link)
    position: Optional[int] = None
    origin: str = "serp"                        # 来源路径: 'serp'(自建HTML) | 'serpapi'(商业API)
    doi: Optional[str] = None                   # 若能从链接/富化得到
    def to_dict(self) -> Dict[str, Any]: return asdict(self)

@dataclass
class ScholarQuery:
    raw: str                                    # 原始输入
    kind: str                                   # 'title' | 'doi' | 'freeform'
    q: str                                      # 传给 Scholar 的 q 串
    num: int = 10                               # 期望条数
    start: int = 0                              # 分页偏移（每页 10）
    year_low: Optional[int] = None
    year_high: Optional[int] = None
    lang: str = "en"                            # hl 参数
    extra: Dict[str, Any] = field(default_factory=dict)

@dataclass
class SerpPage:
    """一页 SERP 的解析产物 + 元信息。"""
    results: List[ScholarResult] = field(default_factory=list)
    has_next: bool = False
    total_hint: Optional[int] = None
    blocked: bool = False                       # 命中验证码/风控页
    raw_url: Optional[str] = None

@dataclass
class FetchOutcome:
    """fetcher 取回一个 URL 的统一信封（无论哪层引擎）。"""
    ok: bool = False
    html: Optional[str] = None
    final_url: Optional[str] = None
    status: Optional[int] = None
    engine: Optional[str] = None               # 命中层: 'curl_cffi'|'nodriver'|'patchright'|'serpapi'
    blocked: bool = False                       # 命中验证/风控页
    captcha: bool = False
    proxy_used: Optional[str] = None
    elapsed_ms: int = 0
    error: Optional[str] = None

@dataclass
class ScholarFetchResult:
    """一条输入的最终结果（对齐父包 FetchResult，便于复用 report/CSV）。"""
    raw_input: str
    kind: Optional[str] = None
    query: Optional[str] = None
    title: Optional[str] = None
    doi: Optional[str] = None
    success: bool = False                       # 是否下到合规校验通过的 PDF
    pdf_path: Optional[str] = None
    pdf_bytes: int = 0
    pdf_url: Optional[str] = None
    source_used: Optional[str] = None          # 'scholar-pdf'|'oa:unpaywall'|...
    cited_by: Optional[int] = None
    n_results: int = 0
    engine_used: Optional[str] = None          # 取 SERP 命中的反爬层
    blocked: bool = False
    elapsed_ms: int = 0
    error: Optional[str] = None
    def to_dict(self) -> Dict[str, Any]: return asdict(self)
```

### 3.2 `config.py`

```python
@dataclass
class ScholarConfig:
    # —— 模式 —— 'serpapi'(商业合规) | 'self'(自建反爬) | 'auto'(先 serpapi 有 key 否则 self)
    mode: str = "auto"
    serpapi_key: Optional[str] = None          # 回落环境变量 SERPAPI_KEY
    # —— 自建取回：引擎降级顺序 —— 
    engine_order: List[str] = field(default_factory=lambda: ["curl_cffi", "nodriver"])
    impersonate: str = "chrome"                # curl_cffi impersonate 目标
    browser_engine: str = "nodriver"           # 'nodriver'|'patchright'|'seleniumbase'
    # —— 限速/退避/冷却（合规、避免触发；默认偏保守）——
    page_interval_low: float = 45.0            # 两次 Scholar 页请求最小抖动下界(秒)
    page_interval_high: float = 90.0           # 抖动上界(秒)
    rotate_ip_every_low: int = 5               # 每 N 次请求换 IP/会话(下界)
    rotate_ip_every_high: int = 15             # (上界)
    backoff_base: float = 2.0                  # 指数退避底
    backoff_cap: float = 60.0                  # 单次退避上限(秒)
    cooldown_after_block: float = 900.0        # 命中风控后冷却(秒, 默认15min)
    max_retries: int = 3
    timeout: float = 30.0
    # —— 翼A 代理（默认关=直连）——
    proxy_enabled: bool = False
    proxy_pool: List[str] = field(default_factory=list)   # 或经 env SCHOLAR_PROXIES
    # —— 翼B 打码（默认关）——
    captcha_enabled: bool = False
    captcha_provider: Optional[str] = None     # '2captcha'|'capsolver'
    captcha_key: Optional[str] = None
    # —— 下载 / 兜底 ——
    oa_fallback: bool = True                    # 结果无 PDF 时复用父包 OA 源兜底
    enable_scihub: bool = False                 # 合规风险, 默认关
    min_pdf_bytes: int = 1024
    max_pdf_bytes: int = 80 * 1024 * 1024
    # —— 批量/输出 ——
    out_dir: str = "out_scholar"
    concurrency: int = 1                        # Scholar 抓取默认串行(强合规); OA 兜底可另行并发
    num: int = 10
    year_low: Optional[int] = None
    year_high: Optional[int] = None
    resume: bool = True
    naming_template: str = "{year}_{author}_{title}"
    email: str = "anonymous@example.com"        # 供 OA 兜底(Unpaywall 需真实邮箱)
    log_level: str = "INFO"
```

## 四、分层反爬策略与降级顺序（承 R1「三层+两翼」+ R2）

### 4.1 检测模型与对应工具（谁拦你 → 加什么）

| 层/翼 | 被拦点 | 对应工具（选型见 §5） | 本子包落点 |
|---|---|---|---|
| **L1 TLS/HTTP2·3 指纹** | 加密握手就露馅 | `curl_cffi`(impersonate=chrome) | `fetcher.CurlCffiEngine`（首选、最省） |
| **L2 JS/浏览器指纹** | 需执行 JS / canvas 等指纹 | `patchright`(Playwright drop-in) / Camoufox | `fetcher.PatchrightEngine`（备选） |
| **L3 自动化协议指纹** | `Runtime.enable` 等驱动痕迹 | **`nodriver`**（benchmark 31 目标零封锁） | `fetcher.NodriverEngine`（浏览器侧首选） |
| **翼A IP 信誉** | IP 段被判风险 | 住宅/移动代理 + 轮换 | `proxy.ProxyPool`（默认关） |
| **翼B 行为/验证码** | 行为评分 / reCAPTCHA | 限速+人类化行为；打码 CapSolver/2Captcha | `fetcher` 限速 + `captcha`（默认关） |

### 4.2 降级顺序（默认策略；每步失败/被拦才进下一步）

```
① curl_cffi 静态抓 (impersonate=chrome) + [代理] + 强限速/退避
     └─ serp.detect_captcha==False 且解析出结果 → 完成(最省)
② 命中风控/验证页 or 需 JS → 升级浏览器 nodriver(直连CDP) / patchright
     └─ 升级前: proxy.rotate() + 指数退避 + (连续被拦)冷却
③ 浏览器仍弹 reCAPTCHA 且 captcha_enabled==True → captcha.solve_recaptcha 注入 token 重试
     └─ 默认 captcha_enabled=False: 记 blocked 事件、跳过该条(不硬刚)
④ SERP 拿到结果 → 交 download; 结果无直链/付费墙 → OA 兜底(复用父包)
```

**升级触发条件（`fetcher` 内判定）**：HTTP 429/503；`serp.detect_captcha` 命中；`/sorry/` 重定向；结果块为空但页面非空；连续 N 次同层失败。

### 4.3 限速 / 退避 / 冷却 默认值（`ScholarConfig`，偏保守、以“避免触发”为纲）

| 参数 | 默认 | 说明（依据 R1/R2） |
|---|---|---|
| `page_interval_low/high` | **45 / 90 s** | 两次 Scholar 页请求之间随机抖动（R1：45–90s 抖动） |
| `rotate_ip_every_low/high` | **5 / 15** 次 | 每 5–15 次请求换 IP/会话（R1） |
| `backoff_base` / `backoff_cap` | 2.0 / 60 s | 429/5xx 指数退避、尊重 Retry‑After |
| `cooldown_after_block` | **900 s** | 命中硬风控后整体冷却（或强制换 IP+会话） |
| `concurrency` | **1**（串行） | Scholar 抓取默认串行；OA 兜底可另配并发 |
| `max_retries` | 3 | 单页最大重试 |

### 4.4 开关（默认全部“最安全”）

- `proxy_enabled=False`（直连；开启需自备住宅代理池，经 `--proxy` 或 `SCHOLAR_PROXIES`）。
- `captcha_enabled=False`（**默认关**；打码=灰色+持续付费+随 reCAPTCHA 升级可能失效，仅作可选最后手段）。
- `enable_scihub=False`（合规风险，默认关）。
- `mode='auto'`：有 `serpapi_key` 走**商业合规 SerpApi**（不自建反爬），否则走自建栈。

## 五、选型结论（库 + 许可证 + 可选依赖隔离）

### 5.1 采用 / 整合 / 不用（承 R1/R2，2026‑07 核验）

| 角色 | 库 | 许可证 | 裁决 | 在本子包 |
|---|---|---|---|---|
| L1 静态抓 | **curl_cffi** ~5.9k | MIT | **直接采用** | `fetcher` 首选引擎 |
| L3 浏览器 | **nodriver** ~4.4k | **AGPL‑3.0** | **直接采用**（内部/研究用；对外服务需法务确认） | `fetcher` 浏览器首选 |
| 浏览器备选(Playwright栈) | **patchright** ~2.8k | Apache‑2.0 | 整合/备选 | `fetcher` 备选引擎 |
| 工程外壳/调试 | **SeleniumBase** ~12.8k（`solve_captcha`,CDP Mode） | MIT | 整合/备选 | 调试期或备选引擎 |
| 一键批量骨架 | **botasaurus** ~4.8k（缓存/并行/重试） | MIT | 借鉴设计 | `pipeline` 设计参考（不强依赖） |
| 打码(reCAPTCHA) | **2captcha-python** ~0.77k(v2.0.9) / **python3-capsolver** | MIT | 采用(默认关) | `captcha` 适配器 |
| 商业 Scholar API | **SerpApi**(全字段+法律盾) / Scrapeless / SearchApi | 商业 | 采用(合规首选路径) | `serp.parse_serpapi` + B1 客户端 |
| GS 元数据库 | scholarly（+住宅代理，低频） | Unlicense | 整合(仅低频) | 可选：`serp` 备选来源 |
| PDF 下载器 | pypaperretriever / paper-fetch / PyPaperBot | MIT | 参考/整合 | OA 兜底思路对照（本项目已具备） |
| **不用** | undetected-chromedriver（作者弃）、playwright-stealth（POC）、Python‑Tls‑Client 原版（停维护） | — | **不用** | — |
| **谨慎** | DrissionPage（v4 起非商用/需授权）、hrequests（2024‑12 停更） | 特殊 | 仅个人研究/不引入商用 | 不作默认依赖 |

打码报价参考（角度4/R2）：2Captcha reCAPTCHA v2 ≈ **$1–3/1k**；Anti‑Captcha ≈ **$0.0005/token** 起。

### 5.2 关键事实（R2）：**当前主要失败是付费墙 403 / 无 OA 候选，几乎不是 reCAPTCHA**

→ 因此 `captcha` 只在“**直抓 GS 拿被引/版本等原生字段**”时才需要；若目标是“**拿全文 PDF**”，走 OA 下载（本项目既有能力）基本碰不到 reCAPTCHA。这决定了默认 `captcha_enabled=False` 的合理性。

### 5.3 可选依赖隔离方式（硬规则）

1. **不进强制依赖**：`fulltext_fetcher/requirements.txt` 不新增 curl_cffi/nodriver/patchright/seleniumbase/2captcha-python/python3-capsolver；另出 `requirements-scholar-optional.txt` 仅作说明。
2. **一律函数内延迟导入**：每个引擎/打码适配器 `try: import X except ImportError: return {available:False, reason:...}`（**沿用 B1/A4 已验证的信封模式**）。
3. **统一「不可用信封」**：`{"available": False, "reason": "need <dep/key>"}`；缺库、缺 key、开关关闭都走同一形态，`pipeline` 据此优雅跳过并记事件。
4. **许可证隔离**：AGPL 的 `nodriver`、非商用的 `DrissionPage` 仅在用户显式开启对应引擎时才 import，默认路径（curl_cffi/serpapi）为宽松许可。

## 七、实现顺序与并行切分（供总指挥编排构建 DAG）

> 目标：最大化并行、最小化接口漂移。**冻结契约（§3）后**，多数模块可独立并行实现，各自带离线 selftest。

```
Phase 0（地基·阻塞全部·先做/单点冻结）
  P0  models.py + config.py           ← 冻结 §3 数据结构与 ScholarConfig（其余全部依赖它）
      logsetup.py（复用父包，薄封装）

  P1a query.py         P1b serp.py(纯解析,含 parse_serpapi 迁移 B1)
  P1c naming.py        P1d proxy.py            P1e captcha.py
      （P1b 可复用父 landing；P1e/P1d 只读 config，天然隔离）

Phase 2（依赖 P1·可 2 路并行）
  P2a fetcher.py（依赖 proxy/captcha/serp.detect + 引擎适配器；反爬核心，工作量最大）
  P2b download.py（复用父 download/landing/render_fetch/sources + models）

Phase 3（集成·串行）
  P3  pipeline.py（依赖 P0/P1/P2 全部）→ 然后 cli.py + __main__.py

Phase 4（验收）
  P4  selftest_e2e.py（依赖 pipeline；不联网端到端）→ 全子包 compileall + 各 *_OK
```

**并行分配建议**（约 8 路可同时开工）：P0 单人先行并广播契约；P0 完成后 P1a–P1e（5 人）+ 预备 P2a/P2b（2 人先读依赖/搭骨架）。**跨成员必对齐点**：`ScholarResult` 字段命名（serp ↔ download ↔ naming ↔ pipeline）、`FetchOutcome`（fetcher ↔ pipeline）、事件名（各模块 ↔ logsetup/report）——这些一律以 §3/§6 为唯一真源，变更走总指挥。

**验收标准（每模块）**：`python -m fulltext_fetcher.scholar.<mod>` 打印 `<MOD>_OK`；`python -m compileall fulltext_fetcher/scholar/<mod>.py` 退出 0；不新增强制依赖；无 linter 报错。整包验收：`python -m fulltext_fetcher.scholar --selftest` → `SCHOLAR_E2E_OK`。

## 八、合规边界（全子包统一，必须写进各模块头部）

1. **优先正门**：能用角度2 开放 API / 商业合规 SerpApi 就不自建直抓；自建栈仅供**研究/小规模自用**。
2. **默认最安全**：代理、打码、Sci‑Hub **默认全关**；只“避免触发”，不主动对抗；限速/退避/冷却默认从严。
3. **ToS/robots**：直抓 Google Scholar 属灰色，违反其 ToS 与 robots.txt；使用者自负；对外提供服务前须过法务（含 nodriver=AGPL、DrissionPage 非商用等许可证传染）。
4. **付费墙**：付费墙全文不在本子包强求；仅在用户拥有合法权限时经既有 OA/机构通道兜底。
5. **打码=灰色+持续付费**，仅作可选最后手段，且随 reCAPTCHA 升级可能失效。
