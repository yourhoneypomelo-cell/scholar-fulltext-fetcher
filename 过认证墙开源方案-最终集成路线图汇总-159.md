## 一、TL;DR（决策版）

1. **技术栈方向正确、无需换栈**：现有 **route-B = nodriver 真 Chrome + curl_cffi TLS 回放 + FlareSolverr `/v1` 免 Docker nodriver-shim + 浏览器内直下字节** 已是 **2026 开源最优组合**。下一步是**小步升级 + 补合规前置层**，不是引入新框架。
2. **单一最高杠杆升级 = nodriver → `zendriver`**：一步同时修掉 nodriver 的 Chrome146 cookie 崩溃 + CDP schema 漂移 + 支持 headless 过 CF + 官方 Docker（近 drop-in，pip 可装）。
3. **净增天花板已诚实见底**：免费 + CF 路线**物理到顶 ≈ 34%**（权威 `coverage.json` **326/673/32.63%** @2026-07-03 12:50:24）。开源侧剩余增量是**个位数~低两位数 pp**，集中在特定桶。
4. **最大净增来源 = A5 机构订阅（路线A），不是过认证墙开源工具**：A5 可救 ≈**94% still_missing**（Elsevier+ACS+RSC 占 85%）；**唯一 gate = 用户凭据**，框架已就绪、ROI 随凭据到手即释放。
5. **开源救不了三件事（认账）**：RSC governor 的**坏 reCAPTCHA**（`Invalid domain for site key`，真人/打码都解不了）、**数据中心 IP 过 CF**（指纹矛盾）、**纯订阅付费墙**（唯 A5/官方 TDM）。

## 三、三堵墙 × 开源最优解 × 真机实测收口

| 墙 | 桶（规模） | 门本质 | 开源最优解 | **真机实测收口** | 净增（诚实） |
|---|---|---|---|---|---|
| **① CF-managed** | ACS 128 / Wiley 23 / AIP 5 / T&F 3 | CF Managed Challenge(403) | nodriver/zendriver 过盾 → 不绑 JA3 站 curl_cffi 回放；绑 JA3 站 route-B 内直下 + `nodriver-cf-verify` 处理 Turnstile | ACS-authorchoice CF403 **已实测真解为 PDF**（`acscatal.0c01253` 4.41MB）；**但 ~93% 是 SI，QC 净增真正文 ≈1** | **个位数** |
| **② RSC-JA3** | RSC 金 OA ~8 | 伪命题：非 passive-JA3，是"浏览器解盾→curl 异 JA3 回放→403" | route-B **同会话内抓字节**（已在）；真 passive-JA3 长尾 → `curl_cffi impersonate`（`FTF_IMPERSONATE_HTTP=1` 已落地） | FS+curl_cffi 对 RSC **ROI≈0**（三源实证，cf_clearance 绑 JA3） | **≈0~低** |
| **③ RSC-governor** | RSC 金 OA ~8（+订阅 ~59） | 应用层 rate-gate + **坏 reCAPTCHA（不可解）** | 165 补丁：governor 检测 + landing 预热(勿直怼 articlepdf) + per-host 30–90s 限速/冷却 +（可选）住宅 IP | **补丁已落地可观测**（7/8 明确 `blocked:rsc-governor`），但**真机 0/8**——本机 IP/会话仍被压制 | **本波 0**；点估 +0~8，需**换干净 IP/更保守节流** |

**RSC governor 收口结论（-160 真机）**：补丁归因清晰化成功（`blocked:rsc-governor` 与 CF 分开），但坏 reCAPTCHA 只能"不触发"不能"解"；单条 PoC（`d5ra08493h` 484KB）通、批量 8 条 0 → **生产路径未就绪，当前 defer**，待换未 burned IP + 小样本 ≥3 篇再验。RSC 订阅 ~59 **唯 A5**。

### 4.2 明确不采纳 / 淘汰（附因）
- **Go 系 TLS**（azuretls / utls / tls-client / requests-go）：Go 栈违背零重依赖，curl_cffi 已等效覆盖。
- **patchright / camoufox / Scrapling**：Playwright/Firefox 驱动，automation-protocol 第③层更弱，不为过硬 CF 替换 nodriver；camoufox 2026 维护动荡=HOLD。
- **undetected-chromedriver / cloudscraper / cfscrape / puppeteer-extra-stealth / hrequests**：淘汰/停滞（uc 作者转投 nodriver；cloudscraper 死；puppeteer-extra-stealth 2025-02 停维）。
- **免费开源代理池**（proxy_pool / ProxyBroker2 / requests-ip-rotator）：数据中心/公共 IP，RSC governor 盯死（通过率 ~5%），且免费代理=蜜罐（合规+安全风险）。
- **Scrapoxy**（2026-02 EOL）、**scidownl / PyPaperBot**（SciHub 依赖失效+合规风险，本项目一律不采灰色源）、**NopeCHA / Theyka**（成功率低/许可不明）。

### 4.3 源码借鉴（不整包引入）
`auto-paper-harvester`（24 DOI 前缀路由表 + 14 出版社族 CSS selectors）、`metapub FindIt`（publisher registry）、`botasaurus`（Bézier 鼠标行为层）、自建 `ProxyLease`（~100 行 sticky+预算+探封轮换）。**A5 侧**：`instsci`(MIT/star96) 出版商登录 URL 表 + detect 启发式、`SciTeX`(**AGPL 仅借思路**) 的 provider 抽象 + verify_live（详见 `A5三机制适配预研-141.md`）。
