## 4. 一键重跑命令模板(门过即发,PowerShell,仓库根执行)

前置(一次性,CF 桶必须;非 CF 桶带着也无害——download.py 只在遇到 CF 质询时才调 FS 端点):

```powershell
# 终端A:常驻 FlareSolverr 兼容端点(免 Docker,nodriver;有头模式通过率高)
python tools/flaresolverr_nodriver.py --port 8191

# 终端B:环境变量(本会话)

$env:FULLTEXT_EMAIL   = "you@uni.edu"   # 换成真实邮箱(Unpaywall 礼貌要求)
```

按优先序逐分片(总指挥定序 elsevier→acs→rsc,其余按 ROI 收尾;每条独立 RUNROOT,可 2 条并行,但 CF 桶同时最多 1 条——FS 端点是串行求解):

```powershell
# ① elsevier 344 | 非CF低ROI,轻量重跑捞 OA 例外;常规并发即可
python run_all.py -f out/still_missing_shards/elsevier.txt -o out/rerun_elsevier --email $env:FULLTEXT_EMAIL -c 3

# ② acs 75 | CF403 高ROI(FS+curl_cffi 回放已实测可落 PDF);低并发保 FS 稳定
python run_all.py -f out/still_missing_shards/acs.txt -o out/rerun_acs --email $env:FULLTEXT_EMAIL -c 2

# ③ rsc 58 | JA3 绑定,FS 回放仍 403 → 轻量过一遍捞 OA,主力等路线B(浏览器内直下)
python run_all.py -f out/still_missing_shards/rsc.txt -o out/rerun_rsc --email $env:FULLTEXT_EMAIL -c 2

python run_all.py -f out/still_missing_shards/springer.txt -o out/rerun_springer --email $env:FULLTEXT_EMAIL -c 3

# ⑥⑦⑧ aip 3 / iop 4 / other 28 | 小桶收尾(aip 是 CF 桶须带 FS;other 内 T&F/OUP 为 CF 桶、MDPI 应常规可救)
python run_all.py -f out/still_missing_shards/aip.txt   -o out/rerun_aip   --email $env:FULLTEXT_EMAIL -c 2

python run_all.py -f out/still_missing_shards/other.txt -o out/rerun_other --email $env:FULLTEXT_EMAIL -c 2
```

限速说明:仓内 `HttpClient` 自带按域限速(pipeline 初始化即配置),无需额外参数;并发建议 CF 桶 c=2、非 CF 桶 c=3,礼貌真流量。run_all 默认消费 QC 黑名单(`out/qc_merge_{highconf,union}_wrong.csv`)+ 跨批 resume,发射时自动按**当时最新**净成功集去重——即使采用 §2 方案 B,也不会重复下载任何已判对的件。

跑后统一口径:每个 RUNROOT 自动产出 coverage.json / still_missing.txt / run_all_summary.json;全局汇总仍以 `python tools/build_coverage.py`(coverage-root=out)为准。
