## M.4 OpenAlex 口径订正:mailto 礼貌池 2026-02 已废,免费 key 是唯一动作;Crossref 礼貌池仍有效、别混改

- **OpenAlex**:`mailto` 参数 2026-02 起被**忽略**(官方 deprecations:Polite pool → Replaced by API keys)。不设 key 只剩 **$0.10/天**(仅够 demo),批量必 **429/降级**。仓内代码**已就绪**(aggregators 对 OpenAlex 同时传 mailto+api_key,mailto 被忽略无害、api_key 生效)——**唯一动作是运营层**:`openalex.org/settings/api` 申请免费 key(30 秒)→ 设 `OPENALEX_KEY` 环境变量。按 DOI 单条查在免费档"无限",本仓用法成本≈0。
- **Crossref 的 mailto 礼貌池仍有效**——crossref/preprints 源的 mailto 保留是**正确的**,⚠️ 千万别看到"礼貌池已废"就把两家一起改。
- 证据:`检索成果-OpenAlex强制APIKey核查与免费Key落地清单.md`(核验 2026-07-02)。**教训**:外部 API 的"礼貌池/限速"约定有时效,同类源之间不可类推,改口径前先查各自官方 deprecations 页。

## M.5 openaire 新根因(150 审计):递归收 URL 不校验记录 DOI → 邻近记录错 PDF;假阳不只 websearch,聚合源同样会错

- **根因**:`OpenAire.find_candidates` 旧版对 OpenAIRE 返回的**整包 JSON 递归**抽一切"像 PDF"的 url,不校验该 url 属于哪条记录——连 `rels.rel[]`(引用/相关文献=**别的论文**)的 webresource 一起收。实证:`10.1016/j.susc.2014.02.019` 与 `10.1016/j.apcata.2013.07.028` 两个不同 Elsevier DOI 都误收筑波大学仓库 **record/38855** 的 CARS 光谱 PDF(同 MD5,详见《检索成果-标题输入路径-端到端实测与错文件揭示.md》)。
- **修复(已派 148,现已在码)**:`fulltext_fetcher/sources/aggregators.py` 的 OpenAire 已改**记录级解析**——`_openaire_record_urls(data, paper.doi)` 仅收「记录 pid/DOI == 查询 DOI」的记录**自身** URL,再过"像全文/PDF"白名单,rels 相关文献一律不收。
- **教训(升级 L.5 的认知)**:此前定论"DOI-keyed 源 0 假阳、天然干净"**需要收窄**——该结论只对"链接由 DOI 权威解析"的源(unpaywall/openalex/europe_pmc/crossref/s2/publisher_oa)成立;**openaire 这类"按 DOI 查但返回一包含邻居记录的聚合 JSON"的源,解析层不做记录级 DOI 校验照样错配**。凡"一次查询返回多记录/带关联记录"的聚合 API,candidates 必须绑记录校验,否则就是下一个 websearch。
