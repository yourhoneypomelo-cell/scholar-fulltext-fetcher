# fulltext_fetcher 实现路径与原理

> 记录"为什么这么做"与"怎么做",供维护与二次开发。版本 1.0 · 2026-07-01 · 谷歌学术人机认证-141

---

## 1. 北极星目标与核心判断

**目标**:输入 DOI/标题 → 全自动榨干全网可及免费全文 → 下载入库,高成功率 + 高速率,全局程序化(运行+读日志即可迭代)。

**核心判断(来自七角度研究,见 `检索成果-00`)**:与其硬刚 Google Scholar 的人机认证(军备竞赛、灰色、不稳定),不如**走开放获取正门**——用 DOI/标题向十几个免费学术 API/仓储要 OA 全文。它们**没有验证码**、合规、稳定,且能拿到同样的元数据 + OA PDF。本程序就是这条主线的生产级落地。

## 2. 开源借鉴(取长补短)

| 项目 | 借鉴点 |

## 3. 架构总览

```
输入(DOI/标题/arXiv)
   │  resolve.py: 判类型 → 富化元数据(OpenAlex 单条免费 / Crossref 反查标题)
   ▼
Paper{doi,title,arxiv_id,pmcid,pmid,is_oa,...}
   │  pipeline.py: 按优先级遍历源(sources/*),每源产出 PdfCandidate[]
   ▼                直链(pdf/render)即时下载;首个成功即短路;落地页最后兜底
候选打分排序 → download.py: HTTP 200 + 非 HTML + %PDF 魔数 + 大小达标 → 落盘
   │
   ▼  结构化日志: attempts.jsonl(每事件) + metadata.jsonl(每条) + summary.json(汇总)
```

模块职责:
- `config.py` 全部可调参数(源顺序、并发、限速、阈值、开关、凭据)。
- `models.py` 数据结构(`WorkInput/Paper/PdfCandidate/Attempt/FetchResult`),全 dataclass 便于序列化。
- `http_client.py` 带**重试/退避 + 按域限速 + 熔断器**的 HTTP 客户端(线程安全,多 work 共享)。
- `resolve.py` 输入解析与元数据富化(DOI/arXiv/标题三类)。
- `sources/` 各源连接器:`base`(基类+注册表)、`aggregators`(Unpaywall/OpenAlex/S2/Crossref/CORE/OpenAIRE)、`repositories`(arXiv/EuropePMC/PMC/bioRxiv·medRxiv/DOAJ/Zenodo/HAL)、`scihub`(可选)。
- `download.py` 下载 + PDF 校验 + 命名落盘。
- `pipeline.py` 编排器(并发、回退、断点续跑、汇总)。
- `cli.py` / `__main__.py` 命令行入口。

## 4. 关键设计决策与原理

### 4.1 源优先级 + "首个直链成功即短路"
源按"直链命中率/速度/稳定性"排序(见 `config.DEFAULT_SOURCE_ORDER`):Unpaywall → OpenAlex → Europe PMC → arXiv → bioRxiv → S2 → PMC → CORE → Crossref → DOAJ → OpenAIRE → HAL → Zenodo。
下载模式下逐源尝试,**第一个成功下载的直链即停**——既保证回退覆盖(高成功率),又避免无谓请求(高速率)。实测一条命中靠前源的 DOI 仅需 3–7s。

### 4.2 候选打分(PdfCandidate.confidence)
每个候选带 `kind`(pdf/render/landing/file)与 `confidence`(0–100)。直链(pdf/render)优先于落地页;`url_for_pdf`/`pdf_url` 给高分,`oa_url`/落地页给低分。排序后先试高分直链,落地页仅作兜底(可能 302 到真实 PDF)。

### 4.3 PDF 校验(解决"落地页冒充 PDF"第一坑)
下载后必须:HTTP 200 → `Content-Type` 不含 `text/html` → 内容前 1KB 含 `%PDF` 魔数 → 大小 ≥ 阈值。任一不过即判失败并记录原因(`landing-page`/`not-pdf`/`too-small`/`http-xxx`),绝不把 HTML 当 PDF 落盘。

### 4.4 按域限速 + 重试退避 + 熔断器(高速率 & 稳)
- 每个 host 维护最小请求间隔(默认 0.34s;arXiv API 强制 3s),线程安全,多并发共享。
- 429/5xx 指数退避并尊重 `Retry-After`。
- **熔断器**:某 host 连续 2 次连接类错误(超时/SSL)即本次运行内**直接跳过**,避免反复超时拖垮整体(实测把环境内无法连通的 NCBI 从拖慢 25s/条 降到几乎 0)。

### 4.5 元数据富化减少重复定位
resolve 阶段用 OpenAlex 单条(**按 DOI 免费**)一次性拿到 title/year/arxiv_id/pmcid/pmid/is_oa,下游 Europe PMC/PMC/arXiv 可直接用 pmcid/arxiv_id 命中,省去各源自己再定位。

### 4.6 全局程序化 + 断点续跑
所有状态进结构化日志,无需人工盯;重复运行自动跳过已成功项,只重试失败——支撑"改程序→重跑→读日志"的闭环。

## 5. 合规边界
- 主线全部是**官方开放 API / OA 仓储**,合规稳定。
- Sci-Hub 默认**关闭**,仅 `--enable-scihub` 显式开启,风险自负。
- 付费墙内全文:任何免费手段都拿不到,属预期边界(此时应走机构订阅或角度4商业服务)。

## 6. 已知局限与后续路线
- CORE 需 key(无 key 自动跳过);OpenAIRE 旧版 JSON 杂质多,已做精确过滤但召回有限。
- PMC 2026-08 将停用 oa.fcgi 转 AWS S3;现走 Europe PMC render 规避(详见源接口速查表)。
- 后续可加:HTML 落地页内 PDF 链接解析(提升 bronze OA 命中)、Zotero 直接入库、成功率按源/出版商分桶统计、异步 IO 进一步提速。
