## 〇、一句话结论（BLUF）

- **先分清"要什么"**：过 Cloudflare 有**两种产物**——① **cf_clearance cookie**（挑战页 Challenge，靠**真浏览器过**、cookie 绑 IP）；② **Turnstile token**（`cf-turnstile-response`，独立小组件，靠**token 服务或真浏览器渲染**拿，单次有效、300s 过期）。**选方案前必须先判目标站是哪种**。
- **开源自托管**（零边际成本、无数据外泄）应作**首选**：本仓已用 **nodriver**，**低成本升级到 zendriver** 即可稳过大多数 **Turnstile 组件**；挑战页 cookie 走 **byparr**（Camoufox 内核，**与本仓现有 `flaresolverr.py` 的 `/v1` API 完全兼容，换 URL 即用**）。
- **第三方付费**只作**默认关、计量兜底**：**CapSolver**（最便宜 + 最快，$0.6–1.2/千，本仓 `captcha.py` 已接其 SDK）作主兜底，**2Captcha**（Turnstile 基准成功率最高 ~97%，但贵、慢）作**高价值目标的准确率兜底**。
- **对本项目的诚实判断（承接 141/155/165）**：本仓失败**主因是出版商付费墙 + RSC 自家 `governor` 的"坏 reCAPTCHA"（site-key/域不匹配，真人和打码都解不了）**，**不是 Turnstile**。所以**验证码求解对本项目是低频兜底、非主路径**——正解仍是「多源 OA 回退 + 合规正门（SerpApi / 出版商开放 API / 机构订阅）」。**Turnstile 求解只用在你有权访问、且恰好被独立 Turnstile 组件挡住的下载站。**

## 一、统一评分维度与权重

7 维，每维 **1（差）–5（优）**；`↓` 的维度**分越高=成本/风险越低**。加权求和得 **综合（0–5，一位小数）**。

| 代号 | 维度 | 权重 | 说明 |
| --- | --- | :--: | --- |
| **T** | Turnstile 能力 & 成功率 | 25% | 对现代 Turnstile（含 Managed）的真实通过能力/基准成功率 |
| **P** | 可编程 / 无头 / API | 15% | 能否脚本化、无头、有稳定 API（非人辅扩展） |
| **D** | 部署成本 ↓ | 10% | pip 零基建 > 单 API key > Docker > 需显示环境 |
| **$** | 运行成本 ↓ | 15% | 自托管≈免费；付费按每千次价 |
| **M** | 维护 / 时效 / 热度 | 10% | 最近提交、版本、stars、社区 |
| **C** | 合规 / 许可风险 ↓ | 10% | 许可传染 + 是否外泄目标 URL/sitekey + ToS 灰度 |
| **F** | 本项目适配 ↓成本 | 15% | Windows / 无代理 / 已有集成点 / 低频兜底契合度 |

> 权重取向：**Turnstile 能力(25%) + 适配(15%) + 运行成本(15%)** 为重（本项目 Windows·无代理·低频兜底、且要控成本）；合规单列 10% 提醒风险。

## 二、第三方付费服务 · 原始事实表（Turnstile，2026-07-03 实测/披露）

> 成功率优先取**独立基准**（Scrapewise 2026-04「4 API 实测」：token 被目标接受口径）；厂商自称 95–99% 另注。价格为各家 Turnstile 每千 token 披露价（USD）。

| 服务 | Turnstile 成功率 | 价格 / 1k token | 均解时长 | API 任务类型 | 现成 SDK / 本仓状态 |
| --- | :--: | :--: | :--: | --- | --- |
| **CapSolver** | ~89%（基准）/ 95–99%（自称） | **$0.60–1.20** | **~4–5s** | `AntiTurnstileTaskProxyLess` | `python3-capsolver`，**本仓 `captcha.py` 已接**（仅 reCAPTCHA，需补 Turnstile 任务类型） |
| **2Captcha** | **~97%（基准最高）** | $1.00–2.99 | ~15–18s | `TurnstileTaskProxyless` | `2captcha-python`，**本仓 `captcha.py` 已接**（同上） |
| **Anti-Captcha** | ~95% | $1.80 | ~14s | `TurnstileTaskProxyless` | `anticaptchaofficial`（本仓未接） |
| **CapMonster Cloud** | 99%（自称，无独立 Turnstile 基准） | $1.30 | 快 | 2Captcha 兼容 API | 2Captcha 兼容 SDK 可复用 |
| **NopeCHA** | ~82%（基准最低） | $0.60 | ~7s | 统一 token API | 官方 SDK（本仓未接） |

**Turnstile token 服务通用调用契约**（两家一致）：`createTask`（提交 `websiteURL`+`websiteKey`[+挑战页 `action`/`data`/`pagedata`]）→ 轮询 `getTaskResult` 拿 `solution.token` → 把 token 作 `cf-turnstile-response` 提交给目标站。**token 单次有效、300s 过期、需与解题时同一 UA/IP 语境使用**；**挑战页（非独立组件）**要额外抓 `turnstile.render` 的 `action/cData/chlPageData`，复杂度高。

## 五、成本估算（按本项目量级）

前提：Turnstile 求解在本项目是**低频兜底**（still_missing 约 600 内仅少量落在"独立 Turnstile 挡住、且有权访问"的下载站）。自托管系**边际成本≈0**（仅自有算力/IP）。付费按 token 计：

| 求解量 | CapSolver（$0.6–1.2/k） | 2Captcha（$1.0–2.99/k） | CapMonster（$1.3/k） | NopeCHA（$0.6/k） |
| --- | --- | --- | --- | --- |
| 100 次 | $0.06–0.12 | $0.10–0.30 | $0.13 | $0.06 |
| 1,000 次 | $0.60–1.20 | $1.00–2.99 | $1.30 | $0.60 |
| 10,000 次 | $6–12 | $10–30 | $13 | $6 |

**结论**：即便把付费服务当兜底，本项目量级下**总花费仅几美元级**，成本**不是**决策瓶颈；决策瓶颈是**合规**与**是否真的是 Turnstile**（多数场景不是）。因此**"先自托管、付费仅计量兜底"**在成本上完全可承受，且把数据外泄面降到最低。

## 六、合规与许可风险

- **行为灰度（两类都存在）**：自动过 Cloudflare / 求解验证码以抓取，均**违反目标站 ToS**，属灰色；**仅限对你"有权合法访问"的 OA/订阅内容**使用。
- **付费服务额外风险**：调用时会把**目标 URL + sitekey**（挑战页还含 `chlPageData` 等）**发往外部打码农场** → **数据外泄面**；且为**持续付费**、部分机构/合同**明令禁止**第三方打码。→ 合规维度 C 给 2 分即因此。
- **许可**：`byparr` = **GPL-3.0**（作**独立服务进程**经 HTTP 调用，传染风险低；**勿把 GPL 源直接 import 进本仓**）；`Theyka/Turnstile-Solver` = **NOASSERTION（许可不明）**，商用/分发前必须澄清，故仅登记；`nodriver/zendriver/EzSolver/SeleniumBase/cloudscraper/FlareSolverr/各付费 SDK` = **MIT 系**（宽松）。
- **总原则（承接 155/165）**：**能走开放/合规 API（SerpApi、出版商开放 API、机构订阅正门）就别硬刚验证码**；RSC 第二道"坏 reCAPTCHA"**任何求解器都解不了**（site-key/域不匹配），别把预算浪费在打码上——那是**限速 + landing 预热 + 住宅 IP + 会话复用**（-165 补丁）的活。

## 七、集成示例（贴合本仓现有接口 · 建议样例，未落代码）

### 7.1 挑战页 cookie —— 用 byparr 复用现有 `flaresolverr.py`（**零代码改动**）
本仓 `fulltext_fetcher/flaresolverr.py` 已是标准 **FlareSolverr `/v1` 客户端**；byparr **同 API**，只需把端点指向 byparr 容器：

```bash
# 启动 byparr（Windows 经 Docker Desktop；Camoufox 内核，过 Turnstile 挑战页开源最强）
docker run -d --name byparr -p 8191:8191 --restart unless-stopped ghcr.io/thephaseless/byparr:latest
# 让本仓现有客户端直接用它（换 URL 即可，solve()/fetch_via_flaresolverr() 无需改）
set FLARESOLVERR_URL=http://localhost:8191      # PowerShell: $env:FLARESOLVERR_URL="http://localhost:8191"
```

```python
# 现有接口即可拿到过挑战后的 HTML + cf_clearance cookie（绝不抛、连不上优雅返回 None）
from fulltext_fetcher.flaresolverr import solve
sol = solve("https://protected.example.org/article/xxx")   # cfg 可注入 flaresolverr_url/timeout
if sol:
    html, cookies = sol["html"], sol["cookies"]            # cookies 可回放给 curl_cffi 直连续下载
```
> 注意（承接 165 N.3）：对 **JA3 绑定型强 CF 站**，"solve 拿 cookie → 换 HTTP 客户端回放"仍可能 403；byparr 价值在**普通 CF 挑战页**，对 RSC 帮助有限。

### 7.2 独立 Turnstile token —— 建议在 `scholar/captcha.py` 加 `solve_turnstile()`（沿用现有信封契约）
现有 `solve_recaptcha()` 只覆盖 reCAPTCHA v2/v3。**建议**新增一个同契约、默认关、缺库优雅降级的 Turnstile 分支（下为**示例草图**，本任务只读未落码）：

```python
# 建议样例：与现有 _unavailable()/信封 {available, token|error|reason} 完全一致；provider 复用 captcha_provider
def solve_turnstile(site_key: str, page_url: str, cfg, *, action=None, cdata=None):
    if not getattr(cfg, "captcha_enabled", False):
        return {"available": False, "reason": "captcha disabled"}   # 默认关
    provider = (getattr(cfg, "captcha_provider", "") or "").lower()
    key = getattr(cfg, "captcha_key", None)
    if not provider or not key:
        return {"available": False, "reason": "need captcha key"}
    if provider == "capsolver":
        try:
            import capsolver                                        # 延迟导入·缺库降级
        except ImportError:
            return {"available": False, "reason": "need python3-capsolver"}
        try:
            sol = capsolver.solve({"type": "AntiTurnstileTaskProxyLess",
                                   "websiteURL": page_url, "websiteKey": site_key,
                                   **({"metadata": {"action": action}} if action else {})})
            tok = (sol or {}).get("token")
            return {"available": True, "token": tok} if tok else {"available": True, "error": "capsolver: empty"}
        except Exception as e:                                      # 任何异常→失败信封,绝不外抛
            return {"available": True, "error": f"capsolver: {e}"}
    if provider in ("2captcha", "twocaptcha"):
        try:
            from twocaptcha import TwoCaptcha
        except ImportError:
            return {"available": False, "reason": "need 2captcha-python"}
        try:
            res = TwoCaptcha(key).turnstile(sitekey=site_key, url=page_url, action=action, data=cdata)
            tok = res.get("code") if isinstance(res, dict) else res
            return {"available": True, "token": tok} if tok else {"available": True, "error": "2captcha: empty"}
        except Exception as e:
            return {"available": True, "error": f"2captcha: {e}"}
    return {"available": False, "reason": f"need known captcha provider (got {provider!r})"}
```
拿到 `token` 后作 `cf-turnstile-response` 随表单/请求提交给目标站；**token 单次有效、300s 过期**，超时即重取。落地位置与新增 config 字段（如 `captcha_key`/`captcha_provider` 已有，Turnstile 无需新字段）**须先报总指挥统一加**，不擅自改核心码。

## 八、对本项目的落地建议（按"改动小→收益大"）

1. **默认路径不变**：验证码求解**默认关**（`captcha_enabled=False`）、失败即停，保护唯一干净 IP —— 与现有 `captcha.py`/`fetcher.py` 设计一致。
2. **自托管优先（零成本、无外泄）**：需过 **Turnstile 组件** → 维持 **nodriver**、可小样本对照升级 **zendriver**（治 CDP schema 漂移/事件循环，见 165）；需过 **CF 挑战页 cookie** → 上 **byparr**，**直接复用现有 `flaresolverr.py`**（换 `FLARESOLVERR_URL`）。
3. **付费仅计量兜底**：自托管失败且**确属独立 Turnstile、且有权访问** → **CapSolver** 主兜底（便宜快、SDK 已在仓），**2Captcha** 作准确率兜底（Turnstile 97%）。默认关、按次计量、设每日上限。
4. **不投入**：任何"求解器"对 **RSC 第二道坏 reCAPTCHA 无效**（site-key/域不匹配）；NopeCHA（成功率最低）不用；`Theyka` 许可不明仅登记。
5. **合规闸门**：启用付费前确认目标内容"有权访问" + 无机构/合同禁令；优先「SerpApi / 出版商开放 API / 机构订阅正门」。
