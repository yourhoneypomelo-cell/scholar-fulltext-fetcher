## 〇、一页速览（TL;DR）

- **直接采用（核心两件套）**：`curl_cffi`（L1 TLS 指纹，静态抓 Scholar 结果页）＋ `nodriver`（L3 自动化协议指纹，浏览器侧兜底）。二者是 2026 benchmark 的实测赢家，且都活跃。
- **吸收整合（工程外壳 / 备选）**：`SeleniumBase`（CDP Mode + `solve_captcha()`，工程化最完善、交互调试友好）；`botasaurus`（一体化框架，`google_get(bypass_cloudflare=True)` + 缓存 / 并行 / 重试，契合"一键批量"）；`patchright`（若被锁死在 Playwright 栈，L2/L3 最佳 drop-in）。
- **不用**：`undetected-chromedriver`（作者已弃、引导转 nodriver）；`playwright-stealth`（POC，够不到 L3）；`Python-Tls-Client` 原版（停维护）。
- **谨慎 / 有坑**：`DrissionPage`（能力优雅，但 **v4 起许可证转"非商用 / 需授权"**，商用有法务风险）；`hrequests`（能力全但 **2024-12 起停更**，其能力已被 botasaurus-requests 继承）。
- **最优路径**：`curl_cffi + 住宅代理 + 限速退避` 先静态抓 → 被拦再上 `nodriver` → 仍弹验证码才接 `打码服务`（CapSolver/CapMonster，见角度3）；PDF 走本仓已有 `fulltext_fetcher`（出版商 OA 落地页多数**无需**反爬）。**先定位被拦的层，再加最便宜的对应工具**。

## 二、逐库尽调与裁决理由

### 2.1 curl_cffi — 直接采用（L1 首选）
- **是什么**：`curl-impersonate` 的 Python 绑定，在加密层逐字节复刻真实浏览器的 TLS/JA3/JA4 与 HTTP2 帧顺序；2026 新增 **HTTP/3 指纹**与**运行时可更新指纹库**（Chrome 出新版无需 `pip upgrade`，拉 DB 即可）。
- **为何采用**：轻量极快、无浏览器进程；角度3 实测"21 行包装即覆盖 31 目标里 26 个"，与一整个 Chromium fork 打平。Scholar 结果页大部分可静态解析，这是**性价比最高**的一层。
- **注意**：仅解决 L1；页面若强依赖 JS 或触发行为评分，需升级到浏览器侧。近期版本修了重定向型 SSRF，接受外部 URL 时设 `allow_redirects="safe"`。

### 2.2 nodriver — 直接采用（L3 首选浏览器侧）
- **是什么**：undetected-chromedriver 的**官方继任者**（同作者）；纯异步、**直连 CDP**，无 Playwright shim、无 `Runtime.enable` 启动序列。
- **为何采用**：2026 benchmark（7 工具 / 31 CF 目标 / 651 判定）里**唯一 31 目标零封锁**——因为胜负手是 L3"怎么驱动浏览器"，Playwright 系全在这层留痕。
- **注意**：① **AGPL-3.0** 许可证；② 让你"**不被识别**"，但**不自动解** Turnstile/reCAPTCHA token，硬验证仍需打码兜底；③ 异步 API，需按 asyncio 组织代码。

### 2.3 SeleniumBase — 吸收整合 / 备选（工程外壳）
- **亮点**：`UC Mode`（改版 uc_driver，靠"断连/重连 WebDriver"避检）+ `CDP Mode`（MyCDP，纯 CDP 更隐蔽），且 **v4.44.2 起内置 `sb.solve_captcha()` 点选 Turnstile**；带 Recorder 录制、pytest 集成、可把 Playwright 变 stealth。维护极勤（1157 releases、v4.50.1 2026-06-18）。
- **定位**：需要"可视化调试 + 断言 + 录制脚本"或团队工程化时，比裸 nodriver 更省心；可作为浏览器侧的**备选/上层封装**。

### 2.4 botasaurus — 吸收整合（一键批量的现成骨架）
- **亮点**：`@browser / @request / @task` 装饰器一体化；`driver.google_get(url, bypass_cloudflare=True)` 伪装为"从 Google 来的自然流量"；human mode 类人鼠标；**缓存 / 并行 / 重试 / 结果落盘**开箱即用；HTTP 侧 `botasaurus-requests` 是 hrequests 去 Playwright 依赖的轻量分支。
- **定位**：与本组"抓元数据 + 批量 + 文件名标准化 + 缓存"目标高度契合，**其工程骨架值得吸收**（哪怕不整包引入，也可借鉴装饰器 + 缓存 + 并行的设计）。风险是框架较重、绑自研 driver。

### 2.5 patchright — 吸收整合 / 备选（Playwright 栈的最佳补丁）
- **亮点**：Playwright 的 **drop-in 替代**，改掉 `Runtime.enable`/`Target.setAutoAttach` 泄漏，`channel=chrome` 驱动真 Chrome；自述过 Cloudflare/Kasada/Akamai/Datadome/Fingerprint.com 等；py 包 v1.61.1（2026-06-29），自动跟随上游 Playwright。
- **定位**：**若项目已在 Playwright 生态**（复用其 API/选择器/等待），patchright 是 L2/L3 的最佳落点；否则 nodriver 更直接。

### 2.6 DrissionPage — 谨慎 / 吸收（能力优雅，但许可证有坑）
- **亮点**：自研内核**不依赖 webdriver**；`s` 模式（HTTP，等价 requests + 会话 + lxml）、`d` 模式（CDP 控浏览器）、`WebPage`（两者合一、**状态互通**，可"先 HTTP，遇到 JS 再升级到浏览器"）；跨 iframe 取元素、多标签并行、内建等待重试。⭐~12,060、持续高频发版。
- **⚠️ 关键坑（本次新发现）**：**v4 起许可证从 BSD 改为自定义条款**——GitHub 标 `Other (NOASSERTION)`，README 明确"仅限学习/个人非营利使用，未获授权不得用于商业/营利"。**本组若为个人研究/非营利可用；一旦商用需向作者授权**，务必先过法务。文档以中文为主（对本组反而友好）。

### 2.7 不用清单
- **undetected-chromedriver**：⭐虽高（~12,700，历史存量），但**推送停在 2025-07**、作者本人引导转 nodriver、对现代 Turnstile/hard challenge/TLS 指纹渐失效。新项目**不作为首选**。
- **playwright-stealth**：当前 fork 作者自述"只能过最简单检测，概念验证"；只改 navigator，够不到 L3。要补 Playwright 请用 **patchright**（或 `pw-stealth-enhanced` 补 Canvas/WebGL/Audio 指纹）。
- **Python-Tls-Client（FlorianREGAZ 原版）**：二进制不更新、issue 长期不修，已被社区 fork 取代；若需 tls-client 能力，直接用 `curl_cffi`，或活跃 fork `tls-requests`。
- **hrequests**：能力设计很好（tls-client + BrowserForge + JS 渲染 + Evomi 代理），但**自 2024-12 停更**；其价值已被 `botasaurus-requests` 延续。除非能接受停更风险，否则不作为新依赖。

## 三、映射到角度3 的"三层 + 两翼"检测模型

| 检测层 | 角度3 结论 | 本次逐库尽调补充（2026-07-01） |
| --- | --- | --- |
| **L1 TLS/HTTP2/3 指纹** | curl_cffi / tls-client | curl_cffi ⭐5,910、MIT、**新增 HTTP/3 + 远程指纹库**，仍是第一选择；tls-client 的 Python 侧要认准活跃 fork（原版已弃）；hrequests 停更 |

## 四、相较角度3 的"更新点"（是否有新增）

1. **playwright-stealth 明确降级为"不用"**：角度3 已提示它够不到 L3，本次进一步核实其 2026 版作者自述仅 POC，正式剔除；替代为 patchright / pw-stealth-enhanced。
2. **curl_cffi 能力升级**：新增 **HTTP/3 指纹**与**运行时可更新指纹库**，进一步降低"随 Chrome 升级失效"的维护成本——角度3 结论"L1 首选 curl_cffi"更稳。
3. **DrissionPage 许可证变更（重要）**：角度3 未收录；本次发现其 **v4 起转非商用/需授权**，属新增法务风险点，需单独标注。
4. **新增 4 个角度3 未逐一评估的库**：DrissionPage、botasaurus、SeleniumBase、hrequests——其中 **SeleniumBase CDP Mode + solve_captcha** 与 **botasaurus 一体化框架**对"一键批量"工程目标有直接价值，是本次增量的主要贡献。
5. **tls-client 生态提示**：Python 原版停更、需转活跃 fork——角度3 只提了 tls-client 概念，本次补充了"选哪一个绑定"的可执行结论。

## 五、对本组"自建 Scholar 爬虫"的最优路径建议

```
① 抓 Scholar 结果页（元数据）
   curl_cffi(impersonate=chrome/最新) + 住宅代理 + 限速(如每5–15次换IP、45–90s抖动) + 指数退避
   └─ 能静态解析就到此为止（最省、无浏览器）
② 被拦升级（跳验证页 / 加载后被拦 / 指纹对了仍拦）
   nodriver（直连CDP，L3）；如已在 Playwright 生态则用 patchright；要交互调试/录制用 SeleniumBase CDP Mode
③ 仍弹 reCAPTCHA/Turnstile（翼B）
   CapSolver / CapMonster 打码兜底（见角度3 第五节）+ 行为模拟 + 养IP/会话
④ 拿到可及 PDF / 落地页
   复用本仓 fulltext_fetcher（landing.py 出版商专属 selector 已增强）；出版商 OA 页多数无需反爬
⑤ 工程外壳（一键批量 / 文件名标准化 / 缓存 / 并行）
   借鉴 botasaurus 的 @request/@browser/@task + 缓存 + 并行 设计（或直接用其框架）
```

- **依赖与许可证组合建议**：核心用 `curl_cffi(MIT)`；浏览器侧若接受 AGPL 内部使用则 `nodriver`，若要 MIT 更宽松则 `SeleniumBase(MIT)`；**避免把 DrissionPage 引入任何商用产物**（个人研究可用）。
- **别做的事**：不要无脑堆栈；不要把 UC / playwright-stealth 当唯一手段；不要只换 IP 不改指纹（代理≠万能，见角度3 翼A）。

### 与其他成员的接口衔接（协同提示）
- 本文仅为**选型结论**，不改任何代码/接口；若后续要落地"自建抓取器"，其**产出的元数据结构 / PDF 候选 URL** 需与本仓 `fulltext_fetcher`（`extract_pdf_links(html, base_url)->List[str]` 等）对齐——届时应由相关成员与 download.py/pipeline 负责人对齐命名，再由总指挥编排。
