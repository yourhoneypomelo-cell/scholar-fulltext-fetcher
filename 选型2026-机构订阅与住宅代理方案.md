## 二、instsci 架构细节

### 2.1 基本信息

| 项目 | 值 |
|------|------|
| 仓库 | [Rimagination/instsci](https://github.com/Rimagination/instsci) |
| Stars | 96 |
| 许可证 | MIT |
| 创建 | 2026-05-02 |
| 最后更新 | 2026-06-23 |
| 关系 | scansci-pdf 的底层组件（同一作者 Rimagination）|

### 2.2 核心架构

**OA 优先 + 机构兜底** 分层设计：

```
输入 DOI → OA 检索（Unpaywall/arXiv/S2/出版社元数据）
              ↓ 失败
         → 机构访问（Shibboleth/OpenAthens/CARSI/WebVPN/EZproxy）
              ↓ 需要认证
         → CloakBrowser 可见窗口（用户手动完成 SSO/2FA/CAPTCHA）
              ↓ 认证成功
         → 持久化 Cookie/localStorage/IndexedDB → 后续批量复用
```

### 2.3 CloakBrowser 集成机制

- **二进制管理**：缓存在 `instsci/_browsers/cloakbrowser`，自动下载，不入 Git
- **会话持久化**：使用持久化 Profile 目录，保留 cookies/localStorage/IndexedDB/cache
- **长存 Broker**：出版商会话代理常驻，减少重复认证
- **配置记忆**：记住最近配置的机构名称（中英文别名）

### 2.4 安全设计原则

- **绝不存储密码**：SSO/2FA/CAPTCHA 全由用户在可见浏览器中完成
- **运行时状态本地化**：config.json/cookies.json/chrome-profile/carsi_cookies/broker queues 均本地，被 .gitignore 排除
- **HTTP 诊断仅为预检查**：最终 PDF 可用性以可见浏览器工作流结果为准

### 2.5 出版商专属工作流

覆盖 11 家主要出版商：ACM、ACS、AIP、IEEE、IOP、Nature、Oxford、RSC、ScienceDirect、Springer、Wiley。每家有定制的认证路径和 PDF 提取逻辑。

**Elsevier 特殊处理**：
- 支持 Elsevier API Key + Institutional Token 配置
- 优先走 `view=FULL XML → object/eid → PDF` 路径
- 先尝试 direct route（让 api.elsevier.com 走校园网/VPN/规则VPN/图书馆出口）

## 三、CARSI/WebVPN 集成方案

### 3.1 中国高校机构认证路径总览

| 认证方式 | 适用场景 | 代表工具 |
|----------|----------|----------|
| **WebVPN** | 校外访问，通过高校 WebVPN 代理 | scansci-pdf (`vpnsci_*`), instsci |
| **CARSI** | CERNET 联邦认证，出版商页面"Access through institution" | scansci-pdf (`carsi_*`), instsci |
| **EZproxy** | 图书馆代理服务器（hostname 重写模式）| scansci-pdf (`ezproxy_*`), instsci |
| **Shibboleth/OpenAthens** | 国际标准联邦认证 | instsci, claude-research-gateway |
| **CAS** | 高校统一认证（如北理工）| library-access-mcp |

### 3.2 具体开源实现方案

#### 方案 A：scansci-pdf（推荐，最完整）

```bash
pip install "scansci-pdf[cloakbrowser,instsci]"
scansci-pdf setup "北京大学"          # 配置学校
scansci-pdf vpnsci-login              # WebVPN 认证（浏览器窗口）
scansci-pdf get "10.1038/s41586-020-2649-2"  # 下载
scansci-pdf papers dois.txt --output ./papers  # 批量
```

- 100+ 中国高校 WebVPN 支持
- CARSI 覆盖 6+ 主要出版商
- Phase 1 / Phase 2 两阶段回退

#### 方案 B：library-access-mcp（浏览器插件模式）

```bash
# 安装浏览器插件 + MCP 服务器
cd mcp-server && pip install -r requirements.txt
python server.py
```

- 复用已登录浏览器 Session
- 已验证：北京理工(CAS)、北航(CARSI/Shibboleth)
- WebSocket 通信 + 心跳保活
- Session 过期自动重新认证

#### 方案 C：claude-research-gateway（MCP 专用）

```json
{
  "institution": {
    "name": "MIT",
    "ezproxy_url": "https://ezproxy.library.mit.edu/login",
    "wayf_institution_name": "Massachusetts Institute of Technology"
  }
}
```

- Shibboleth/SAML SSO 认证
- Cookie 加密存储在 `~/.claude-research/sessions/`
- 适合国际高校

### 3.3 与本项目的整合路径

**推荐方案**：参考 instsci 的 CloakBrowser + Cookie 持久化架构，在 fulltext_fetcher 中新增机构订阅源连接器。

实现要点：
1. 新增 `sources/institutional.py`，排在现有源之后作为 Phase 2 兜底
2. 使用 CloakBrowser（`pip install cloakbrowser`）作为可见浏览器引擎
3. 支持三种认证：WebVPN（中国高校）/ CARSI / EZproxy
4. Cookie 持久化到本地 Profile 目录，批量不重复登录
5. 绝不存储密码，认证全在可见浏览器中完成
6. 默认关闭，通过配置显式启用

- 每次启动自动生成随机指纹种子
- Chromium 146 内核

### 4.3 使用方式

Drop-in Playwright 替换：

```python
# 原代码（Playwright）
from playwright.sync_api import sync_playwright
with sync_playwright() as p:

# 替换为 CloakBrowser（仅改 import）
from cloakbrowser.playwright import sync_playwright
with sync_playwright() as p:
    browser = p.chromium.launch()
```

## 五、住宅代理最佳实践与免费试用导航

### 5.1 Google Scholar 专用配置要点（2026 核实）

1. **必须住宅代理**：数据中心 IP 立即被封，这是角度 6 的核心结论
2. **Sticky Session 1-5 分钟**：不要每请求换 IP，模拟真实用户浏览
3. **TLS 指纹匹配**：curl_cffi impersonate Chrome 131，配合代理地区的 Accept-Language
4. **随机延迟 60+ 秒**：Scholar 限速极其激进（难度 4/5）
5. **配套请求头**：sec-ch-ua / sec-fetch-* / Referer 完整模拟
6. **429 指数退避**：429 响应后等待（不换 IP），IP 本身没问题只是太快
7. **低并发**：建议单线程或 <=2 并发

### 5.4 配置示例

```python

response = session.get(
    "https://scholar.google.com/scholar?q=your_query",
    proxies=proxies,
```
