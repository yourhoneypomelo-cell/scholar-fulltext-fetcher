## 三、每条「明确终态」精确枚举（承总指挥 157 校准）

**每条输入恰好落入一个稳定终态，无第三种「悬空」态：**

- **success**：`metadata.success` 且 PDF 落盘（build_coverage 口径）。
- **still_missing（miss）**，reason 桶**全枚举**：`no-source`(no-candidates) ｜ `cf-403`(cloudflare-challenge) ｜ `paywall`(403/401) ｜ `download-fail`(landing-no-embedded-pdf 等) ｜ `http-404` ｜ **`timeout`(straggler-timeout)** ｜ …——**均入 `still_missing.txt` + `coverage.miss` + `by_reason`**。

**timeout 是稳定终态、非悬空（代码级核实 + 已测试锁定）**：
- 机制（`fulltext_fetcher/pipeline.py`）：`run()` 内**尾部卡死看门狗** `_straggler_timeout()`（无进展阈值默认 `max(180, timeout*8)=240s`；CF/浏览器兜底启用时 **≥900s**，防误杀慢但合法的 CF 尾部）。阈值内零完成 → `_record_straggler()` 把每条在途输入**强制判为** `FetchResult(error="straggler-timeout", success=False)`、`emit result`、写 metadata/csv，再 `break` 收尾——即「**有限等待后判终态**」，绝不因单条 straggler 让整批悬挂（P1 无人值守硬保证）。
- 证据：监管者 run 的 `zzq…nonsense title` 条 → status=MISS、reason=`timeout`、error=`straggler-timeout`、进 `still_missing.txt`(L13) + `coverage.miss`(11) + `by_reason`(timeout=1) + `run_all_detail.tsv`(L12)。
- 回归锁定：`python -m fulltext_fetcher.pipeline`（PIPELINE_OK）——造永不完成的 `HANG` 输入，断言 run() 看门狗后停等、记 `straggler-timeout` 失败、计入 summary(processed2/success1/miss1)、四产物（summary/metadata/csv/report）必落盘、且该 error 在**续跑重试白名单**（`--resume` 自动重试）。
- 结论：**稳定失败桶 + 掉线可续，零悬空 → 无需改 run_all**。
