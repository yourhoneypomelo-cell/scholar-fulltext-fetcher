## 根因（三层，按重要性）

### ① 主因 · 环境阻塞：Cloudflare 质询未通过
分步诊断（`_route_b_diag_152.py`，对真·金OA样本 `10.1021/acsomega.6c04195`，Unpaywall `is_oa=True`）：

```
final_url = https://pubs.acs.org/doi/10.1021/acsomega.6c04195
title     = Just a moment...
body      = pubs.acs.org / Performing security verification / This website uses a security
            service to protect against malicious bots ... verifies you are not a bot /
            Ray ID: ... / Performance and Security by Cloudflare
extract_pdf_links -> []   （仍停在 CF 质询页，无 PDF 链接）
```
- **浏览器停在 CF「Performing security verification」质询页**，nodriver 0.50.3 + Chrome 133 有头在 45–120s 内、连续 **4 次尝试**均未过盾。
- 因此文章页从未加载 → 找不到 PDF 直链 → `no-pdf-captured`。
- 可能诱因：Chrome/ nodriver 版本错配（见②）；以及**有头窗口是否真正可见/可交互**（若 agent shell 非交互桌面会话，CF 反自动化判定必失败——需在用户本人交互式 PowerShell 复跑确认）。

### ② 次因 · CDP 字段版本错配（依赖）
- Chrome 133 发**旧** CDP 字段 `privateNetworkRequestPolicy`；nodriver 0.50.3 只认**新**名 `localNetworkAccessRequestPolicy`。
- `nodriver/cdp/network.py:2324` `ClientSecurityState.from_json` 对每条 `requestWillBeSentExtraInfo` 事件抛 `KeyError: 'localNetworkAccessRequestPolicy'`（单次 CLI 跑刷出 ~2800 行 traceback），网络事件被 `process_event` 丢弃。
- 运行时 monkeypatch 令 `from_json` 容旧/容缺后（`_route_b_probe_152.py`），KeyError 洪水消失，但 **CF 仍未过、仍 no-pdf** → 故②是次因，非决定因素。

### ③ 代码缺口（bug，按边界"先报不改"）：`_BLOCK_SIGNALS` 漏配当前 CF 文案
- `render_fetch.py` `_BLOCK_SIGNALS` = `just a moment / verify you are human / enable javascript and cookies / attention required / checking your browser / checking if the site connection`。
- 实测 CF 质询 body 文案为 **"performing security verification" / "verifies you are not a bot" / "needs to review the security of your connection"**，**6 条现有信号无一命中**（且代码只查 `body.innerText`，未查 `document.title`——而 title 恰为 "just a moment..."）。
- 后果：CF 质询被误判成 `no-pdf-captured` 而非 `blocked:challenge-page`，SUMMARY 里 `blocked=0` **掩盖了真实的被墙状态**，误导验收。

## 附带发现：冒烟样本标注错误
- `10.1021/acsanm.1c00959` 被 `_smoke_render_bytes.py` 标为 "ACS-OA happy path"，但 **Unpaywall：`is_oa=False`、`oa_status=closed`、0 个 OA 位置** → 实为订阅闭刊，**无法验证 happy path**（即便过了 CF 也无免费 PDF）。happy-path 应换成核实过的金 OA 样本（如 ACS Omega `10.1021/acsomega.6c04195`）。

## 建议修复（按优先级，供 -140 排期/派发）
1. **对齐 Chrome↔nodriver 版本**：升级 Chrome 至约 140+（匹配 nodriver 0.50.3 的 CDP 架构），或将 nodriver 钉到与 Chrome 133 CDP 匹配的版本。消除 KeyError 洪水。
2. **在真·可见交互桌面复跑**：确认有头 Chrome 窗口确实弹出并可交互（若当前是非交互会话，CF 必败）。
3. **修 `_BLOCK_SIGNALS`**（report-first，未改仓）：补 "performing security verification" / "verifies you are not a bot" / "needs to review the security of your connection"，并增查 `document.title`。让 CF 质询正确落 `blocked:challenge-page`。
4. **修冒烟样本**：happy-path 换成核实的金 OA ACS DOI（`10.1021/acsanm.1c00959` 已闭刊）。
5. 上述①–④处理后再复跑验收；届时方能判定"机制端到端是否通"。当前证据只能证：**代码路径可跑、但被 CF 挡在门外**。
