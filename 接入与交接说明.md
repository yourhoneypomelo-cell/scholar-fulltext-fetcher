# fulltext_fetcher 接入 / 交接说明(作为子程序嵌入)

> 本程序设计为可被更大任务**当作子模块调用**。本文件说明它的**入口(接入)**、**出口(导出)**、I/O 契约与交接方式。
> 版本 1.0 · 2026-07-01 · 谷歌学术人机认证-141

---

## 1. 两种接入方式

### 方式 A:Python 同进程导入(推荐,拿结构化对象)
```python
from fulltext_fetcher import fetch_one, fetch_many, FullTextFetcher, Config

# 单条
r = fetch_one("10.7717/peerj.4375", email="you@uni.edu", out_dir="out")
print(r.success, r.pdf_path, r.source_used, r.pdf_url)

# 批量(复用一个抓取器,效率最高)
with FullTextFetcher(email="you@uni.edu", out_dir="out", concurrency=8) as ft:
    results, summary = ft.fetch_many(["10.x/y", "Attention is all you need", "1706.03762"])
    for r in results:
        if r.success:
            ingest(r.pdf_path, r.doi)   # 交给你的下游
```

### 方式 B:子进程 CLI(跨语言,拿 JSON)
```bash
python -m fulltext_fetcher "10.7717/peerj.4375" --email you@uni.edu --print-json
```
stdout 输出一行 JSON:`{"summary": {...}, "results": [ <FetchResult>, ... ]}`;
**日志与人类提示走 stderr / 文件,不污染 stdout**,父进程直接 `json.loads(stdout)` 即可。
退出码:`0`=至少一条成功(或无输入);`1`=全部失败;`2`=用法错误。

## 2. 入口(接入点)汇总

| 入口 | 签名 / 形式 | 用途 |
|---|---|---|
| `fetch_one(item, config=None, **overrides) -> FetchResult` | 函数 | 单条,始终返回结果(不受续跑跳过影响) |
| `fetch_many(inputs, config=None, **overrides) -> (list[FetchResult], summary)` | 函数 | 一次性批量 |
| `FullTextFetcher(config=None, **overrides)` | 类(支持 `with`) | 复用实例批量处理;`.fetch_one()/.fetch_many()/.close()` |
| `Config(...)` | dataclass | 配置注入(见 §4) |
| `python -m fulltext_fetcher ... --print-json` | 子进程 | 跨语言接入 |

`**overrides` 即 `Config` 的字段名(如 `email=..., concurrency=..., oa_only=True`),非法字段会抛 `TypeError`。

## 3. 出口(导出)契约

### 3.1 内存返回:`FetchResult`(可 `.to_dict()`)
```jsonc
{
  "raw_input": "10.7717/peerj.4375",  // 原始输入
  "kind": "doi",                       // doi | title | arxiv | null
  "doi": "10.7717/peerj.4375",
  "title": "The state of OA: ...",
  "success": true,
  "pdf_path": "out/pdfs/10.7717_peerj.4375.pdf", // 成功时的落盘路径;失败为 null
  "pdf_bytes": 2432111,
  "source_used": "europe_pmc",         // 命中的源
  "pdf_url": "https://europepmc.org/articles/PMC.../?pdf=render",
  "candidates": 25,                     // 收集到的候选总数
  "attempts": [ {"source":"unpaywall","ok":true,"n_candidates":8,"elapsed_ms":932,"error":null}, ... ],
  "elapsed_ms": 7092,
  "error": null                         // 失败原因(如 no-downloadable-pdf / unresolvable-input)
}
```

### 3.2 文件产物(落在 `out_dir/`,供审计与断点续跑)
| 文件 | 每行/结构 | 字段 |
|---|---|---|
| `pdfs/*.pdf` | 二进制 | 按 DOI/arXiv 命名 |
| `metadata.jsonl` | 每条输入一行 = 一个 FetchResult | 见 §3.1 |
| `attempts.jsonl` | 每事件一行 | `{ts,event,...}`;event ∈ `input/resolved/source/download/located/result/resolve_error` |
| `summary.json` | 单对象 | `total_inputs,skipped_resume,processed,success,miss,success_rate,by_source{},elapsed_sec,no_download,oa_only` |
| `run.log` | 人类日志 | UTF-8 |

> `metadata.jsonl` 既是结果总账,也是**断点续跑依据**(下次对同 `out_dir` 运行会跳过 `success=true` 的输入,除非 `--no-resume` / `resume=False`)。

## 4. 配置注入(Config 全字段)

| 字段 | 默认 | 含义 |
|---|---|---|
| `email` | `anonymous@example.com` | 联系邮箱(Unpaywall 必需真实) |
| `openalex_key`/`s2_key`/`core_key` | None | 各源 key;`core_key` 为空则跳过 CORE |
| `out_dir` | `out` | 产物目录 |
| `concurrency` | 4 | 并发输入数 |
| `timeout` | 30 | 单请求超时(秒) |
| `max_retries` | 3 | 429/5xx 重试次数 |
| `per_host_interval` | 0.34 | 每域最小请求间隔(秒) |
| `min_pdf_bytes`/`max_pdf_bytes` | 1024 / 80MB | PDF 大小下/上限 |
| `oa_only` | False | 仅直链,不兜底落地页 |
| `no_download` | False | 只定位不下载 |
| `resume` | True | 断点续跑 |
| `enable_scihub` | False | ⚠️ Sci-Hub 兜底(合规风险) |
| `sources` | 全部 | 源及顺序(list[str]) |
| `log_level` | INFO | 日志级别 |

## 5. 交接清单(交给下一棒/父程序时)
1. **代码**:整个 `fulltext_fetcher/` 包(自包含,唯一三方依赖 `requests`)。
2. **依赖**:`pip install -r fulltext_fetcher/requirements.txt`。
3. **入口**:本文件 §1–§2。
4. **产物契约**:本文件 §3(父程序据此消费)。
5. **配置**:§4 注入(邮箱、key、并发、源选择)。
6. **行为保证**:单源失败不影响整体(连接器内部吞异常返回空);坏 host 自动熔断;PDF 经 `%PDF` 校验,绝不把 HTML 落地页当 PDF 落盘;`fetch_*` 不抛业务异常(失败体现在 `FetchResult.success/error`)。
7. **背景资料**:`实现路径与原理.md`、`经验记录-踩坑与发现.md`、`fulltext_fetcher资料-各源接口速查.md`。

## 6. 嵌入范式(作为复杂流水线的一环)
```python
# 上游:产出一批 DOI/标题  → fulltext_fetcher → 下游:入库/解析全文/喂给 LLM
from fulltext_fetcher import FullTextFetcher

def stage_fetch(identifiers, email, out_dir):
    with FullTextFetcher(email=email, out_dir=out_dir, concurrency=8, oa_only=False) as ft:
        results, summary = ft.fetch_many(identifiers)
    got = [r for r in results if r.success]
    missed = [r.raw_input for r in results if not r.success]  # 交回上游重排/换策略
    return got, missed, summary
```
- **幂等/续跑**:同 `out_dir` 重跑只补失败项;父程序据 `missed` 决定是否升级策略(加 key、开 scihub、转商业 API)。
- **可观测**:父程序读 `summary.json` 的 `success_rate`/`by_source` 做门控;读 `attempts.jsonl` 排错。
- **版本**:`fulltext_fetcher.__version__`(当前 `1.0.0`);I/O 契约变更将升版本并在此注明。
