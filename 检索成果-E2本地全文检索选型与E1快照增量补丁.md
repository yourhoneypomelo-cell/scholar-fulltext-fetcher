## 一、E2 本地全文检索：选型与设计

### 1.1 需求与定位
- 路线图 E2（见《功能优化与新增功能建议.md》E2、《角度8》§三）：下载后建本地全文索引，「下完即可全文搜」，闭环「检索→下载→再检索」。**当前未建**；角度8 判 ROI 中、非急、合规 ✅。
- 手上数据：`out/pdfs/*.pdf`（正文）+ `out/metadata.jsonl`（题录：doi/title/authors/year/source/pdf_path）。目标：对这些做**本地可搜**，零联网。

### 1.2 分层选型（2026-07-01 核验）

| 层 | 方案 | 许可 | 2026 现状 | 依赖 | 结论 |
|---|---|---|---|---|---|
| **核心（必选）** | **SQLite FTS5** | Public Domain（SQLite） | 稳定成熟；BM25 内置、列权重、phrase/NEAR/boolean/prefix、`snippet()`/`highlight()`；分词 `unicode61`（默认，折叠大小写/去音）、`porter`（英文词干）、`trigram`（子串/CJK） | **零**（`sqlite3` stdlib 内置，多数构建含 FTS5） | **采用为核心**——数百万行内的本地/CLI 场景「够用且零装」 |
| 可选语义 | **sqlite-vec**（asg017） | Apache-2.0 | ~7.8k★；2026-03 `v0.1.7`「回归」、2026-05 `v0.1.10-alpha`；纯 C 无依赖、可加载扩展、Linux/macOS/Windows/WASM 皆可；brute-force + 新 **rescore ANN**；与 FTS5 **同一 SQLite 文件**即可 KNN | 可选扩展 | **采用为可选语义层**（与 FTS5 同库最贴合；pre-v1 有破坏性变更风险，故默认关） |
| 可选嵌入 | **model2vec / Potion 静态嵌入** | Apache-2.0 | 纯 numpy CPU、80–500× 快于 MiniLM、~85–98% 质量；`potion-base-8M`≈8 MB、`potion-mxbai-256d-v2` 7.5 MB（MTEB 71.45）、`potion-mxbai-micro` 0.7 MB | 可选 | **本仓无 GPU → 静态嵌入最合适**（秒级加载、CPU 高吞吐） |
| 备选（不建议） | DuckDB VSS / LanceDB | MIT / Apache-2.0 | DuckDB VSS：HNSW，实验、默认仅内存持久化；LanceDB：列式向量库、依赖重 | 重 | **不采用**——分析/百万级 ANN 才需要，违背本仓零依赖哲学 |

- **混合检索（可选进阶）**：FTS5（BM25 词法）+ sqlite-vec（向量语义）各取 topN，用 **RRF（Reciprocal Rank Fusion）** 融合——2026 本地混合检索通行做法，召回质量最好；默认可只用 FTS5。

### 1.3 面向本仓的设计（契合现有哲学；实现交实现者）
建议新模块 `fulltext_fetcher/localsearch.py`（本文只给设计草案）：

- **schema**：
  - `docs`（canonical）：`doi TEXT PRIMARY KEY, title, authors, year, source, pdf_path, text`
  - `docs_fts`：`CREATE VIRTUAL TABLE docs_fts USING fts5(title, authors, text, content='docs', content_rowid='rowid', tokenize='porter unicode61')` + 三触发器（ai/ad/au，delete 先于 insert）
  - 列权重：查询用 `bm25(docs_fts, 10.0, 2.0, 1.0)`（题>作者>正文）
- **建索引**：遍历 `out/metadata.jsonl` 的成功项 → 抽 PDF 文本（可选 `pypdf`（已在 D2 用）/ `pymupdf4llm`；**缺库→仅索引题录，降级不报错**）→ upsert `docs` → 触发器自动同步 FTS。
- **查询 CLI**：`python -m fulltext_fetcher.localsearch "query"` → BM25 排序 + `snippet()` 高亮 + 回链 `pdf_path`；`--semantic` 开可选向量（sqlite-vec + potion，缺依赖则友好提示）。
- **纪律**：零依赖核心、可选语义、缺库降级、离线 selftest（`LOCALSEARCH_OK`）——与 `snapshot.py`/`download.py`/`render_fetch.py` 同哲学。
- **契约对齐**：复用 `models` 字段；若需 config（如 `localsearch_db` 路径、`semantic` 开关）由 **config 负责人**加（`getattr` 兜底、默认关）——与 -145/config 负责人对齐。

### 1.4 ROI / 边界
- **价值**：闭环增值（离线可搜已下语料），合规 ✅，**零核心依赖**；**非急**（角度8）。
- **不做**：不引 ES/DuckDB/LanceDB 重栈（违背零依赖）；语义层默认关（可选）。

## 二、E1 快照增量：`snapshot_bootstrap.GUIDE_TEXT` 可交付补丁说明

> 《角度8》附B 已判 E1 **已建**（`snapshot_bootstrap.py` + `snapshot.py` upsert + `ingest.py`），仅余 guide 文本增强。以下为**可直接粘贴**的补丁点（交实现者；本文不改代码）。均 2026-07-01 核验。

### 2.1 三处增强（附核验事实）

**① Parquet 进免费公开季度快照（2026-06 起）**
- 事实：OpenAlex 自 **2026-06 季度版**起，Parquet（snappy）随**免费公开**快照发布，路径 `s3://openalex/data/parquet/`（JSONL 仍在 `/data/jsonl/`）；两格式各是全量副本，同时拉 >660 GB，故按需取单一格式。列式，分析/DuckDB 更快。
- 补丁文本（并入 GUIDE）：
  ```
  # Parquet（列式，2026-06 起随免费公开季度快照；分析更快）：
  aws s3 sync s3://openalex/data/parquet/works openalex-parquet/works --no-sign-request
  ```

**② DuckDB httpfs 直查 S3（免下 1.6 TB）**
- 事实：DuckDB `httpfs` 可直接 `read_parquet` 远端 S3，按需拉取，免全量本地（公共桶只读走 HTTP Range）。
- 补丁文本：
  ```
  # 免下载直查（DuckDB httpfs，适合不想本地存全量者）：
  INSTALL httpfs; LOAD httpfs;
  SELECT id, doi, title FROM read_parquet('s3://openalex/data/parquet/works/*.parquet') LIMIT 5;
  # 若报 HTTP HEAD 连接错，显式设 region/endpoint 或 CREATE SECRET(TYPE s3, PROVIDER credential_chain)
  ```
- OSS 参考：`pyalexs3`（S3→DuckDB 免下载）、`chrisgebert/open_alex_snapshot`（dbt-duckdb）。

**③ Changefiles 增量端点**
- 事实：OpenAlex `/changefiles` 与 `/changefiles/{date}`**浏览「哪些日期/文件变了」免费、无 key、无限**；但**下载实际数据文件**（`content.openalex.org/...` URL）需**付费计划 API key**（JSONL + Parquet 均有）。Unpaywall Data Feed changefiles：`GET api.unpaywall.org/feed/changefiles?api_key=&interval=day|week`（付费）。
- 用法：下载变更文件后走**现有 `--incremental` / `--since`** 持续同步。
- 补丁文本：
  ```
  # 增量：OpenAlex /changefiles（浏览免费无 key；下载数据文件需付费 key）
  #   或 Unpaywall Data Feed changefiles（付费）→ 落地后：
  python -m fulltext_fetcher.snapshot_bootstrap ingest --incremental --since 2026-06-01
  ```

### 2.2 落地方式
- **纯文本增强** `GUIDE_TEXT` 常量，零风险、非阻塞；E1 主体（`snapshot_bootstrap`/`snapshot`/`ingest`）**无需再动**。实现者只需把上述三段并入 `GUIDE_TEXT`。
