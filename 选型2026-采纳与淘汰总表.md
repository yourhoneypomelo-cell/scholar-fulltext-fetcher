## 四、技术口径修订（刷新母表，⑤ 定稿据此）

> 沿用母表五档，只登记**相对母表(-154, 07-01)的 delta**；模块级完整决策仍以母表为准。

1. **CF 绕过口径（重大修订，承 N5/N7/179）**：
   - 母表原列「byparr（Docker）作 CF 兜底 P2」。**新口径**：通用 CF 能力**首选=仓内 `tools/flaresolverr_nodriver.py`（免 Docker、nodriver、已实测）或点亮 `render_fetch.py` 内置 nodriver（`pip install nodriver`）**——零安装、免 uc3.5.5/Chrome 死结；**byparr（Camoufox）降为「nodriver headed 仍被特定 Turnstile 拦」时的升级项**；**FlareSolverr(uc3.5.5) 淘汰**（仅老站登记）；住宅代理为最后兜底（cf_clearance 绑 IP）。
   - ⚠️ **PowerShell 起服注意**：`FLARESOLVERR_URL` 必须用 `$env:FLARESOLVERR_URL="..."`（N7 文档里的 `set VAR=` 是 cmd 语法、PowerShell 不生效、且不报错——`_flaresolverr_enabled` 静默为 False）。
   - 🔬 **强 CF 的真正瓶颈在「回放下载」而非「求解器」（-145 07-02 端到端实测，重要）**：nodriver/byparr 都能解出 CF 拿到 `cf_clearance`，但 `download.py` 把 cookie 交给 `curl_cffi`/`requests` **回放**下载时，**RSC 把 `cf_clearance` 绑定到 JA3 指纹** → 回放仍 403（`flaresolverr_failed`）；换更强求解器（byparr）走**同一回放链**，救不了 RSC/ScienceDirect 这类 JA3 绑定站。**ACS 类可回放**（`acscatal` 已 `flaresolverr_recovered`）。⇒ **口径再修订**：仓内 shim（Top1）对**可回放型 CF（ACS 等）依然有效**（已解 145 活阻塞）；但对 **JA3 绑定型强 CF（RSC/ScienceDirect），真正解法是「浏览器内直接下 PDF」——solve 与 download 用同一浏览器会话/同一指纹出口**，而非换求解器。这与 N5(176)『别为 RSC 单独上 FlareSolverr』殊途同归、但定位更深（N5 归因于 websearch 已兜底，-145 归因于 JA3 回放失效）。**建议新增实现项：`render_fetch.py` 浏览器内直下 PDF 路径**（承接 N5 的 nodriver 点亮，覆盖 JA3 绑定型强 CF），列 P2。
2. **D3 限速/熔断口径（承 N6）**：路线图「未建」应改为「**已建 ~60%**」（http_client 已有按域限速+表头自适应收紧+连续失败熔断+Retry-After 退避）；真实空白=Jitter/AIMD/half-open 三增量 + 异步侧(aio.py/C4)完全无节流。
3. **A3 出版商适配口径（承预备包 §C）**：`publisher_direct.py`(144) 已实现 **20 个 DOI 前缀直构**（非「待实现」）；「待实现」仅指 keyed-TDM 与 +8 前缀增强（N2）。
4. **新增行（母表未列）**：D3 限速/熔断（N6）、单条内竞速+EMA+可疑 PDF（N1）、机构订阅 A5（N3/N4）、`publisher_access_catalog.json`（N2）。
