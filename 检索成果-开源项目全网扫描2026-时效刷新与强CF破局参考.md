## 二、重点破局：强 CF「浏览器内直接下 PDF」——从「设计」到「照抄现成 OSS」（承 ⑤ P2 + -145 JA3 发现）

**背景（本仓已确诊）**：RSC/ScienceDirect 把 `cf_clearance` **绑 JA3/TLS 指纹** → nodriver/byparr 解出 cookie 后，`download.py` 交 `curl_cffi` **回放**下载仍 403（换更强求解器无效）。⑤ 已列 P2「`render_fetch.py` 浏览器内直下 PDF」，但**尚是设计**。本轮找到 **2026 现成 OSS 把这条链跑通**，可直接照抄：

| OSS 参考 | 关键技术（正是本仓缺的那一环） |
|---|---|
| **GAO-pooh/paper-scraper** | ScienceDirect 走 **CDP（Chrome DevTools Protocol）** 直接**拦截 PDF 响应字节**（`Network`/`Fetch` 事件），**不经 Playwright、不把 URL 交外部 HTTP 客户端** → 天然绕开「JA3 回放 403」；另用 `curl_cffi` 做 TLS 伪装、支持机构 SSO/CARSI |
| **Given-Dream/sciencedirect-live-session-fetcher** | 复用**已授权的活浏览器会话**（Edge/Chrome DevTools 远程调试口），先开**文章页**建立 cookie 上下文，再取**短时签名 PDF 直链**并**在授权页上下文内**取回 |
| **go-rod/rod #953（配方）** | ①`always_open_pdf_externally=true`（阻止 PDF viewer 吞流）②`headless=false`+点 CF iframe 复选框 ③`WaitDownload()` 捕获字节落盘 |
| **CloakBrowser #365（避坑）** | **别直接 goto PDF 的 `md5=` 短链**（短时+绑会话，冷取必 403）；先开文章页；住宅代理+`geoip` 降低质询难度 |

**给实现者的落地要点（映射到本仓 `render_fetch.py` 扩展点）**：
1. 本仓 `render_fetch.render_get_pdf_url` 现在只**渲染取 URL** → 再交外部下载（这正是 JA3 回放失败点）。**新增一条「在浏览器会话内直接取 PDF 字节」路径**（CDP `Fetch.enable`+`Fetch.requestPaused` 或 `Page.printToPDF`/下载事件），**solve 与 download 用同一浏览器/同一指纹出口**。
2. 触发条件：`download.py` 检出 `cloudflare-challenge` 且属 JA3 绑定站（RSC/ScienceDirect 前缀）时，才走此重路径（低频兜底，按 origin 缓存会话）。
3. 边界：真机浏览器活、AGPL(nodriver) 内部回收无碍、住宅代理为最后兜底；**属代码活，出骨架后交实现者**（本岗只出参考，不改生产码）。

> 这一项是 still_missing 两大主体之一（另一是纯订阅付费墙→A5 机构订阅）的**唯一免费突破口**，ROI 取决于 JA3 绑定站在 still_missing 里的占比（建议用 coverage 数据量化后再拍，见 §四）。
