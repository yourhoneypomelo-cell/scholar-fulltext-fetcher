# 谷歌学术爬虫（fulltext_fetcher/scholar/）使用说明

> 输入**标题 / DOI** → 抓 Google Scholar 元数据（被引/版本/作者/年）+ Scholar 可及 **PDF 直链** → 下载为**文件名标准化**的原文 → 全程程序化、一键批量、结构化日志。
> 版本 1.0 · 2026-07-01 · 配套：`谷歌学术爬虫-架构与选型.md`（架构与接口契约）、`经验记录-踩坑与发现.md` F 节（踩坑）。

---

## 一、30 秒上手

```bash
# 单条（标题或 DOI 均可）
python -m fulltext_fetcher.scholar "Attention is all you need"
python -m fulltext_fetcher.scholar "10.1038/nature14539"

# 批量：-f 读文件（.txt 逐行 / .csv / .xlsx 自动识别 doi,title 列，复用父包读取器）
python -m fulltext_fetcher.scholar -f titles.txt --num 5

# 不联网端到端自检（验证程序完好）
python -m fulltext_fetcher.scholar --selftest      # → SCHOLAR_E2E_OK
```

产物默认落 `out_scholar/`。**依赖**：Python 3.8+，仅需 `requests`（父包已用）；其余（curl_cffi / nodriver / openpyxl / 2captcha-python 等）全为**可选**，缺失自动优雅降级。

## 二、三种取回模式（`--mode`，默认 auto）

| 模式 | 行为 | 何时用 |
|---|---|---|
| `auto`（默认） | 有 SerpApi key 走 `serpapi`，否则走 `self` | 通常保持默认 |
| `serpapi` | 走 **SerpApi**（商业合规、有法律盾、无需自己过验证码） | **推荐用于真实抓取**：稳、快、合规 |
| `self` | 自建反爬栈：`curl_cffi`(L1) → `nodriver`(L3)，翼A 代理、翼B 打码 | 无 SerpApi key 且愿自担风险时 |

### ⚠ 关于「拿到真实结果」的现实（重要）
- **数据中心 / 机房 IP 直连 Scholar 几乎必被 HTTP 403 拦截**（实测：即便 curl_cffi 伪装 Chrome TLS 指纹也即刻 403）。这是 **IP 信誉**边界，不是程序缺陷。
- 要真实抓到结果，二选一：
  1. **SerpApi key**（最省心）：`--serpapi-key <KEY>` 或环境变量 `SERPAPI_KEY`；
  2. **住宅代理**：`--proxy` 开启，代理池经环境变量 `SCHOLAR_PROXIES`（逗号分隔）或配置传入。
- 被拦时程序**不硬刚**：记 `block` 事件 → 冷却/换代理/升级浏览器（按开关）→ 拿不到就如实记 `blocked`，不崩。

## 三、常用参数

| 参数 | 说明 |
|---|---|
| `inputs...` | 位置参数：标题 / DOI / arXiv id（可多个） |
| `-f, --input-file` | 从 `.txt`/`.csv`/`.xlsx` 读入（自动识别 doi/title 列） |
| `--mode {auto,serpapi,self}` | 取回模式（见上） |
| `--serpapi-key KEY` | SerpApi key（缺省回落 `SERPAPI_KEY`） |
| `--proxy` | 启用住宅代理（默认关=直连）；池经 `SCHOLAR_PROXIES` |
| `--captcha` `--captcha-provider {2captcha,capsolver}` `--captcha-key` | 启用打码（默认关；灰色+持续付费，仅最后手段） |
| `--engine-order curl_cffi,nodriver` | 自建模式引擎降级顺序 |
| `--num N` | 每条输入期望结果数（默认 10） |
| `--year-from / --year-to` | 年份过滤（as_ylo/as_yhi） |
| `--oa-fallback` | **默认关=纯 Scholar**；开启后 Scholar 无 PDF 时复用父包 OA 源（Unpaywall/OpenAlex…）兜底 |
| `-o, --out DIR` | 输出目录（默认 `out_scholar`） |
| `-c, --concurrency N` | 并发（默认 1，强合规串行） |
| `--email you@uni.edu` | 供 OA 兜底的 Unpaywall（建议真实邮箱） |
| `--print-json` | stdout 输出 `{summary, results}` JSON（供父程序/子进程接入） |
| `--enable-scihub` | Sci-Hub 兜底（⚠ 合规风险，默认关） |
| `--selftest` | 不联网端到端自检 → `SCHOLAR_E2E_OK` |

## 四、产物（`out_scholar/`）

```
out_scholar/
├── pdfs/            # 标准化命名的 PDF：{year}_{author}_{title}.pdf（重名自动 _2/_3）
├── serp/            # 每次取回的原始 SERP HTML 快照（<idx>_<engine>_<status>.html）——便于校对选择器/复盘
├── metadata.jsonl   # 每条输入最终结果（断点续跑依据）
├── serp.jsonl       # 抓取类结构化事件：query / serp_fetch / block / captcha / proxy_rotate / result
├── attempts.jsonl   # 下载/兜底类事件
├── summary.json     # 运行汇总（成功率 / by_source / by_engine / blocked …）
├── results.csv      # 带状态结果表（UTF-8-SIG）
├── report.html      # 自包含 HTML 报表
└── run.log          # 人类可读日志
```

**运行 = 跑程序 + 读日志**：`serp.jsonl` 里每步 `ok/blocked/status/engine/ms` 一目了然；被拦、降级、命中都能从日志复盘。断点续跑：已成功的输入第二次运行自动跳过（读 `metadata.jsonl`）。

## 五、编程接口

```python
from fulltext_fetcher.scholar.config import ScholarConfig
from fulltext_fetcher.scholar.pipeline import ScholarPipeline

cfg = ScholarConfig(mode="serpapi", serpapi_key="…", out_dir="out_scholar", num=5)
pipe = ScholarPipeline(cfg)
summary = pipe.run(["Attention is all you need", "10.1038/nature14539"])
print(summary["success"], summary["success_rate"])
for r in pipe.results:
    print(r.raw_input, r.success, r.pdf_path, r.cited_by)
```

`ScholarPipeline(cfg, *, engines=…, client=…, proxy=…)` 三个注入点供测试/复用（离线 selftest 即用 FakeSerpEngine + FakeClient 驱动全链路）。

## 六、合规边界（务必知悉）

1. **优先正门**：能用开放 API / 商业合规 SerpApi 就不自建直抓；自建栈仅供研究/小规模自用。
2. **直抓 Google Scholar 属灰色**：违反其 ToS 与 robots.txt，使用者自负；对外提供服务前须过法务（注意 `nodriver`=AGPL 等许可证传染）。
3. **默认最安全**：代理 / 打码 / Sci-Hub / OA 兜底 **默认全关**；只「避免触发」，不主动对抗；限速/退避/冷却默认从严。
4. **付费墙是物理边界**：付费墙内全文任何免费手段都拿不到；本程序聚焦「Scholar 所能达到的」，闭源全文请走机构订阅/其它渠道。
