## 执行摘要(30 秒看懂)

- **课题**:输入**标题/DOI** → 自动过/绕谷歌学术人机验证 → **直取 PDF** → 全程程序化、一键批量。
- **核心结论**:**能用官方开放 API(OpenAlex/Semantic Scholar/Crossref/Unpaywall)就别硬抓 Scholar**——开放 API 无验证码、合规、免费、稳定,能拿到同样的元数据与开放获取 PDF;**付费墙内全文任何免费路线都拿不到**(物理边界,靠机构订阅解决)。
- **核心交付物**:`fulltext_fetcher/` —— 一个生产级**多源开放获取全文抓取包**(零第三方依赖,仅 `requests`),22 源回退 + 校验下载 + 并发限速熔断 + 断点续跑 + CLI/API/子进程三接入。
- **新增子系统**:`fulltext_fetcher/scholar/` —— **谷歌学术爬虫**(输入标题/DOI → 抓 Scholar 元数据 + 可及 PDF → 文件名标准化 → 批量),三层+两翼分层反爬(curl_cffi→nodriver,代理/打码默认关)+ SerpApi 商业合规路径(默认 auto);代码全绿、一键回归 **PASS=36**;**真实抓取需 SerpApi key 或住宅代理(数据中心 IP 直连 Scholar 几乎必 403)**。详见 §1.7。
- **实测(同一 500 DOI 语料,订阅刊为主)**:启用全部 22 源(尤其 `websearch` 搜索引擎找自存稿/机构库副本 + `publisher_oa`)后**免费公开边界内成功率 ~82%**(`out/batch6` 410/500;batch7 108/213 子集重跑;去重合并 412/500=82.4%,websearch 贡献约 72%);早期仅聚合 OA-API 源(未启用 websearch/publisher_oa)时为 **10.8%**(54/498),口径不同、勿直接相比。
- **下一步重点(投产比最高)**:① 机构订阅集成(合法拿付费墙内全文——免费边界之外那部分)② CORE+绿色 OA(低成本增益)③ Excel 入口+报表(体验质变)。

# 第一部分 · 项目现状(摘自《项目结构与功能介绍》v2.0)

## 1.1 项目定位

本仓库两层产物:**研究层**(七角度检索 + 聚合选型决策书 `检索成果-*.md`)+ **工程层 ★核心**(`fulltext_fetcher/` 生产级抓取包)。工程层要解决的具体问题:**输入 DOI/标题/arXiv → 全自动榨干全网可及的免费全文 → 校验下载入库**,追求高成功率 + 高速率 + 全程程序化("运行 + 读日志"即可迭代)。

## 1.2 目录与核心模块

```
fulltext_fetcher/        ★核心抓取包(仅依赖 requests)
├─ cli.py / __main__.py  命令行入口
├─ api.py                编程接口 fetch_one/fetch_many/FullTextFetcher
├─ config.py             配置 + DEFAULT_SOURCE_ORDER(源顺序/并发/限速/阈值)
├─ models.py             WorkInput/Paper/PdfCandidate/Attempt/FetchResult
├─ http_client.py        重试退避 + 按域限速 + 熔断器(线程安全)
├─ resolve.py            判类型 + 元数据富化(OpenAlex→Crossref)
├─ pipeline.py           编排器:并发/多源回退/首链短路/断点续跑/汇总
├─ download.py           下载 + %PDF 魔数校验 + 落地页二次解析
├─ landing.py            从 HTML 落地页抽内嵌 PDF 直链(自带 selftest)
├─ snapshot.py / ingest.py  官方快照→本地 SQLite 零联网线
└─ sources/              插件式源连接器(@register):snapshot/aggregators/repositories/scihub
poc/                     早期 PoC(已被生产程序取代)+ self_built_stack 自建栈样例
val500_out/              500 DOI 验证产物(证据链)
检索成果-*.md            七角度研究报告 + 聚合决策 + 验证报告
```

**新增「免费拿 PDF 方法」批次(10 模块,均带离线 selftest,已纳入一键回归 → 自检项由 26 增至 36)**:源插件 `publisher_oa`(OA 出版商 DOI→PDF 直构)、`publisher_direct`(订阅/混合刊直链,`--institutional` 默认关)、`oa_button`(OA.works API)、`preprints`(ChemRxiv/Research Square/Preprints.org)、`websearch`(DuckDuckGo/Bing 兜底)、`wayback`(Internet Archive 存档)、`free_adapters`(免费方法→源注册表集成层);父包顶层 `browser_search`(无头浏览器搜索,默认关)、`flaresolverr`(Cloudflare 挑战代解)、`bench_free_methods`(免费方法命中率评测台)。

## 1.3 全链路流程

`输入 → resolve 判类型+富化 → Paper → pipeline 按源优先级逐源定位 → 直链按置信度即时下载、首个成功短路 → 落地页兜底 → 校验落盘 + 三类结构化日志(attempts.jsonl/metadata.jsonl/summary.json)`。

关键设计:**源优先级+首链短路**(快)、**置信度排序**(准)、**%PDF 魔数校验防落地页冒充**、**按域限速+退避+熔断**(稳)、**元数据富化减少重复定位**、**断点续跑**、**本地快照零联网线**、**落地页二次解析**。

## 1.4 数据源一览(22 源默认启用 + Sci-Hub/浏览器搜索默认关)

`snapshot(本地快照)→ unpaywall → openalex → publisher_oa → oa_button → europe_pmc → arxiv → biorxiv → preprints → semantic_scholar → pmc → core(需key)→ base → crossref → doaj → openaire → hal → osf → zenodo → scienceopen → websearch → wayback`;均为 `@register` 插件,新增源只需加类并排位,无需改编排器(另 `browser_search`/`scihub`/`publisher_direct` 默认关,按需 `--sources`/`--enable-scihub`/`--institutional` 开启)。

## 1.5 验证结果(关键指标,同一 500 DOI 语料两种口径)

| 口径 | 成功 / 处理 | 成功率 | 主力源 |
|---|---|---|---|
| **现行:全部 22 源(含 websearch/publisher_oa)** | **410 / 500**(batch6) | **~82%** | websearch ≈72% + publisher_oa + 聚合 OA-API |
| 早期基线:仅聚合 OA-API 源(val500_out) | 54 / 498 | 10.8% | unpaywall 23、s2 20、openaire 5、crossref 3、europe_pmc 2、hal 1 |

> batch7 子集重跑 108/213;去重合并 **412/500 = 82.4%**。跃升几乎全来自 websearch(搜索引擎找自存稿/机构库 PDF)。早期基线的按出版商成功率(Springer Nature 37% > MDPI 13.3% > Wiley 9.5% > ACS 8.6% > RSC 7.7% > Elsevier 7.5%)与候选/失败细分见《项目结构与功能介绍》§五 与《规模化验证报告》。

> **产物留存说明**:`val500_out/` 完整留存 `pdfs/`(54 篇成功下载的正文 PDF)+ `summary.json/metadata.jsonl/attempts.jsonl/references.bib/zotero.csl.json/run.log`,证据链与 54 篇文献清单完整、正文可直接取用。

## 1.7 谷歌学术爬虫子系统(fulltext_fetcher/scholar/)

在"绕开 Scholar、走开放 API"主线之外,本仓另建了**正面实现**的**谷歌学术爬虫子系统** `fulltext_fetcher/scholar/`:**输入标题/DOI →(避免或自动过人机验证)→ 抓 Scholar 元数据(被引/版本/作者/年)+ Scholar 可及 PDF → 文件名标准化 → 一键批量**,全程结构化日志。架构与接口契约见 `谷歌学术爬虫-架构与选型.md`,详细用法见 `谷歌学术爬虫-使用说明.md`,踩坑见 `经验记录-踩坑与发现.md` F 节。

- **架构(三层+两翼反爬,默认最安全)**:L1 `curl_cffi`(TLS 指纹静态抓)→ L3 `nodriver`(直连 CDP 浏览器)→ `patchright`(备选);翼A 住宅代理、翼B reCAPTCHA 打码**默认全关**(避免触发优先、不硬刚)。`--mode` 默认 `auto`:有 SerpApi key 走**商业合规 SerpApi**,否则走自建栈。复用父包 `download/landing/sources/resolve/report`;可选反爬库(curl_cffi/nodriver/patchright/2captcha-python/python3-capsolver)全部**函数内延迟导入、不进强制依赖**。
- **模块**(scholar/ 共 14 个 .py):`models/config/logsetup/query/serp/fetcher/proxy/captcha/download/naming/pipeline/cli/__main__/selftest_e2e`。父包 OA 多源作"其它渠道"兜底(`--oa-fallback`,默认关)。
- **用法**:`python -m fulltext_fetcher.scholar "标题或DOI" [--serpapi-key <KEY> | --proxy | --oa-fallback]`;不联网自检 `--selftest` → `SCHOLAR_E2E_OK`;产物落 `out_scholar/`(`pdfs/` + `serp/` + `metadata.jsonl` + `serp.jsonl` + `attempts.jsonl` + `summary.json` + `results.csv` + `report.html` + `run.log`)。
- **现状与边界(如实)**:代码全绿(各模块离线 selftest + 端到端 `SCHOLAR_E2E_OK`),一键回归 `python run_all_selftests.py` → **PASS=36 / FAIL=0 / SKIP=0(total=36)**;但**数据中心 / 机房 IP 直连 Scholar 几乎必被 HTTP 403**(IP 信誉边界,非程序缺陷,实测 curl_cffi 伪装 Chrome TLS 亦即刻 403),**真实抓取需 SerpApi key 或住宅代理**;付费墙内全文仍是物理边界。另新增可选异步并发原型 `fulltext_fetcher/aio.py`(C4,httpx/aiohttp 可选、同步核心 100% 不变),用于大批量"定位/元数据"阶段提吞吐。

# 第二部分 · 优化与新增功能建议(摘自《功能优化与新增功能建议》)

## 2.1 目标对齐与差距

| 目标 | 状态 | 差距 |
|---|---|---|
| 输入标题/DOI | ✅ | 缺 Excel/CSV 批量入口 |
| 过/绕人机验证 | ⚠️ 走开放 API 绕开 | 无 Scholar 原生口径;直抓仅 PoC 骨架 |
| 直取 PDF | ✅ 22 源 | 免费公开边界 ~82%(websearch 主力);剩余下不动的多为付费墙内全文 |
| 程序化/批量 | ✅ | 缺 GUI/报表、Zotero 直写、失败分类重试 |

## 2.2 分类建议(标注优先级)

**A. 提升成功率(对齐"直取 PDF")**
- A1【P0】集成 CORE key + 补绿色 OA 仓储(BASE/OSF/SSRN)——低成本 +2~4pp。
- A2【P0】落地页解析增强(出版商专属 selector + JSON-LD)——回收 403 落地页 +2~5pp。
- A3【P1】出版商适配器层(按 DOI 前缀路由 + TDM 协议)。
- A4【P1】浏览器渲染兜底(Playwright/nodriver,可选默认关)。
- **A5【P1】机构订阅/EZproxy/Shibboleth 集成 ★唯一能合法拿付费墙内全文,成功率可跃升至 70%+。**

**B. 补 Scholar 原生能力(对齐"谷歌学术检索/验证")**
- B1【P1】SerpApi Google Scholar API(商业合规)补被引/版本;或用 S2/OpenAlex 被引近似。
- B2【P2】反爬栈骨架产品化(curl_cffi→nodriver→代理→打码,默认关、强合规警告)。
- B3【P2】代理池集成(轮换+健康检查)。

**C. 工程化与一键批量(对齐"全程程序化")**
- C1【P0】Excel/CSV 入口 + 带状态结果表(贴合验证用的 xlsx)。
- C2【P1】失败分类重试(永久失败如 403/404 不重试,只重试 5xx/超时)。
- C3【P1】Zotero 直写(从 PoC 移植到生产)。
- C4【P1】异步并发(httpx/aiohttp)。 C5【P2】Web/GUI 面板。 C6【P2】运行报表可视化。

**D. 质量可靠性**:D1【P1】测试+CI、D2【P1】PDF 完整性校验、D3【P2】自适应限速。
**E. 数据闭环**:E1【P1】快照线落地引导、E2【P2】本地全文检索。

> **进度更新(2026-07-01)**:上表中 **B1(SerpApi)/ B2(反爬栈骨架)/ B3(代理池)/ A4(浏览器渲染兜底)/ C4(异步并发)** 已在新的 `fulltext_fetcher/scholar/` 子系统与 `fulltext_fetcher/aio.py` 中落地实现(详见 §1.7);均**默认关闭、合规优先**。付费墙天花板与机构订阅(A5)结论不变。

## 2.3 优先级路线图

| 期 | 包含 | 预期效果 |
|---|---|---|
| 一期(P0,1–2 天)| A1+A2+C1 | 订阅刊 +4~9pp;一键读 xlsx |
| 二期(P1,3–5 天)| A3+A5+C2+C3+D1 | 有订阅者成功率破 50%+;更稳更快 |
| 三期(P2,按需)| B1+A4+C5+E2 | 补 Scholar 口径、产品化 |

## 2.4 边界提醒

1. **维持"绕开 Scholar"主线**:开放 API 是唯一白色合规路线;直抓 Scholar 灰色高风险,仅按需启用。
2. **付费墙是物理边界**:免费公开边界已达 ~82%(websearch 等);剩余下不动的主要是付费墙内全文,根本解法是机构订阅(A5),不靠"更强反爬"。
3. **被引/版本 ≠ 全文**:要 Scholar 口径用 SerpApi(B1)即可,无需自建反爬。
4. **合规底线**:所有灰色路线默认关闭 + 显式风险提示,使用者自负。

# 第三部分 · 结论与行动建议

1. **现状评价**:工程层已是可用的生产级包,合规、零依赖、可扩展;同一订阅刊语料上免费公开边界内实测 ~82%(全部 22 源,websearch 主力;早期仅 OA-API 口径为 10.8%)——**作为"合规免费全文抓取器"已达成目标**。
2. **离目标还差什么**:① 真正"过 Scholar 人机验证"未做(主线选择绕开,正确);② 付费墙内全文受限(需机构订阅);③ 批量入口/报表/Zotero 直写等体验项待补。
3. **建议立即做(一期 P0)**:`CORE key + 落地页增强 + Excel 入口` —— 投入小、合规、直接抬升成功率与可用性。
4. **建议尽快评估(二期 P1 重点)**:`机构订阅集成(A5)` —— 免费公开边界已达 ~82%,机构订阅是合法拿到剩余付费墙内全文的唯一根本途径,优先级最高。
