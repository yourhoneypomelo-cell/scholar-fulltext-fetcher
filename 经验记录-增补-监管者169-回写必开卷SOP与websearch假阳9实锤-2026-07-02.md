## N.1 系统性根因 = 「信下载期 metadata,不信落盘 PDF 字节」(与 147 逮到的 1.5053761 同源)

- 147 逮到 `10.1116/1.5053761`:metadata.jsonl 记 match/100/expected-doi-present,但落盘 PDF 实为《AIP Content Platform User Guide》操作手册(正文/DocInfo/XMP/超链接四处均无期望 DOI)。
- 本轮监管者全量开卷 websearch(151 条)+ allow_override(13 条)发现同一根因批量存在:**内容 QC 的 verdict 若来自下载环节记录、而非对最终落盘 PDF 重新开卷,就会把「顶包文件」当成功**。
- websearch 是头号重灾区:它抽搜索引擎结果页的 URL 去下,极易下到**同名/撞关键词的完全无关 PDF**(教科书、设备手册、政府章程、医疗规范、产品目录…),且这些文件往往 >100 页、体积大,反而"看起来像"成功。

## N.2 回写「必开卷读盘」SOP(任务3 · 建议固化为回写强制前置门)

**规矩(写死):凡 qc-allow / 内容 QC 判据要用于 coverage 回写或纠假阳,必须对「最终落盘 PDF」重新开卷,核 `expected-doi-present`,禁止直接采信 `metadata.jsonl` 下载期 verdict。**

判定优先级(从强到弱):
1. **正文含期望 DOI 或其尾号**(去标点容错)= 真(黄金判据,天然击穿 openaire M.5 邻居假阳:邻居 PDF 只含邻居 DOI)。
2. DOI 未检出但**标题 token 高度命中正文**(title_overlap ≥ ~0.7)= 大概率真(DOI 常因首页为图/老刊未印而抽不到)—— **不可据此误杀**。
3. DOI 未检出**且**标题 token 命中低(≤0.3)**且**正文有可读文本 = **高危抓错文**,须开卷人核真身。
4. 正文抽取为空/乱码(chars≈0,或字体 CMap 未实现/加密)= **无法判定**,转 OCR 或换源重取,不得计成功。

可复现探针(只读,已入库):
- `_mon169_qc_openbook_scan.py` — 全量扫 coverage 里 success 的 websearch+allow,出 `out/_mon169_openbook_scan.csv`(doi/source/bytes/pages/chars/doi_in_body/title_overlap/verdict)。
- `_mon169_identify_suspects.py` / `_t0_adjudicate147_probe.py`(147 产)— 定向开卷识别真身。

**建议(需总指挥/pipeline 属主拍):把该 SOP 硬编进 `tools/build_coverage.py` 的 qc_allow 分支** —— qc_allow 命中项在计 success 前,先对其落盘 PDF 跑一次 `expected-doi-present` 开卷校验,不过则拒绝 allow(防止再把顶包文件洗白)。此改动触及核心码,监管者不擅自改,提交属主实施。

## N.4 待复核桶(不进实锤、不误杀)

- **抽取失败/乱码(chars≈0,~13 条)**:如 `10.1002/anie.201203185`、`10.1002/cctc.202201338`、`10.1021/cm061060s/cm062127e`、`10.1021/cs500979c`、`10.1016/j.matlet.2015.09.140`、`10.1016/s0926-860x(98)00143-4`、`10.1039/c5cy00124b`、`10.1021/acssuschemeng.4c05125`、`10.1002/9783527626854.ch16`、`10.1016/j.jcou.2026.103385`、`10.1021/ie020677q`(抽出日文)、`10.1016/j.catcom.2018.07.014`(CMap 损坏)、`10.1021/acs.iecr.4c02255`(加密无法读)。→ 转 OCR / 换源,判定前不得计成功。
- **可能真但 DOI 未抽到(title_overlap≥0.7,~30 条)**:一律**保留为 success**,不误杀(如 cr970037a、b918763b、susc.2005.08.015 等,标题全中)。

## N.6 教训

1. **coverage 是活的、会在你巡检期间被重建**:本轮开工时 coverage=379/allow 4,巡检中被 151 重建到 388/allow 13。报基线必带 `generated_ts` 与 allow_override 数,别拿旧快照当现值。
2. **success 数越大越要开卷**:websearch 顶包文件常又大又厚,体积/页数不是成功信号,反而是危险信号。
3. **只读、可复现、不误杀**:监管者只出证据与候选清单,判黑/回写交属主;title_overlap 高者一律保留,宁可漏放不可错杀。
