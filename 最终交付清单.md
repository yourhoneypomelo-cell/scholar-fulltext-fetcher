## 一、怎么跑（全程程序化）

```bash
# —— OA 多源下载（父包）——
python -m fulltext_fetcher "10.1038/nature14539" --email you@uni.edu     # 单条
python -m fulltext_fetcher -f dois.txt --email you@uni.edu -c 6           # 批量(.txt/.csv/.xlsx)

# —— 谷歌学术爬虫（子系统；两种入口等价）——
python -m fulltext_fetcher.scholar "Attention is all you need"           # 独立入口
python -m fulltext_fetcher scholar "10.1038/nature14539" --num 5          # 统一子命令
python -m fulltext_fetcher.scholar -f titles.txt --serpapi-key <KEY>      # 真实抓取(合规) 或 --proxy

# —— 自检 / 一键回归 ——
python -m fulltext_fetcher.scholar --selftest    # → SCHOLAR_E2E_OK（不联网）
python run_all_selftests.py                      # → PASS=26（父包 + scholar + 增强，全项目）
```

产物落 `out_scholar/`（Scholar）或 `out/`（OA）：`pdfs/`（文件名标准化）+ `serp/`（原始 SERP 快照，便于校对选择器）+ `metadata.jsonl` + `serp.jsonl` + `attempts.jsonl` + `summary.json` + `results.csv` + `report.html` + `run.log`。断点续跑：已成功输入二次运行自动跳过。

## 三、还差什么（剩余 ~30%：Scholar 真实验收，仅差凭据）

从本机（数据中心 IP）直连 Scholar 必 403，无法真实产出/验证。**三步全部依赖凭据，无代码可绕**：

1. **环境就绪**：提供 **SerpApi key**（serpapi.com，免费档 100 次/月，最省）或 **住宅代理**（环境变量 `SCHOLAR_PROXIES`，逗号分隔）。
2. **真实 SERP 选择器验证**：跑一次真实抓取 → `out_scholar/serp/` 存下真实页面 → 校对 `serp.py` 选择器是否匹配当前 Scholar HTML。
3. **规模化验收**：50–100 条真实 title/DOI 批量跑，统计 Scholar 通路 PDF 获取率。

> 解锁后预计 10 分钟内可补跑并给出真实数据。
