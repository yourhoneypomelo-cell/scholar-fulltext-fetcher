# fulltext_fetcher 程序使用说明

> 输入 DOI / 标题 / arXiv id,全自动榨干全网可及的**免费**全文资源并下载入库。
> 主线哲学:走开放获取(OA)正门,**不**碰 Google Scholar 人机认证(零验证码、合规、稳定)。
> 版本 1.0 · 2026-07-01 · 署名 谷歌学术人机认证-141(总指挥)

---

## 1. 安装

只依赖 `requests`(其余均为 Python 3.8+ 标准库)。

```bash
cd e:\AI项目\谷歌学术人机认证
pip install -r fulltext_fetcher/requirements.txt
```

## 2. 快速开始

```bash
# 单个 DOI
python -m fulltext_fetcher "10.7717/peerj.4375" --email 你的真实邮箱@xx.edu

# 标题(自动 Crossref 反查 DOI 再抓)
python -m fulltext_fetcher "Attention is all you need" --email 你@xx.edu

# arXiv id
python -m fulltext_fetcher "1706.03762" --email 你@xx.edu

# 批量(文件每行一个 DOI/标题/arXiv,# 开头为注释)
python -m fulltext_fetcher -f dois.txt --email 你@xx.edu --concurrency 6
```

> ⚠️ **务必填真实邮箱**:Unpaywall 对 `@example.com` 占位邮箱返回 422;真实邮箱才能用上命中率最高的源。

## 3. 命令行参数

| 参数 | 默认 | 说明 |
|---|---|---|
| `inputs...` | — | 位置参数:一个或多个 DOI / 标题 / arXiv id |
| `-f, --input-file` | — | 从文件逐行读取输入 |
| `--email` | 环境变量 `FULLTEXT_EMAIL` | 联系邮箱(Unpaywall 必需真实) |
| `-o, --out` | `out` | 输出目录 |
| `-c, --concurrency` | `4` | 并发输入数 |
| `--timeout` | `30` | 单请求超时(秒) |
| `--max-retries` | `3` | 429/5xx 重试次数 |
| `--oa-only` | 关 | 仅尝试直链候选,跳过落地页兜底 |
| `--no-download` | 关 | 只定位不下载(快速验证各源命中,**诊断神器**) |
| `--sources` | 全部 | 逗号分隔自定义源及顺序,如 `unpaywall,openalex,arxiv` |
| `--openalex-key` | 环境变量 `OPENALEX_KEY` | OpenAlex API key(大批量建议申请,免费) |
| `--s2-key` | 环境变量 `S2_KEY` | Semantic Scholar key(生产建议申请) |
| `--core-key` | 环境变量 `CORE_KEY` | CORE key(不填则自动跳过 CORE 源) |
| `--enable-scihub` | 关 | ⚠️ 启用 Sci-Hub 兜底(合规风险,使用者自负) |
| `--log-level` | `INFO` | DEBUG/INFO/WARNING/ERROR |

## 4. 输出产物(`out/` 目录)

| 文件 | 内容 | 用途 |
|---|---|---|
| `pdfs/*.pdf` | 下载的全文 PDF(按 DOI/arXiv 命名) | 最终成果 |
| `metadata.jsonl` | 每条输入一行:DOI/标题/是否成功/命中源/PDF路径/各源 attempts | 结果总账 + **断点续跑依据** |
| `attempts.jsonl` | 每个事件一行(input/resolved/source/download/result) | **日志驱动调试**的核心 |
| `summary.json` | 本次运行汇总:成功率、各源命中数、耗时 | 一眼看效果 |
| `run.log` | 人类可读运行日志(UTF-8) | 排错 |

> Windows PowerShell 控制台中文可能显示乱码(GBK),这只是**终端显示**问题;所有落盘文件均为 UTF-8,内容正确。

## 5. 日志驱动的迭代闭环(本程序的设计目标)

你**不需要**在抓取过程中手动干预。标准工作流:

1. **运行**:`python -m fulltext_fetcher -f dois.txt --email ...`
2. **读日志**:看 `summary.json`(成功率/各源命中)→ 对失败条目在 `attempts.jsonl` 里按 `raw` 过滤,看每个源的 `ok/n/ms/error` 与 `download` 的 `error`(如 `landing-page` / `not-pdf` / `http-403` / `too-small`)。
3. **改程序**:据日志定位是"某源字段路径变了 / 某源限速 / 落地页未识别"等,改对应 `sources/*.py` 或参数。
4. **再运行**:已成功的输入会自动跳过(断点续跑),只重试失败项。
5. 重复 2–4,直到成功率满意。

## 6. 断点续跑

重复对同一 `--out` 目录运行,程序会读取 `metadata.jsonl` 中 `success=true` 的输入并**自动跳过**,只处理未成功项。想全量重跑就换一个 `--out` 目录或删除该目录。

## 7. 调优建议

- **提速**:加大 `--concurrency`(8–16);程序按"首个直链下载成功即短路"——命中越靠前的源越快。
- **提命中**:填 `--email`(开 Unpaywall)、`--openalex-key`、`--core-key`;生物医学文献 Europe PMC/PMC 命中高。
- **只看覆盖**:`--no-download` 跑一遍,看哪些源对你的数据集命中率高,再用 `--sources` 裁剪提速。
- **合规优先**:默认不启用 Sci-Hub;付费墙内全文任何免费手段都拿不到,属预期。
