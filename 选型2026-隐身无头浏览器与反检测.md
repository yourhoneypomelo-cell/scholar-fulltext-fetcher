## 〇、一页速览（TL;DR，面向 `browser_search`）

- **主力不变、已落地**：`browser_search` 的隐身取页应以 **`nodriver`（真 Chrome，直连 CDP）为首选**，`patchright`（`channel=chrome`）为回退，二者均已在本模块加固接线（`_fetch_via_nodriver` → `_fetch_via_patchright` → 兜底复用 `scholar.fetcher`）。这与 153/155 的通用结论一致，且**被本仓真机实测证实**。
- **一手实测（本机 Windows 11 + 真 Chrome 148 + nodriver 0.50.3，headless）**：对 5 条 batch6 样本（2 标题 + 3 冷门催化 DOI）驱动 **Bing** —— **5/5 全部过 bot 检测（无验证码/软验证）**；解析出 **55 个 PDF/自存稿候选 vs 纯 HTTP(curl_cffi) 仅 4 个，其中 52 个是 HTTP 完全拿不到的**。→ **对"搜索引擎 SERP 过验证 + 多拿候选"，nodriver headless 在本机确实可用且显著优于纯 HTTP。**
- **2026 基准复核（Ian L. Paterson，31 CF 目标 / 651 判定）**：`nodriver` **28 OK / 0 blocked**（唯一零封锁，用系统 Chrome 148）＞ `patchright` 25 ＞ `camoufox` 25(Firefox135beta) ＞ vanilla/rebrowser 24。`curl_cffi`（21 行）26 OK，与整只 Chromium fork 打平。
- **过"最狠"出版商验证（DataDome / CF Managed Challenge）**：若 `nodriver` 也被拦，**升级 `camoufox`**（Firefox 引擎 C++ 级指纹，canvas/WebGL/OS 一致性最强）；但 **camoufox 2026 处于"维护断档后恢复期、releases 高度实验性、不建议生产"**，仅作**受控兜底**，不进默认路径。
- **不用 / 非本栈**：`undetected-chromedriver`（作者已引导转 nodriver、渐失效）、`playwright_stealth`（POC，puppeteer/playwright-stealth 2025-02 已停）、`rebrowser-patches`（**Node-only**，仅补 puppeteer/playwright 的 `Runtime.enable` 泄漏；Python 侧对应物是 patchright，故本仓不直接用）。
- **反检测没有银弹**：胜负手在 L3「怎么驱动浏览器」+ IP 信誉 + 行为，不在 L1「声称什么」。`browser_search` 的纪律是 **命中即止、低频、Bing 主（与 146 的纯 HTTP-DDG 分工错峰）**。

## 三、本仓一手实测（区别于二手评测的关键证据）

**环境**：Windows 11（本机公网 IP、无代理）、Python 3.11、真 Chrome 148（`C:\Program Files\Google\Chrome\Application\chrome.exe`，nodriver 自动定位）、nodriver 0.50.3 / patchright / curl_cffi 0.15.0 均已装。

**方法**：对同一检索式，分别用「纯 HTTP（curl_cffi impersonate=chrome）」与「nodriver headless（真 Chrome 渲染 + 滚动）」取 **Bing** SERP，复用 `browser_search.parse_pdf_candidates` 解析，只比较取候选能力（唯一变量=取页方式）。样本=5 条 batch6 miss（`attention is all you need`、`deep residual learning`、`10.1021/acscatal.7b01827`、`10.1039/c3ee44078h`、`10.1002/anie.200603931`）。

**结果**：

| 指标 | 纯 HTTP（curl_cffi） | nodriver headless（真 Chrome） |
| --- | --- | --- |
| 过 Bing bot 检测 | 页面能取（200）但**结果链常抽不出** | **5/5 全过，无验证码/软验证** |
| PDF/自存稿候选总数 | **4** | **55** |
| 仅浏览器能拿到（HTTP 拿不到） | — | **52** |
| 单条 DOI 例：c3ee44078h / anie.200603931 | 0 / 0 | 9 / 7 |
| 单次取页耗时 | ~1s | ~10–13s（含启动） |

**解读**：Bing 结果链依赖 **JS 渲染 + `ck/a` base64 包装**，纯 HTTP 抓到页却抽不出结果链（5 条里 4 条 HTTP=0 候选）；真 Chrome 渲染后能完整解包。→ **对"搜索引擎 SERP 过验证 + 召回自存稿"，nodriver headless 在本机既能过验证、又显著多拿**（尤其冷门催化 DOI，Bing 召回远优于 DDG）。代价是每条 ~10s 启动开销。

> 与二手评测的一处校正：ByteTunnels 称"nodriver headless 在激进站点易被额外检测"，但**对 Bing/DDG SERP 这类目标，本机 headless 实测稳定通过**。对更狠的出版商 DataDome/CF-Managed，才需要考虑 headed/Xvfb 或升级 camoufox。

## 四、Top1-2 深读（隐身手法 / Windows 可用性 / 是否需真实 Chrome）

### 4.1 nodriver（Top1，首选）
- **隐身手法**：undetected-chromedriver 官方继任者（同作者），**纯异步直连 Chrome DevTools Protocol**，无 Selenium/chromedriver、无 Playwright shim、无 `Runtime.enable` 启动序列、无 `navigator.webdriver`/`cdc_`/`--enable-automation` 痕迹——"启动方式就像人双击图标"。0.50.x 切 flat-mode 连接。
- **新增（2026 文档核到）**：内置 CF 挑战辅助 `cf_verify()` / `verify_cf()`（用 OpenCV 定位并点击"I'm not a robot"，需 `opencv-python`）；多数 CF 站"navigate 即自动过"。
- **Windows / 真 Chrome**：✅ 本机实测可用，**用系统真 Chrome 148**（非 bundled Chromium，TLS/版本戳真实）。
- **代价**：① **AGPL-3.0**（对外提供网络服务须开源修改；个人/内部研究无碍，商用前过法务）；② 不自动解 Turnstile/reCAPTCHA **token**（只让你"不被识别"）；③ 官方称 **headless 更易被检测**、代理鉴权（user:pass）尚不成熟——但本课题无代理、且实测 SERP headless 稳定，影响可控。

### 4.2 patchright（Top2，回退）
- **隐身手法**：Playwright 的 **drop-in 替代**，在协议层打掉 `Runtime.enable` / `Target.setAutoAttach` 泄漏，`channel=chrome` 驱动**系统真 Chrome**（真 TLS + 真版本）；自述过 Cloudflare/Kasada/Akamai/DataDome/Fingerprint。基准 25/31（比 vanilla 多解锁 stackoverflow，因跑真 Chrome 148）。
- **Windows / 真 Chrome**：✅ 需系统 Chrome；`new_context` 可定制 UA/locale/viewport/Accept-Language（本仓 `_fetch_via_patchright` 已如此）。
- **定位**：代码已在 Playwright 生态（选择器/等待 API）时零改动升级；**Apache-2.0 商用友好**。作为 nodriver 之外的第二真 Chrome 通道，抗"单引擎失效"。

### 4.3 camoufox（受控兜底，非默认）
- **隐身手法**：Firefox 引擎 **C++ 级**拦截 navigator/WebGL/screen/fonts/WebRTC 等所有指纹（非 JS 注入，无注入痕迹），逐运行取真实设备分布的指纹、可轮换；Playwright 兼容、headless-first、~200MB。对 **canvas/WebGL/OS 一致性交叉核验**（DataDome/CF-Managed 的杀手锏）最强。
- **2026 现状（重要）**：**曾有约一年维护断档**，因基 Firefox 版本旧 + 新指纹不一致导致性能下滑；**当前恢复开发中，releases 高度实验（v150.x-beta，预期破坏性变更），官方明确"不适合生产"**；开发迁至 `CloverLabsAI/camoufox`。⭐~9,100（远超 155 文档记的 ~1,000）。
- **裁决**：**仅作"nodriver/patchright 都被最狠出版商验证挡住时"的受控兜底**，用稳定分支（如 `releases/135` 或官方 `camoufox` pip 稳定包），**不进 `browser_search` 默认路径**（稳定性/Firefox 系差异/Win 支持随版本）。

## 五、按目标分场景（哪个最能过谁的验证）

| 目标 | 特征 | 首选 | 兜底 | 依据 |
| --- | --- | --- | --- | --- |
| **Bing / DuckDuckGo SERP** | JS 渲染 + 重定向包装，软风控 | **nodriver（真 Chrome）** | patchright | 本仓实测 5/5 过 Bing、55 候选 |
| **Google（SERP / Scholar）** | 强行为评分，易弹"unusual traffic" | **nodriver headed/Xvfb** + 强限速 | SerpApi（合规正门） | 155/153：Scholar 首要"不触发"；本仓 `scholar.fetcher` 已分层，`browser_search` 默认不打 Google |
| **出版商 OA 落地页（多数）** | 通常无反爬 | 复用 `fulltext_fetcher/download+landing` | curl_cffi | 155 §4.3 |
| **出版商 / 镜像 走 Cloudflare** | Turnstile / Managed Challenge | **nodriver（基准 0 封锁，`cf_verify()`）** | camoufox（受控） | 基准 28/31；nodriver CF 文档 |
| **出版商走 DataDome** | TLS(JA4)+行为+canvas 交叉核验最狠 | **camoufox（引擎级指纹）** | patchright（CDP 层修补） | DataDome-2026 评测：需引擎/CDP 级、非 JS 插件 |

> 一句话：**Google Scholar / 一般出版商用 nodriver 最稳**（本仓实测 + 基准双证）；**只有撞上 DataDome 这类"指纹交叉核验最狠"的出版商，才升级 camoufox 引擎级方案**（受控、非默认）。

## 六、对 `browser_search` 的集成建议（Top 采用与要点）

1. **保持现状主力**：`nodriver` 首选、`patchright` 回退、兜底 `scholar.fetcher`——已在 `browser_search.py` 加固接线（UA / `--disable-blink-features=AutomationControlled` / 语言 / 视口 / 渲染等待 + 滚动 / 验证码页识别 + 引擎回退）。**无需换栈。**
2. **配置项（建议总指挥统一加入 `ScholarConfig`，本仓已用 `getattr` 兜底、字段不存在也不崩）**：`browser_headless: bool=True`、`browser_render_wait: float=3.0`、`browser_max_queries: int=2`（`user_agent` 已存在，复用）。
3. **过更狠验证的升级位（可选、受控）**：预留"当 nodriver/patchright 连续被拦（`reason=search-engine-blocked` 或出版商 CF/DataDome）时，可选启用 camoufox"的扩展点；因 camoufox 当前实验期，**默认关闭**，仅单用户/生产 IP 且确有 DataDome 目标时开。
4. **纪律**：命中即止、低频、**Bing 主**（与 146 的纯 HTTP-DDG 分工、同 IP 错峰）；headless 已够用于 SERP，若遇 Google/强验证再考虑 headed/Xvfb。
5. **合规/许可**：nodriver=AGPL-3.0、camoufox=MPL-2.0、patchright=Apache-2.0；自动化驱动搜索引擎/绕人机验证处灰色地带，仅研究/小规模自用，对外服务前过法务。
