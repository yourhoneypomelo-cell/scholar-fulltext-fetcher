## 〇、TL;DR — 真源码比 -177 骨架多出的 5 点

| # | SciTeX 真机制（实读 v1.4.x） | -177 骨架缺什么 | 对本仓的落地增量 |
|:--:|---|---|---|
| **1** | **`verify_live` 活校验**：`ensure_authenticate_async(verify_live=True)` / `is_authenticate_async(verify_live=True)` —— 缓存会话每次用前**发一次廉价探针确认仍授权**，不只看本地过期戳 | 骨架 `CookieStore.get(max_age_s)` **只按 `saved_at` 年龄判有效**，会话被 IdP 提前失效时仍误用 | CookieStore 增 `verify_live` 钩子：用前打 1 个已知订阅页 HEAD/小 GET，命中登录墙即判失效、触发重登 |

## 一、真源码架构（实读 v1.4.x API Reference + 文件树 + v1.4.0 发布说明）

**主类**：`Scholar`（Mixin 组合：Enricher / URLFinding / **PDFDownload** / Loader / Search / Saver / ProjectHandler / LibraryHandler / Pipeline / Service）。
**全链一次调用**：`paper fetch --doi <DOI>` = `enrich → resolve URL → authenticate → download → store`（落 `MASTER/<HASH>/` + metadata + PDF + 每项目 symlink）。

**模块树**（与认证/下载相关）：
- `auth/` —— **OpenAthens / EZProxy / Shibboleth / Lean Library** 自动化器（`ScholarAuthManager` 统一接口）。
- `browser/` —— 持久化 profile 的浏览器管理（stealth + interactive），`ScholarBrowserManager`。
- `url_finder/` —— PDF URL 发现（**Zotero translators** + 启发式策略），`ScholarURLFinder`。
- `pdf_download/` —— `paper fetch` 三策略：**chrome viewer / direct / fallback**。
- `storage/` —— MASTER-hash 库 + `metadata.path.files` 注册表（`{role,name,sha256,size,added_at,source,immutable}`）。

**认证态位置**：`~/.scitex/scholar/cache/auth/<provider>.json`（cookie）+ Chrome profiles；`SCITEX_DIR` 可覆盖。

**关键 API 签名（实读，供映射）**：
```text
ScholarAuthManager(email_openathens=, email_ezproxy=, email_shibboleth=, config=)
  async ensure_authenticate_async(provider_name=None, verify_live=True, **kwargs) -> bool
  async is_authenticate_async(verify_live=True) -> bool
  async authenticate_async(provider_name=None, **kwargs) -> dict
  async get_auth_headers_async() -> Dict[str,str]
  async get_auth_cookies_async(essential_only=True) -> List[Dict]
  set_active_provider(name) / get_active_provider() / list_providers()
  async logout_async()
ScholarBrowserManager(browser_mode=, auth_manager=, chrome_profile_name=, config=)
  async get_authenticated_browser_and_context_async(**ctx) -> (Browser, BrowserContext)   # 注入 auth cookie + 扩展
ScholarURLFinder(context, config=)
  async find_pdf_urls(page_or_url, base_url=None) -> List[{"url","source"}]
```

> 依赖差异：SciTeX 用 **Playwright**（重）。本仓首选 **nodriver**（已装、route-B 已用）。下面映射一律落到 nodriver。

## 二、映射到本仓：修正 -177 骨架的「授权会话内下载」这一步（增量重点）

**-177 骨架原设计**：`Institutional.find_candidates()` 产 `PdfCandidate(meta={"needs_authorized_session":True})` → 交 `download.py` 走 `render_download_pdf_bytes(url)`。
**真源码校正**：授权会话内下载的**正确形态**是「持久化 cookie **注入同一 nodriver context** → 同会话内 `fetch()` 直下」，与 **N.8 route-B B1** 完全同构。据此把 A5 下载支路明确为：

```python
# 概念骨架(实现归 -153;此处只给正确的机制次序,不改任何 .py)
async def a5_authorized_download(article_url, doi, cookie_store, provider, cfg):
    host = _host(article_url)
    # ① verify_live:缓存 cookie 用前活校验(真源码 delta-1),失效则重登
    cookies = cookie_store.get(host)
    if not cookies or not await _verify_live(host, cookies):
        cookies = await open_login_browser(login_url_for(provider), cookie_store, host)  # 人工 SSO
    # ② 注入同一 nodriver context 再导航(真源码 delta-2;JA3 站唯一正解)
    browser = await nd.start(headless=False)          # 复用 render_fetch 的 CDP 兼容 patch
    ctx = browser  # nodriver: 用 Network.setCookies 注入 cookies 到该 context
    await _inject_cookies(ctx, cookies)               # 导航前注入
    tab = await ctx.get(article_url)                  # 命中机构授权会话
    await _wait_pass_cf(tab)                           # 过盾期不开 CDP 拦截(N.8 铁律#1/#2)
    # ③ 同会话内 fetch() 直下(N.8 B1);B1 被 CSP/跨源挡→B2(过盾后 Fetch.enable)
    pdf_bytes = await _b1_inpage_fetch(tab, pdf_url_for(article_url)) \
                or await _b2_nav_capture(tab, pdf_url_for(article_url))
    # ④ 落盘前必过内容 QC 门③(真源码 delta-4 佐证):page-1 DOI ≠ 期望 → 拒
    if not content_qc_ok(pdf_bytes, doi):             # 复用 tools/qc_content_match
        return None
    return pdf_bytes
```

要点：
- **过盾纪律沿用 N.8**（过盾期绝不开 Fetch/Network 拦截、以 `cf_clearance` + 可见文案判过盾、B1 优先 B2 兜底、`Page.printToPDF` 不用、CDP 双键 monkeypatch）。机构站多在 CF/JS 后，A5 与 route-B 是**同一套过盾+直下**，仅多了「注入机构 cookie」这一步。
- **别再走 curl_cffi 裸回放**（-177 骨架那步）——对 JA3 绑定订阅站（RSC/ScienceDirect）必 403。

## 三、`verify_live` 活校验（真源码 delta-1，补 CookieStore）

-177 的 `CookieStore.get(max_age_s=14d)` 只按本地年龄判有效。真源码 `verify_live=True` 说明：**SSO 会话可能在过期戳之前就被 IdP 失效**（登出、并发会话上限、策略变更）。补一个活校验钩子：

```python
async def _verify_live(host, cookies, *, probe_url=None) -> bool:
    """用前活校验:注入 cookie 打一个该出版商已知订阅页,若跳到登录墙/entitlement 缺失 → False。
    廉价(HEAD 或小 GET),命中即信任;失败即触发重登,避免"过期戳没到但其实已失效"误用。"""
    ...  # 命中订阅正文特征=True;命中 SSO/login/paywall 提示=False
```
selftest（离线，注入假响应）：`saved_at` 未过期但探针返登录页 → `_verify_live` 判 False → 触发重登路径。

## 五、护栏 / 合规（不变 + 真源码补强）

1. **仅合法权限**：只对你**确有机构订阅**的刊；A5 = 以合法身份走正门，**不破付费墙本身**。
2. **人工登录**：SSO 人工完成，**不自动填凭据**（避违 IdP 条款）；cookie 只落本地、600 权限、可过期 + `verify_live`。
3. **默认关**：`institutional=False`；无 cookie store/凭据 → 优雅跳过、`applicable()` 返 False。
4. **落盘必过内容 QC 门③**（真源码 delta-4 佐证）：机构正门下的 PDF 同样要核 page-1 DOI＝期望 DOI，防"抓错刊/SI/顶包"。
5. **礼貌**：机构出口同样低并发（route-B 全局 concurrency=1、按 origin 缓存会话，见 route-B-150 §四）。
6. **凭据 gate**：无凭据 → ROI=0、永不联网；代码就绪，凭据到手照 `路线A-机构订阅实测Runbook` 3 步。
7. **MASTER-hash 库 / immutability（chmod 444）** 登记不引入：本仓已有标准化命名 + `metadata.jsonl`，SciTeX 的"canonical 只读 + 注释副本旁挂"思路仅登记。
