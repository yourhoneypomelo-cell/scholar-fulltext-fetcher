## 3. 实现原理与路径

### 3.1 北极星与主线判断
输入 DOI/标题 → 榨干全网可及**免费**全文 → 校验下载入库；高成功率 + 高速率 + 全程程序化（"运行 + 读日志"即可迭代）。核心判断：**绕开 Scholar 人机验证、走开放获取正门**（详见 `实现路径与原理.md` §1、总报告 §1.8）。

### 3.2 全链路
`输入 → resolve 判类型+元数据富化 → Paper → pipeline 按源优先级逐源定位 → 候选按置信度即时下载、首个成功短路 → 落地页兜底 → download 校验落盘 + 内容 QC 门 → 三类结构化日志（attempts/metadata/summary.jsonl(json)）`。

### 3.3 关键设计决策（原理）
| 决策 | 要点 | 出处 |
|---|---|---|
| 源优先级 + 首链短路 | `DEFAULT_SOURCE_ORDER` 按命中率/速度/稳定性排序，第一个成功直链即停 | `实现路径与原理.md` §4.1 |
| 候选置信度排序 | 直链(pdf/render) > 落地页；先试高分直链 | §4.2 |
| %PDF 魔数校验 | HTTP200 + 非 text/html + 前1KB 含 `%PDF` + 大小达标，绝不把 HTML 落地页当 PDF | §4.3 |
| 按域限速 + 退避 + 熔断 | 每 host 最小间隔、429/5xx 指数退避尊重 Retry-After、连续连接错熔断 | §4.4 |
| 元数据富化 | OpenAlex 单条(按 DOI 免费)一次拿 title/pmcid/arxiv_id，减少各源重复定位 | §4.5 |
| 断点续跑 | 状态进 metadata.jsonl，重跑只补失败项 | §4.6 |

### 3.4 内容 QC 门①–⑤（防"下到错论文"假阳，生产精度核心）
落盘前每份 PDF 过 `download._content_qc_gate`（需 `[qc]`：pypdf+rapidfuzz；缺依赖默认 **fail-closed** 拒收）。**双门 union + 非正文门 + 元数据门**，判 mismatch→不落盘、uncertain→放行打标、match→落盘：

| 门 | 判据 | 治什么 |
|---|---|---|
| 强正① | 期望 DOI 出现在 URL 或**正文/摘要区**（仅参考文献区命中不算，位置权重见经验记录 T.6/149） | 有正证据即 match |
| 门① content-title-mismatch | 能抽出正文且标题分 < 阈 | 明确他题 |
| 门② url-doi-conflict | URL/正文首部嵌另一家出版商或另一 DOI | title 假匹配、未来年份 DOI |
| **门③ meta-doi-mismatch（-143 落地）** | **PDF 自述元数据 DOI（XMP `prism:doi`/`dc:identifier` + `/Info` *doi* 键）全部 ≠ 期望且期望不在其中** | **"同题他刊"**（标题近同、期望 DOI 不在正文首部，如 `cctc.200900261`） |
| 门④⑤ non-article | SI / citation-report / poster / 卷期目录等非正文版式 | 首页印对标题+DOI 却非正文 |

门③ 精度纪律（-143 抽样实证）：只在**强正①未命中**时触发；元数据含期望/无自述 DOI → 不触发；**只取 PDF 自述身份字段、不扫整段 XMP**（否则会误抽参考文献 DOI 而误杀）。运维要点见 `用户Runbook…§5.4b`、经验记录 **W.4**、回归 `tools/regress_qc_union_189`。

### 3.5 route-B（过 CF 软盾 + 浏览器内直下）与 A5（机构订阅）
- **route-B**（`render_fetch.py` + `download._browser_capture_fallback`，`--route-b cf-only|all`，默认 off）：nodriver 真 Chrome 会话内过 Cloudflare "Just a moment" 等**非交互**质询，过盾后**优先 B1 页内同源 `fetch()`**、跨源回落 **B2 过盾后门控 Fetch.enable**（不在过盾期开 CDP 拦截——这是关键铁律，见 N.8）。**只过软盾、硬墙（RSC governor / JA3 回放 / 交互式 Turnstile）到顶**（T.3/S.2）。全组共一机单头浏览器（`out/.route_b.lock` + concurrency=1 硬护栏）；落盘前强制过内容 QC（`force=True`，W.2）。
- **A5 机构订阅**（`institutional/`，`--institutional` + 凭据，默认关）：合法凭据走 EZproxy/SSO 认证通道，**机构 Cookie 注入 nodriver → 复用 route-B 页内 fetch 直下**，与 CF 过盾链焊成同一条 A5 链。**路线A**（HTTP EZproxy 改写 / `publisher_direct`）+ **路线B**（浏览器内注入，-143 已补齐 `injection_plan` 断线，W.1）。**唯一 gate = 用户真实凭据**；无凭据恒零副作用（`needs_institution_access` 守卫）。就绪核对见 `A5即跑清单-189.md`、`A5联调就绪核-…-177.md`、冒烟输入 `a5_smoke_input.txt`。

### 3.6 coverage 口径（诚实净覆盖）
`tools/build_coverage.py` 扫各批 `metadata.jsonl` + 盘上 PDF + 消费 QC 黑名单 → `out/coverage.json`。**success = metadata.success AND pdf 文件在盘 AND 不在 QC 黑名单**。当前：success_before_qc 558 − rejected 190（硬30+软160）+ allow_override 60 = **success_after_qc 368**。口径纪律见经验记录 **T.2/U.3/V.1**、`基线口径冻结说明-388-173.md`。

## 4. 全功能使用手册

### 4.1 安装（依赖档位，按需装）
| 命令 | 增加 | 用途 |
|---|---|---|
| `pip install -e .` | `requests>=2.28` | 最小可跑（多源 OA 定位 + HTTP 直下） |
| `pip install -e ".[qc]"` | `pypdf` `rapidfuzz` | **内容 QC 门（生产强烈推荐）** |
| `pip install -e ".[cf]"` | `curl_cffi` | CF/TLS 指纹增强 |
| `pip install -e ".[xlsx]"` | `openpyxl` | 读 `.xlsx` 输入表 |
| route-B/A5 | `nodriver`（+ 有头显示环境） | 浏览器内抓字节 / 机构注入（缺则优雅 no-op） |

### 4.2 一键正门 `run_all.py`（无人值守批量，首选）
```powershell
python run_all.py -f dois.txt -o out/my_batch --email 你@xx.edu
```
关键参数（全量见 `run_all.py build_parser` / `用户Runbook`）：
- 输入：位置参数（多个）或 `-f/--input-file`（.txt/.csv/.xlsx）。
- `-o/--out` 独立 RUNROOT（**并发批量务必用不同 -o**，防写锁撞车，见 I.1/V.2）；`-c/--concurrency`（默认 3）；`--timeout`；`--resume/--no-resume`。
- API key：`--openalex-key/--s2-key/--core-key`，默认经 `cli._env_first` **环境变量别名回退**（canonical `OPENALEX_KEY/S2_KEY/CORE_KEY` 优先，兜底 `OPENALEX_API_KEY/SEMANTIC_SCHOLAR_API_KEY/S2_API_KEY/CORE_API_KEY`，裁空白）——本机把 key 存成通用别名也能自动生效（b892eca）。
- QC：默认开；`--no-qc` 仅调试（净成功率虚高）；`--qc-hard/--qc-soft` 指定黑名单；`--coverage-root`（默认 out）。
- A5：`--institutional` + `--ezproxy-prefix` / `--institution-cookie` / `--institution-domain`（可多次）。
- route-B：`--route-b off|cf-only|all`（默认 off）；`--browser-headless`；`--browser-pdf-wait`。
- 自证/排障：`--verify`（确定性复现自报数）、`--explain <DOI>`、`--selftest`（离线自检→RUN_ALL_OK）。
- 产物：`RUNROOT/fetch/`（pdfs + 三类日志 + report.html + results.csv）；覆盖率写 `RUNROOT/coverage.json`（注意：全局 `out/coverage.json` 是人工 extra_dirs 汇总产物，一键默认口径可能与之不同，见 V.1）。

### 4.3 模块 CLI（单条/小批，拿 JSON）
```bash
python -m fulltext_fetcher "10.7717/peerj.4375" --email 你@xx.edu           # 单条
python -m fulltext_fetcher -f dois.txt --email 你@xx.edu -c 6                # 批量
python -m fulltext_fetcher "标题…" --print-json                              # stdout 得 {summary,results}
```
日志/提示走 stderr/文件，不污染 stdout；退出码 0=至少一条成功 / 1=全失败 / 2=用法错。

### 4.4 编程 API（同进程嵌入，拿结构化对象）
```python
from fulltext_fetcher import fetch_one, fetch_many, FullTextFetcher, Config
r = fetch_one("10.7717/peerj.4375", email="you@uni.edu", out_dir="out")
with FullTextFetcher(email="you@uni.edu", concurrency=8) as ft:
    results, summary = ft.fetch_many(["10.x/y", "Attention is all you need", "1706.03762"])
```
`FetchResult` / `Config` 全字段 + 子进程契约见 `接入与交接说明.md`；`Config` 全字段见 `fulltext_fetcher/config.py`（含 QC/route-B/A5/命名模板等开关，均有行内注释）。

### 4.5 谷歌学术爬虫子系统（正面抓 Scholar，可选）
`python -m fulltext_fetcher.scholar "标题或DOI" [--serpapi-key … | --proxy | --oa-fallback]`；默认最安全档（curl_cffi→nodriver，代理/打码默认关）。**数据中心 IP 直连 Scholar 几乎必 403**，真实抓取需 SerpApi key 或住宅代理。详见 总报告 §1.7、`谷歌学术爬虫-架构与选型.md`、经验记录 F 节。

### 4.6 自检 / 回归（改码后必跑）
- 全套离线：`python run_all_selftests.py` → `PASS=47 FAIL=0 SKIP=2`。
- 数据回归（改 QC 门后必跑）：`RUN_DATA_REGRESS=1 python -m tools.regress_qc_union_189` → `REGRESS_UNION_189_OK`（含门③ 负/正样本 + 189 同域桶并集全拒）。
- 联网自检（可选）：`RUN_ONLINE_SELFTESTS=1`（需真浏览器 + 出网）。
