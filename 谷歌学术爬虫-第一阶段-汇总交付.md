## 三、一页决策速览（承 R1/R2/ARCH）

- **正门优先**：有 `SERPAPI_KEY` 走商业合规 SerpApi（`mode=auto`→有 key 即 `serpapi`，否则 `self`）。
- **自建取回分层降级**：`curl_cffi`(L1 静态·首选最省) → `nodriver`(L3 浏览器·AGPL 内部用) → 备选 `patchright`/`SeleniumBase`；**翼A** 住宅代理（默认关=直连）、**翼B** 打码（默认关）。
- **限速偏保守**：页间 **45–90s** 抖动、每 **5–15** 次换 IP/会话、命中风控冷却 **900s**、默认**串行**（`concurrency=1`）。
- **captcha 默认关**：500 样本失败几乎全是付费墙 403 / 无 OA 候选，非 reCAPTCHA（R2）；仅「直抓 GS 拿被引/版本原生字段」才需要。
- **PDF**：先用结果自带 PDF 直链；未果**复用父包 OA 源**（Unpaywall/OpenAlex/S2/…）兜底；Sci-Hub 默认关。
- **许可证**：`curl_cffi` MIT / `nodriver` AGPL（内部研究可，对外服务须法务）/ `DrissionPage` 非商用（不作默认依赖）；**可选依赖一律函数内延迟导入、不进强制依赖**。

## 六、构建阶段并行 DAG（供总指挥编排，源自 ARCH §七）

```
Phase 0（地基·阻塞全部）  P0 models + config（冻结契约）+ logsetup（复用父包）
Phase 1（5 路并行·仅依赖 P0）  query · serp（含 parse_serpapi 迁移 B1） · naming · proxy · captcha
Phase 2（2 路并行·依赖 P1）    fetcher（反爬核心） · download（复用父 download/landing/render_fetch/sources）
Phase 3（集成·串行）           pipeline → cli + __main__
Phase 4（验收）                selftest_e2e（不联网 e2e）→ 全子包 compileall + 各 *_OK
```

约 **8 路**可同时开工：P0 单人先行并广播契约；P0 完成后 P1（5 人）并行 + P2（2 人预读依赖/搭骨架）。

## 八、合规边界（每模块头部必写，源自 ARCH §八）

1. **优先正门**：能用官方开放 API / 商业合规 SerpApi 就不自建直抓；自建栈仅供研究/小规模自用。
2. **默认最安全**：代理、打码、Sci-Hub 默认全关；只「避免触发」不主动对抗；限速/退避/冷却从严。
3. **ToS/robots**：直抓 Google Scholar 属灰色，违反其 ToS 与 robots.txt，使用者自负；对外服务须过法务（含 `nodriver`=AGPL、`DrissionPage`=非商用 等许可证传染）。
4. **付费墙**：不在本子包强求；仅在用户拥有合法权限时经既有 OA/机构通道兜底。
