## 一、现状盘点(as-is，供接线定位)

### 1.1 `render_fetch.py`(能力体，-144 在改)
- `render_download_pdf_bytes(article_url, timeout, *, min_interval, headless, _capture_fn)` — 路线B 主体：过 CF → **B1 页内 `fetch()` 抓字节**（同源最稳，如 ACS-OA）→ B1 失败 →**B2 导航 PDF 直链 + `Fetch.enable` RESPONSE 拦截**（绕 CORS/CSP，如 RSC articlepdf）。默认关（未装 nodriver 即 `{"available": False, "reason": "need nodriver"}`）。
- `is_ja3_bound_cf_host(url)` — 门控纯函数：`_JA3_BOUND_CF_HOSTS = {pubs.rsc.org, rsc.org, sciencedirect.com, sciencedirectassets.com, onlinelibrary.wiley.com, pubs.acs.org}`。
- 合规守卫 `_is_scholar_host`（永不渲染 Scholar）、`_throttle`（进程内最小间隔限流）、`%PDF` 校验、CDP 兼容 monkeypatch（Chrome 133 `localNetworkAccessRequestPolicy` KeyError）。
- **-152 实证**：ACS-OA `10.1021/acsomega.6c04195` CF 6.6s 过、B1 拿 13.7MB %PDF、QC match；RSC `10.1039/d5ra08493h` CF 12–16s 过、需 B2。**根因翻案**：早期"CF 过不了"实为**过盾期开着 `Fetch.enable` 被 CF 探测卡死** → 修法=过盾期绝不开拦截、以 `cf_clearance` cookie 为过盾信号、过盾后再按 B1→B2。**无需升 Chrome**。

### 1.2 `download.py` 分层兜底链(route-B 已在其中，默认关)
```
① _download_pdf_core(基础下载 + 落地页解析)
② is_cf → _flaresolverr_fallback(FlareSolverr 解 CF 质询, gated)
②b JA3 → _browser_capture_fallback → render_download_pdf_bytes   ← route-B(页内直下)
③ not cf → _curl_cffi_fallback(impersonate)
④ not cf → _publisher_fallback(publisher_adapter)
⑤ _render_fallback → render_get_pdf_url(仅 OA 落地页渲染, gated)
⑥ _browser_pdf_download(有头过 Akamai/MDPI 并 CDP 下载, gated)  ← route-B(Akamai)
```
- ②b 门控：`_browser_capture_enabled(cfg)`（`cfg.browser_capture` 或 `FTF_BROWSER_CAPTURE=1`）+ `is_ja3_bound_cf_host(url)`。
- ⑥ 门控：`cfg.browser_pdf_download`；`browser_pdf_headless`（默认有头）、`browser_pdf_wait`。

### 1.3 软开关现状(**关键缺口**)
`browser_capture / browser_pdf_download / browser_pdf_headless / browser_pdf_wait / render_fallback` **均非 `Config` 正式字段**，全靠 `getattr(cfg, ..., default)` 读取（`browser_capture` 另可经 `FTF_BROWSER_CAPTURE` env）。**CLI 未暴露任何 route-B 开关**。→ 目前只能靠 env 或编程 override 启用，不利"默认能力"。

### 1.4 `pipeline.py` 现状
- `_browser_fallback_active()` 已识别这些开关 → 放大 straggler"无进展"阈值(≥900s)，防慢但合法的 CF 尾部被误杀。
- **无 concurrency=1 护栏**：`cfg.concurrency` 默认 4，线程池并发跑；若多 worker 同时命中 CF 域 → 多个有头浏览器并起 → 争用（-144 实测过的"多 shim 重载争用"同类问题）。

## 二、接线点设计(Deliverable ①)

**原则：route-B 是 download 层"最后一层兜底"，不是 source；只在便宜手段(OA 源 + curl_cffi + FS)全败、且 host 属 CF/Akamai 时触发。** 现有分层链位置**已正确**（②b JA3、⑥ Akamai），本方案**不移动位置**，只改"默认是否触发"。

- **JA3 绑定 CF 桶(acs 10.1021 / rsc 10.1039 / wiley 10.1002 / aip 10.1063 / T&F 10.1080 / OUP 10.1093 / sciencedirect)**：走 ②b `_browser_capture_fallback`（页内直下 B1→B2）。
  - 建议把 aip/T&F/OUP 的 host 补进 `_JA3_BOUND_CF_HOSTS`（当前只含 rsc/sciencedirect/wiley/acs）——**代码活归 -144/-153**，本文只标注需补域。
- **Akamai 桶(MDPI 10.3390)**：走 ⑥ `_browser_pdf_download`（有头过 Akamai Bot Manager）。
- **触发条件（默认开后）**：`_download_pdf_core` 返回失败且 `is_cf`（或 Akamai 域）→ 依次 FS → route-B。DOI-前缀/host 不在 CF/Akamai 名单 → 不触发（普通 OA 零副作用）。
- **与 route-A(机构订阅)协同**：A5 授权会话 + route-B 页内直下应合流为"授权会话内 CDP 抓字节"（见 A5 骨架文档）；本方案先落无凭据的 OA/CF 直下，A5 叠加另议。

## 三、默认开关设计(Deliverable ②)

### 3.1 形式化为 Config 字段(替代 getattr 软开关)
建议在 `config.py` 新增(默认值即"CF 域默认开、其余关"策略)：
```python
# 路线B(浏览器内直下 PDF, 破 JA3-CF / Akamai) —— 默认仅对 CF/Akamai 域启用
route_b_enabled: str = "cf-only"     # off | cf-only(默认) | all
browser_capture: bool = True         # ②b JA3 页内直下(cf-only 时仅对 is_ja3_bound_cf_host 生效)
browser_pdf_download: bool = True    # ⑥ Akamai 有头下载(cf-only 时仅对 Akamai 域生效)
browser_pdf_headless: bool = False   # CF/Akamai 有头通过率更高, 默认有头
browser_pdf_wait: float = 13.0
render_fallback: bool = False        # ⑤ 仅 OA 落地页渲染, 与 route-B 正交, 维持默认关
route_b_concurrency: int = 1         # 见 §四, 写死 1
```
- `route_b_enabled="cf-only"`（默认）：`browser_capture`/`browser_pdf_download` 虽为 True，但**只对 `is_ja3_bound_cf_host` / Akamai 域生效**（域门控在 download.py 已有），普通域一律不触发 → 默认零副作用面不变。
- `off`：完全回到当前"默认关"行为（逐字节一致）。
- `all`：对任意 host 都允许 route-B 兜底（省资源原则下不建议默认）。
- **向后兼容**：`FTF_BROWSER_CAPTURE` env 保留为强制开/覆盖。

### 3.2 CLI 暴露(cli.py，-153/-144 落地)
`--route-b {off,cf-only,all}`（默认 cf-only）、`--browser-headless/--no-browser-headless`、`--browser-pdf-wait N`。凭据类不涉及；有头需真实显示环境（见 §五/§六）。

### 3.3 与 sources 优先序共存
- route-B **不进 `cfg.sources`**（它不是候选发现源，是 download 层兜底）；sources 仍先跑 OA 免费源产候选，候选下载失败且 CF/Akamai 才落到 route-B。**不改 sources 优先序**。
- 省资源：route-B 每篇 ~15–45s 有头浏览器开销，故**仅 last-resort**、仅 CF/Akamai 域、且受 §四 串行护栏。

## 四、并发护栏(Deliverable ③ — 防重现争用)

**约束：全组共一台机器、单头浏览器；route-B 必须串行(concurrency=1)。** 现状 `cfg.concurrency=4` 线程池会并起多浏览器 → 争用/CF 判定恶化。

设计**两层锁**（代码活归 -144/-153，本文给方案）：
1. **进程内**：在 `render_fetch.py` 加模块级 `threading.BoundedSemaphore(1)`（或复用/升级现有 `_throttle_lock`），包住 `render_download_pdf_bytes` 与 Akamai 抓取的浏览器启动→抓字节→关闭全过程；同进程任意并发调用自动排队。**与现 `_throttle`(最小间隔)叠加**：先取信号量再限流。
2. **跨进程**：多 PChat 会话可能各起一个 fetch 进程共用一台机。用**文件锁**（`out/.route_b.lock`，仓内已有 `fetch/.lock` 先例）做 advisory 互斥：持锁者跑 route-B，未取到锁的**降级跳过 route-B**（记 `route-b-busy`，不阻塞其余便宜兜底）。避免两进程同时弹有头 Chrome。
3. **pipeline 侧**：当 `route_b_enabled != "off"` 时，`_browser_fallback_active()` 已放大 straggler 阈值(≥900s)——**保留**；另建议对"命中 route-B 的条目"在调度上不计入并发拖累判断（避免看门狗误杀等锁的条目）。

> 关键：**concurrency=1 是硬约束**，不因 `cfg.concurrency` 变化——route-B 域的下载在信号量/文件锁下天然串行，其余非 CF 域仍可 4 并发正常跑（互不影响）。

## 五、回归测试点(Deliverable ④)

**问题：现 `render_fetch._selftest` 全程 mock `_render_fn`/`_capture_fn`，永远测不出"CF 期开 Fetch.enable 被卡死"这类真机 bug**（-152 正是 offline 全绿但真机 0/2）。

设计一个**门控有头冒烟 selftest**（默认 SKIP，仿 `run_all_selftests.py` 的 `RUN_ONLINE_SELFTESTS`）：
- 新增 `RUN_ROUTE_B_SMOKE=1` 触发；入口如 `tools/route_b_smoke.py` 或 `render_fetch --route-b-smoke`。
- **两条已核实金 OA 样本**（-152 实证过）：
  - B1 路径：`10.1021/acsomega.6c04195`（ACS Omega 金OA）→ 断言 `%PDF` + 内容 QC `match`（DOI-in-text / 标题命中）。
  - B2 路径：`10.1039/d5ra08493h`（RSC Advances 金OA，articlepdf 需 B2）→ 断言 `%PDF`。
- **断言**：`pdf_bytes[:4]==b"%PDF"` 且 size 合理 且 `tools/qc_content_match.classify` 非 mismatch；CF 未过要落 `blocked:challenge-page`（不得被误判 `no-pdf-captured` 掩盖）。
- **接入 `run_all_selftests.py`**：加一行 `("route_b_smoke", [...], gated by RUN_ROUTE_B_SMOKE, "ROUTE_B_SMOKE_OK")`，默认 SKIP、不影响 CI 绿。
- **配套修 `_BLOCK_SIGNALS`**（-152 报的 bug，归 -144）：补 "performing security verification" / "verifies you are not a bot" / "needs to review the security of your connection"，并增查 `document.title`——否则 CF 未过会被误判，冒烟结果失真。
