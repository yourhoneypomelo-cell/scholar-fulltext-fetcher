## 4. 可复用点（尤其认证处理）——按「拿来即用」拆解

### 4.1 auto-paper-harvester（最高价值，整体蓝本）
- **三层路由与本项目 route-A/B 一一对应**：
  - `full`(TDM API)：Wiley、Elsevier → 对应本项目订阅直链，可升级为官方 API
  - `oa_only`：Springer Nature/Nature Portfolio/BMC
  - `partial`：PNAS、Beilstein
  - `browser_only`(需 `--use-browser-fallback`)：**ACS、RSC、AAAS/Science、ECS、IOP、AIP、AVS、IEEE、APS、Annual Reviews、T&F、Optica/OSA、KPS**
- **可复用**：① `browser_fallback.py` 复用机构 SSO cookie 的 Playwright 流程（≈ 本项目 `route_b_bridge.py` + `render_fetch.py`）；② **14 出版社族 per-publisher CSS selectors**（ACS/Wiley/RSC/T&F/Nature/AIP/CN 期刊 落地页 PDF 提取）；③ **24 条 DOI 前缀 → 19 出版社族路由表**；④ failed-DOI 结构化残留摘要（≈ 本项目分桶统计）。
- **合规点**：官方 README 原文「This is not a paywall bypass — it relies on your real entitlements」，与本项目 `publisher_direct.py` 合规声明一字不差的同款立场。

### 4.2 metapub / FindIt（直链定位逻辑，已被本项目采用过）
- FindIt 的 publisher-specific「dances」= 每社定制的 PDF URL 构造/跳转算法，**本项目 `publisher_direct.py` 第 88–95 行已注明「metapub FindIt 风格扩展」**并落地了 Atypon 系（Physiology/AHA/AnnualReviews/Liebert/INFORMS/SIAM）。
- **可复用**：用其 registry 继续补齐本项目尚未覆盖的前缀（IEEE 10.1109 / IOP 10.1088 / AIP 10.1063 / Optica / ECS 等）；其 embargo 检测 + `verify` HTTP 校验思路可与本项目 `%PDF` 魔数校验叠加。

### 4.3 官方 TDM 客户端（认证处理的「正道」）
```python
# Wiley 官方（wiley-tdm）：token + 机构 IP 认证
from wiley_tdm import TDMClient
tdm = TDMClient()               # 读环境变量 TDM_API_TOKEN
tdm.download_pdf("10.1111/jtsb.12390")

# Springer 官方（springernature_api_client）：API key（免费申请）
import springernature_api_client.tdm as tdm
c = tdm.TDMAPI(api_key="...")

# Elsevier 官方（elsapy）：API key + 机构订阅
from elsapy.elsdoc import FullDoc   # 按 DOI/PII 取全文
```
- **认证处理要点**：Wiley=TDM token+IP；Elsevier=API key(+机构 token)；Springer=API key。三者都是**出版商授权的合规通道**，比「EZproxy/SSO 直链」更不易被 bot detection 拦，且不涉及 JA3/CF。

### 4.4 paperscraper / PyPaperRetriever / paper-fetch（OA & 兜底）
- paperscraper：内置 **Wiley TDM + Elsevier TDM** 封装 + fallback（BioC-PMC XML / eLife XML / bioRxiv S3）；诚实标注「cannot bypass publisher restrictions」。
- PyPaperRetriever：Unpaywall + NIH Entrez/PMC + Crossref，**SciHub 默认关**（`allow_scihub=False`），解决 Unpaywall「green OA lag / link 不可 bot 取」两类漏网。
- paper-fetch：**纯 stdlib 零依赖** + `%PDF` 魔数校验 + SSRF 防御 + 50MB 上限（与本项目 `download.py` 校验思路一致，可作轻量参考）。

## 5. 集成建议（结合本项目 `fulltext_fetcher/sources/` 体系）

本项目源体系已很成熟：`BaseSource.find_candidates()` → REGISTRY → 按 `cfg.sources` 顺序编排，现有 `publisher_direct`（订阅直链，`--institutional` 开）/ `publisher_oa` / `green_oa` / `aggregators` / `oa_button` / `scihub` 等。建议：

1. **新增 `sources/publisher_tdm.py`（合规官方 TDM 源）**：`--institutional` 下，对 Wiley(10.1002/10.1111)/Elsevier(10.1016)/Springer(10.1007…) 优先走 `wiley-tdm`/`elsapy`/`springernature_api_client` 官方 API（token/key 从 `credential_store` 读），**排在 `publisher_direct` 直链之前**——官方通道命中率与稳定性更高，且规避 bot detection。
2. **用 metapub FindIt registry 扩充 `publisher_direct.py` 前缀表**：补 IEEE/IOP/AIP/Optica/ECS 等尚缺前缀（延续第 88–95 行已有的「FindIt 风格」注释与实现）。
3. **用 auto-paper-harvester 的 per-publisher selectors 补强 `render_fetch.py` / route-B**：把 ACS/RSC/T&F 落地页的 PDF 抽取 CSS selector 并入渲染兜底，提升 `browser_only` 社的落地成功率。
4. **OA 层对照补漏**：以 paperscraper/PyPaperRetriever 的「Unpaywall→PMC→Crossref 交叉校验」补 `green_oa`/`aggregators` 的 green OA lag 兜底。
5. **依赖纳管**：将 `metapub`、`paperscraper`、`wiley-tdm` 作为**可选 extra** 评估纳入 requirements（避免强依赖膨胀）。
6. **合规红线不变**：所有 TDM/直链均以**真实机构订阅授权**为前提，保留 `download.py` 的 `%PDF` 魔数校验防假成功——与项目现状完全一致。

## 6.5 【本波重点·方向③】谷歌学术可达 PDF 最快最全程序化获取方案

> 响应总指挥方向同步（准则⑥）：用户本波重点③=『谷歌学术可达 PDF 链接』最快最全程序化获取。本节专述，供 n7 汇总直接采用；对齐『可直接用 or 吸收整合』+ 可程序化落地。

### 核心判断（先说结论）
「谷歌学术可达的 PDF」绝大多数是 **OA / 仓库 / 预印本 / 作者自存**的 PDF。**用 DOI 走免费 API 多源并发拿，比真去爬 Google Scholar 更快更稳**——Scholar 本身反爬极强（Google WAF+CAPTCHA，直连 5–15 次即封，需住宅代理，归口 140），应只作为「无 DOI 长尾」补充，而非主路径。

### 来源分层（"快"→"兜底"）
| 层 | 源 | 程序化取 PDF 字段/端点 | 成本 | 反爬 |
|---|---|---|---|---|
| ① 免费 API 三件套（主路径） | Semantic Scholar | `GET /graph/v1/paper/DOI:{doi}?fields=openAccessPdf` → `openAccessPdf.url` | 免费·1 req/s | 无 |
| | Unpaywall | `best_oa_location.url_for_pdf`（需 email） | 免费 | 无 |
| | OpenAlex | `open_access.oa_url` / `best_oa_location`（**2026-02 起需免费 key**） | 免费(拿链接) | 无 |
| ② 标识符直链 | arXiv / PMC / bioRxiv | `arxiv.org/pdf/{id}`；PMC OA；externalIds 来自 S2 | 免费 | 无 |
| ③ 出版社直链 | metapub FindIt / 本项目 `publisher_direct` | 68+ 社 DOI→PDF 直链 | 免费 | 视社 |
| ④ Scholar 原生（长尾补充） | scholarly | `bib['eprint_url']`（Scholar 右侧[PDF]）/ `pub_url` | 免费 | **强（需代理，归口 140）** |
| ⑤ 浏览器兜底 | 本项目 route-B | 渲染 + 复用 cookie | 贵 | route-B 专攻 |

> OpenAlex 坑点：**别用 `locations[].pdf_url`**（publisher 原始 URL，lossy）；拿链接用 `open_access.oa_url`/`best_oa_location`，批量下载正文用 content API（`content.openalex.org/works/{id}.pdf?api_key=`，$0.01/篇，`has_content.pdf:true` 过滤）。

### "最快" = DOI 命中即并发三件套
```python
# 伪代码：DOI → 最快 OA PDF（免费三源并发，取首个 %PDF 有效）
candidates = concurrent([
    s2_openaccesspdf(doi),         # api.semanticscholar.org …fields=openAccessPdf → openAccessPdf.url
    unpaywall_best_oa(doi, email), # best_oa_location.url_for_pdf
    openalex_oa_url(doi, api_key), # open_access.oa_url（2026-02 起需免费 key）
])
pdf = first_valid_pdf(candidates)  # %PDF 魔数校验（本项目 download.py 已有）
```
单次 API 往返、毫秒级、不碰浏览器——这是"最快"。

### "最全" = 多源 fallback 链并集去重
`Scholar eprint → S2 openAccessPdf → Unpaywall → OpenAlex oa_url → PMC/arXiv → publisher direct(FindIt) →（最后）route-B 浏览器`。命中即止、失败降级、结果并集去重。

### 可直接用 / 吸收整合（对齐准则(b)）
| 方案 | 判定 | 说明 |
|---|---|---|
| **paper-search-mcp** | **可直接用** | `download_with_fallback`，19+ 源（含 S2/OpenAlex/Scholar/Unpaywall/PMC/CORE/OpenAIRE），OA-first；最全现成一站式，MIT，2026-07 活跃 |
| **paper-fetch** | **可直接用(轻量)** | 7 源 fallback 纯 stdlib 零依赖 + %PDF 校验；易嵌入，注意关掉默认 SciHub |
| **scholarly** | **吸收整合** | 仅取其 Scholar 解析（`eprint_url`/`pub_url`），作"谷歌学术可达"长尾补充源（过 Scholar 反爬归口 140） |
| **S2/Unpaywall/OpenAlex 三件套** | **吸收整合** | 直接补进 `sources/aggregators.py`/`green_oa.py`，免费高命中，是主路径 |
| **metapub FindIt** | **吸收整合** | publisher 直链兜底（`publisher_direct` 已借鉴） |
| **pyalex** | 可选 | OpenAlex 官方风 Python 客户端，`w.pdf.url`/`w.pdf.download()`，简化 OpenAlex 接入 |

### 可程序化落地路径（结合本项目 `sources/`）
1. 在 `sources/aggregators.py` 确保覆盖**免费三件套并发**：S2 `openAccessPdf` + Unpaywall `url_for_pdf` + OpenAlex `open_access.oa_url`（取并集、%PDF 校验去重）——这是「谷歌学术可达 PDF」最快最全**主路径**。
2. 复用/新增 `scholarly` 适配器抽 Scholar `eprint_url`，仅对**无 DOI 或前面全 miss**的长尾启用，走代理（归口 140）。
3. 出版社直链兜底沿用 `publisher_direct.py`（FindIt 风格），浏览器兜底沿用 route-B。
4. 全链路统一 %PDF 魔数校验（`download.py` 已有），防假成功。

> 一句话：**「谷歌学术可达 PDF 最快最全」= DOI 免费三件套（S2+Unpaywall+OpenAlex）并发 + FindIt 出版社直链兜底 + scholarly 长尾补充**；真爬 Google Scholar 非主路径（贵且需代理）。
