## 一、镜像站路线（中文圈最主流的"绕过访问墙"方式）

镜像站把「反爬/过人机检测」这件事**转嫁给镜像运营方**：运营方用反向代理 + IP 池 + 缓存伪装成普通浏览器去抓 Google Scholar，用户只面对一个看似普通的中文站点。

| 站点 | 形态 | 能力 | 备注/风险 |
| --- | --- | --- | --- |
| 烂番薯学术 `scholar.lanfanshu.cn` | 反向代理中转 | 实时抓 GS 元数据（标题/作者/摘要/DOI/被引），支持 GB/T 7714 + APA 引用导出、作者追踪、时间排序 | 前端 Vue3+Element Plus，后端 Node.js+Express，用 `cheerio` 解析 GS 的 HTML；内置随机 User-Agent 轮换 + 请求间隔控制规避反爬；**闭源、匿名运维（黑箱）** |
| 囧蒜学术导航 `jiongsuan.com` | 镜像导航/聚合 | 汇总多个镜像入口，给出「查→下→管」标准链路 | 查（烂番薯）→ 下（Sci-Hub 直链，凭 DOI）→ 管（Zotero/CNKI） |

### 镜像站通用机制
- **反向代理 + 缓存**：用 Redis/Nginx 缓存热门关键词结果（约 24–72h），降低被 GS 封 IP 的概率、提升速度。
- **二次整合**：部分镜像自带爬虫，整合 Scopus / Web of Science / CNKI 元数据，因此中文文献结果常比官方多。
- **部署**：服务器多在香港；伪装成普通浏览器访问 GS 主站以规避检测。
- **自建私有镜像**（howtogu.com 2026 实战指南）：可用 **Docker + Nginx** 在云服务器自建，两种类型——
  1. **反向代理型**：Nginx 直接代理真实 Scholar（需要能访问 Google 的网络环境）。
  2. **前端渲染/聚合型**：爬虫预抓元数据入本地库，用户查的是本地库（不依赖特殊网络，但拿不到最新文章）。
  - HTTPS 连接成功率显著高于 HTTP。

### 合规与风险
- Google Scholar 的 `robots.txt` 明确禁止任何自动化工具抓取，**镜像站本质处于灰色地带**。
- 风险：篡改元数据/BibTeX、记录搜索历史、植入广告或恶意脚本、PDF 链接下到无关内容/病毒。
- 对策：① 选口碑好、更新勤的镜像；② 下载后核对 PDF 标题/作者/摘要与检索结果一致；③ 站内**不登录、不输入敏感信息**；④ 有条件用机构 VPN 核验。

## 三、中文社区（CSDN / 知乎）反 reCAPTCHA 方法论

**reCAPTCHA 版本**：v2（"我不是机器人"复选框 + 图片）、v3（无感行为评分 0–1，阈值常 0.5）、Enterprise。

**检测维度**：浏览器指纹（UA / WebGL / Canvas / 字体）、行为（鼠标轨迹 / 点击 / 停留）、网络（IP 信誉 / 请求频率 / TLS 指纹）、自动化（`navigator.webdriver`）、环境（插件 / 分辨率 / 时区）。

**分层对抗（中文社区共识）**：
1. **环境伪装（指纹）**：Playwright + `playwright-stealth`，修掉 `navigator.webdriver`、对齐 WebGL/Canvas、UA 随机化、语言/时区对齐，**禁用 headless**。→ v2 复选框 60–70%。
2. **住宅代理轮换（决定性）**：用住宅代理而非数据中心代理，每 5–15 次请求换 IP，配 45–90s 抖动延迟。→ 提升 10–20%。
3. **行为模拟**：点击/滚动间插 200–800ms 随机停顿，鼠标走非直线人类曲线。→ v2 图片 80–90%。
4. **第三方打码兜底**：2Captcha / Anti-Captcha / Scrapeless / Bright Data Web Unlocker，提交 site-key 拿 token 自动回填。→ 95%+（付费）。
- **v3 专项**：养 IP/账号信誉让评分稳定 0.7+，或 token 复用（70–80%）。

## 四、对工作组目标的结论与建议

- **镜像站路线**：把反爬转嫁给运营方，适合**人工/低量**地取元数据 + 经 Sci-Hub 下 PDF；但有篡改/恶意风险，且**不便程序化批量**接入。
- **程序化批量路线（推荐）**：以 `scholarly`（元数据）+ `PyPaperBot` / `scholarextractor` / `PaperCrawlerUtil`（PDF）为核心，叠加**住宅代理轮换 + 限速 + stealth 指纹**，受阻时用 2Captcha 兜底。
- **更稳更合规的替代（建议另开角度评估）**：直接走官方开放 API —— Crossref / OpenAlex / Semantic Scholar / Unpaywall 取元数据与开放获取 PDF，**从根本上避开人机验证之争**。
