# 机构订阅 / EZproxy 集成设计

> 关联代码:`fulltext_fetcher/config.py`(配置字段)、`fulltext_fetcher/http_client.py`(接入钩子)。
> 本文件描述「如何在已获机构授权的前提下,合法取用付费墙内全文」的设计与可插拔代码骨架。

---

## ⚠️ 合规声明(置顶,务必先读)

本特性**仅供拥有合法机构订阅的用户、对其有权访问的内容使用**,目的是在「**已获授权**」前提下,
经所在机构的 EZproxy / SSO 通道,以与人工浏览器登录**等价**的方式正常取用全文。

严禁用于下列任何用途:

- ❌ 绕过、破解付费墙或任何访问授权;
- ❌ 使用非本人/非本机构合法持有的凭据(账号、Cookie、代理);
- ❌ 共享、转售、囤积下载所得的订阅内容;
- ❌ 违反出版商服务条款(ToS)、机构使用政策或当地法律(尤其是「系统性批量下载」红线)。

**默认即安全**:`ezproxy_prefix` / `institution_cookie` / `institution_domains` 三项默认全部为空/关闭,
此时程序行为与未引入本特性时**逐字节一致**,只走开放 API 正门,不触碰任何订阅通道。是否启用、
以何种规模启用,完全由掌握合法凭据的最终用户自行决定并自负合规责任。

## 0. 背景与定位

本项目(`fulltext_fetcher`)走**开放 API 正门**:聚合 13+ 免费源(Unpaywall / OpenAlex / Europe PMC /
arXiv / Semantic Scholar / CORE …)做多源回退定位 OA 全文。免费手段对**订阅型语料**的成功率天花板
约 **10.8%** —— 这是付费墙的**物理边界**,任何免费技巧都拿不到订阅全文,这一结论是项目的一贯认识。

要在**合法**前提下突破这条边界,投产比最高的唯一路线是**机构订阅 / EZproxy**:利用用户所在机构
已经购买的订阅权益,经机构的鉴权通道取用全文。其代价是**必须有用户的机构环境与合法凭据**,
因此本项目**不内置任何凭据、不替用户登录、不联网验证**,只提供:

1. 本设计文档(原理、合规边界、集成点、配置与回退);
2. **可插拔代码骨架**:默认关闭、零副作用,启用后作为机构访问的统一接入层。

---

## 1. 三种接入方式:原理、适用场景与优劣

机构订阅的「正常取用」在技术上有三条主流通道。三者可单用,也可组合(如 EZproxy 前缀 + 登录态 Cookie)。

### 1.1 EZproxy 前缀重写(URL Rewriting Proxy)

**原理**:EZproxy 是高校图书馆最常见的校外访问网关。用户先在机构 EZproxy 登录页用机构账号登录,
拿到会话 Cookie;之后把出版商 URL 改写成「经 EZproxy 取用」的形式,请求即被代理、带上机构身份。
两种常见改写形式:

- **前缀式(starting point URL)**:`<prefix> + 原始URL`

```text
https://login.ezproxy.uni.edu/login?url=https://www.sciencedirect.com/science/article/pii/Sxxxx
```

- **主机名改写式(proxy by hostname)**:把域名内嵌机构后缀

```text
www.sciencedirect.com  →  www-sciencedirect-com.ezproxy.uni.edu
```

**适用场景**:校外 / 家庭网络;机构部署了 EZproxy(绝大多数高校)。
**优**:覆盖面最广,一套前缀适配大量出版商;**劣**:需先完成 EZproxy 登录拿到会话态,Cookie 有时效。

### 1.2 机构 SSO / Shibboleth(OpenAthens)Cookie 注入

**原理**:基于 SAML 的联邦身份。用户经 WAYF（Where Are You From）选择机构 → 跳机构 IdP 登录 →
出版商 SP 颁发会话 Cookie。把这份**登录后**的会话 Cookie 注入后续请求,即以已登录身份取用全文。

**适用场景**:出版商支持 Shibboleth / OpenAthens 联邦登录(Nature、ScienceDirect、IEEE 等多数大社)。
**优**:不依赖 IP、不依赖代理,直连出版商即可;**劣**:Cookie 作用域/时效复杂(常按 SP 域名隔离),
需要用户从浏览器登录后导出,自动化登录涉及验证码/MFA,**不建议**也不在本骨架范围内自动化。

### 1.3 IP 白名单 / 校园网(IP Authentication)

**原理**:出版商按机构出口 IP 段授权,**无需登录**——只要请求源 IP 落在机构 IP 段内即视为已订阅用户。
**适用场景**:在校园网内,或通过机构 VPN / 机构 HTTP 代理把出网 IP 变为机构 IP。
**优**:对本程序几乎零改造(直接出网即可命中);**劣**:必须身处机构 IP 环境;若需经机构代理出网,
则要配置 HTTP(S) 代理(见 §4 的 `proxies` TODO)。

### 三种方式对比

| 维度 | EZproxy 前缀重写 | SSO / Shibboleth Cookie | IP 白名单 / 校园网 |
| --- | --- | --- | --- |
| 是否需登录 | 是(EZproxy 登录) | 是(IdP 登录) | 否 |
| 是否依赖机构 IP | 否 | 否 | 是 |
| 本项目改造点 | URL 重写 + Cookie | Cookie 注入 | (可选)代理出网 |
| 覆盖面 | 广(统一网关) | 取决于出版商联邦支持 | 取决于订阅 IP 授权 |
| 凭据时效 | 会话级 | 会话级(常更短) | 无(按 IP) |
| 自动化难度 | 中(登录态) | 高(MFA/WAYF) | 低 |

## 2. 与现有 pipeline 的集成点

### 2.1 唯一出网瓶颈:`HttpClient.get()`

整个流水线**所有出网请求都汇聚到** `fulltext_fetcher/http_client.py` 的 `HttpClient.get()`:

- 各源连接器查询 API 走 `ctx.client.get_json()` → 内部调用 `get()`;
- 下载 PDF(`download.py::download_pdf`)走 `client.get(candidate.url, stream=True)` → 同样是 `get()`;
- 元数据解析(`resolve.py`)亦经同一 client。

因此**只需在 `get()` 这一个点接入**,即可同时覆盖「API 定位」与「PDF 下载」两类请求,
无需散改各源连接器,也不触碰 `report.py / cli.py / resolve.py`。这是本设计选择 `http_client` 作为
单点注入层的根本原因。

### 2.2 数据流(启用机构访问后)

```text
源连接器 / download / resolve
        │  client.get(url) / get_json(url)
        ▼
HttpClient.get(url)
   ├─(1) needs_institution_access(原始域名, cfg)?
   │         否 → 原样请求(开放 API / OA 域名走这里 → 正门路径不变)
   │         是 ↓
   ├─(2) 注入机构会话 Cookie(institution_cookie)── 仅对需机构访问的域名
   ├─(3) rewrite_url_for_proxy(url, cfg) ── 改写为经 EZproxy 取用的 URL
   ▼
   既有的「按域限速 → 重试退避 → 熔断」逻辑(完全复用)
        │
        ▼
   download 层拿到 200 + %PDF → 既有 PDF 校验链(Content-Type / 魔数 / 大小)→ 落盘
```

要点:

- **域名分流**:`needs_institution_access()` 先放行所有开放 API / OA 域名(见 `http_client.py` 中
  `_OPEN_ACCESS_HOSTS`),保证免费正门**永不**被改写,也避免把机构 Cookie 泄露给第三方;
- **Cookie 作用域**:Cookie 仅对「判定需机构访问」的域名注入,不写进全局 `session.headers`;
- **复用既有能力**:重写后的请求继续享受原有的限速 / 重试 / 熔断 / PDF 校验,无需重复造轮子;
- **下载层零改动**:`download.py` 因为经由同一个 `client.get()`,**自动**获得 EZproxy 取全文能力。

## 3. 配置项设计与回退

### 3.1 新增配置字段(`fulltext_fetcher/config.py`)

```python
# 机构订阅 / EZproxy 接入(可选,默认全部关闭;对开放 API 正门零影响)
ezproxy_prefix: Optional[str] = None       # EZproxy 登录前缀,如 "https://login.ezproxy.example.edu/login?url="
institution_cookie: Optional[str] = None   # 机构 SSO/Shibboleth 登录后的会话 Cookie 串,如 "k1=v1; k2=v2"
institution_domains: List[str] = []        # 仅对这些出版商域名启用机构访问;空=不主动改写任何域名
```

均为可选字段、带默认值,因此:

- **编程接口自动支持**:`api.py::_make_config` 以 `hasattr` 校验 overrides,故
  `fetch_one(..., ezproxy_prefix=..., institution_cookie=...)` 直接可用,无需改 `api.py`;
- **CLI 暴露**(可选,属 `cli.py` 维护者范围):如需命令行开关,建议新增
  `--ezproxy-prefix` / `--institution-cookie` / `--institution-domain`(可重复)三个参数,
  本设计不改 `cli.py`,仅给出建议。

### 3.2 回退矩阵(关键:默认零影响)

| `ezproxy_prefix` | `institution_cookie` | `institution_domains` | 实际行为 |
| --- | --- | --- | --- |
| None | None | [] | **默认**:`needs_institution_access` 恒 False;`get()` 两步全空操作;**与未引入本特性逐字节一致** |
| 设置 | 任意 | 非空 | 仅对白名单出版商域名:Cookie 注入 + 前缀重写;OA 域名仍走正门 |
| None | 设置 | 非空 | 仅对白名单域名注入 Cookie(不改 URL);适合「IP/SSO 已授权,仅需带登录态」 |
| 设置 | 任意 | [] | 骨架阶段**保守不改写**(见 §4 TODO);需配白名单才生效 |

**判定逻辑**(`http_client.py::needs_institution_access`,纯函数、可单测):

1. 未配置任何机构凭据 → 直接 `False`(默认零副作用);
2. 命中 `_OPEN_ACCESS_HOSTS`(开放 API / OA 域名)→ `False`(正门绝不改写);
3. 配置了 `institution_domains` 白名单 → 仅命中者 `True`;
4. 配了凭据但白名单为空 → 骨架阶段保守 `False`(避免误导无关流量,见 TODO)。

### 3.3 启用示例(供拥有合法凭据的用户参考)

```python
from fulltext_fetcher import fetch_many

# 仅对 ScienceDirect / IEEE 经机构 EZproxy 取全文;开放 API 仍走免费正门
results, summary = fetch_many(
    ["10.1016/j.xxxx", "10.1109/yyyy"],
    email="you@uni.edu",
    ezproxy_prefix="https://login.ezproxy.uni.edu/login?url=",
    institution_cookie="ezproxy=AbCdEf...; s:edu=...",   # 由用户从浏览器登录后自行提供
    institution_domains=["sciencedirect.com", "ieeexplore.ieee.org"],
)
```

> 上例中的 Cookie 为占位示意,**不含**任何真实凭据;真实凭据由用户在自己的机构环境中获取并自负合规责任。

## 5. 安全与运维注意事项

- **凭据不落库、不入日志**:`institution_cookie` 等绝不写入 `attempts.jsonl / summary.json / metadata.jsonl`;
- **最小作用域**:Cookie 仅对白名单域名注入,杜绝泄露给 OA/第三方;
- **时效管理**:会话 Cookie 会过期,失效时表现为「拿到登录页 HTML 而非 PDF」,由既有 PDF 校验链拦截并记为失败;
- **限速礼貌**:沿用既有「按域限速 + 熔断」,严禁系统性批量抓取触发出版商风控或违反 ToS;
- **审计**:建议机构用户保留运行范围与用途记录,以备合规审计。
