## 0. 一句话结论

**Scholar「过人机」的真实瓶颈不在 reCAPTCHA，而在①出版商付费墙 403 与②强 CF 站的 JA3 绑定回放。** 本仓已把「不触发优先」的分层反爬（curl_cffi→nodriver→SerpApi）、CF 兜底（nodriver shim）、浏览器内直下 PDF 全部**程序化**；**仍须人工的只剩**：开打码/配 key、切 headed/Xvfb、住宅代理鉴权、机构订阅登录、camoufox 受控兜底——这些默认关，属「偶发最后手段」。

## 1. 2026 Scholar 反爬现状（要点）

- **失败面画像**：本仓 500 样本失败**几乎全是出版商 403 付费墙 / 免费源无候选，不是 reCAPTCHA 拦截**。→ 只要目标是「拿全文 PDF」，走 OA 下载器基本碰不到人机验证；只有「直抓 Scholar 拿被引/版本等原生字段」才需要过 reCAPTCHA。
- **Scholar 用 reCAPTCHA v2/v3**，且**开源项目本身几乎不自带「过人机」能力**：GS 抓取靠住宅代理/SerpApi，reCAPTCHA 靠打码服务或音频求解（可用性快速下降），PDF 靠多源 OA 回退。
- **本地音频求解在退化**：Google 持续收紧音频挑战限流，单 IP 高频立即触发 `Your computer or network may be sending automated queries` 硬封；**仅低频+住宅代理尚可**，规模化必须打码服务或 SerpApi 外包。
- **反爬胜负手在 L3「怎么驱动浏览器」+ IP 信誉 + 行为，不在 L1「声称什么」**。2026 基准（31 CF 目标/651 判定）：**nodriver 28/0 封锁（唯一零封锁）** > curl_cffi 26 ≈ patchright 25 ≈ camoufox 25 > vanilla 24。
- **CF 归属**：Scholar 本身**不走 Cloudflare**；CF 出现在**下载 PDF 时的出版商/镜像站**（RSC/ACS/Wiley/Elsevier 等）。
- **最强约束＝无代理单一本地 IP**：不能轮换 IP ⇒ **首要目标是「不触发」而非「过验证」**；本地/住宅 IP 信誉远高于数据中心 IP，是相对优势，但**不可再生，必须养住**。

## 2. nodriver 配置要点

- **是什么**：undetected-chromedriver 官方继任者（同作者），**纯异步直连 CDP**，无 Selenium/chromedriver、无 Playwright shim、无 `Runtime.enable` 启动序列、无 `navigator.webdriver`/`cdc_`/`--enable-automation` 痕迹——「启动就像人双击图标」。用**系统真 Chrome 148**（非 bundled Chromium，TLS/版本戳真实）。v0.50.3（2026-05 活跃），**AGPL-3.0**（对外服务须开源，内部研究无碍）。
- **仓内默认（`ScholarConfig`）**：`browser_engine="nodriver"`、`engine_order=["curl_cffi","nodriver"]`、`impersonate="chrome"`、`concurrency=1`、`page_interval 45–90s` 抖动、`backoff_base=2/cap=60`、`cooldown_after_block=900s`、`captcha_enabled=False`、`proxy_enabled=False`。
- **引擎实现（`fetcher.py::NodriverEngine`）**：延迟导入、`asyncio.run` 包装、`headless=True`、`--proxy-server=`（**不支持内嵌 user:pass**，仅注入 host:port）、渲染后 `get_content()`；**status 诚实置 None**（浏览器渲染拿不到可靠 HTTP 码，不臆造 200，被拦与否交 `detect_captcha` 按 HTML+`/sorry/` 判定）。
- **CF 兜底（`tools/flaresolverr_nodriver.py`，179 实测）**：免 Docker、FlareSolverr `/v1` 逐字段兼容的求解端点，用 nodriver 实现；**只设 `$env:FLARESOLVERR_URL` 一个环境变量即启用兜底，零改码**。headed CF 通过率更高、**按 origin 缓存 1200s**；实测 **4/4 origin `cf_clearance=YES`**。
- **代价/坑**：① AGPL-3.0；② **只让你「不被识别」，不自动解 Turnstile/reCAPTCHA token**；③ 官方称 **headless 更易被检测**、**代理鉴权（user:pass）尚不成熟**；④ **JA3 绑定站陷阱**：shim 拿到 `cf_clearance` 后，`download.py` 交给 curl_cffi **第三方回放**下 PDF 时，RSC 这类校验 JA3 的站会因指纹不匹配仍 403（ACS 不绑 JA3 故可过）——**正解是浏览器内直下 PDF**（同一会话/同一 JA3）。
- **一手实测**：nodriver 0.50.3 + 真 Chrome 148 headless 过 Bing **5/5**，解析出 **55** 候选 vs 纯 HTTP 仅 **4**（52 个 HTTP 拿不到）。

## 3. 哪些路径已程序化 / 哪些仍须人工

| 能力 | 状态 | 落点 / 说明 |
|---|---|---|
| L1 静态抓（curl_cffi impersonate=chrome，JA3/JA4 指纹） | ✅ 已程序化 | `CurlCffiEngine`；缺库降级 requests（低隐蔽） |
| L3 浏览器兜底（nodriver 首选 / patchright 回退，真 Chrome） | ✅ 已程序化 | `NodriverEngine`/`PatchrightEngine`，均延迟导入 |
| 分层降级编排 + 命中风控自动应对 | ✅ 已程序化 | `fetch_serp`：curl_cffi→nodriver，被拦即 rotate 代理+冷却，否则退避 |
| 限速/退避/冷却（45–90s 抖动、指数退避、900s 冷却、串行） | ✅ 已程序化 | `_throttle/_backoff/_cooldown`，默认值即合规纪律 |
| 被拦检测（403/429/503、`/sorry/` 重定向、验证码 HTML） | ✅ 已程序化 | `serp.detect_captcha` + `_is_blocked` |
| SerpApi 商业合规旁路（把过人机整包外包） | ✅ 已程序化 | `mode=serpapi`；`auto` 有 key 即走，`SerpApiEngine` |
| reCAPTCHA 打码适配器（2captcha/capsolver v2/v3） | ✅ 已接线（**默认关**） | `captcha.py`，延迟导入、缺库/缺 key 优雅降级为「不可用信封」 |
| CF 兜底求解（免 Docker nodriver shim，过 Cloudflare） | ✅ 已程序化 | `tools/flaresolverr_nodriver.py`，一个环境变量启用 |
| 浏览器内直下 PDF（解 JA3 绑定站） | ✅ 已程序化 | `download.py::_nodriver_fetch_pdf_bytes` + `cfg.browser_pdf_download=True` |
| **reCAPTCHA token 自动求解（硬刚验证码）** | ⚠️ **须人工** | `captcha_enabled=False`；须人工开开关+配 provider/key（付费）或人在环（buster） |
| **reCAPTCHA v2 音频 + 本地 Whisper 免费自解** | ⚠️ **未接入** | 仅调研，属「极低频最后手段」，v3/hCaptcha 免费无解 |
| **住宅代理鉴权（user:pass）** | ⚠️ **半程序化** | `proxy_enabled=False`；nodriver `--proxy-server` 只注入 host:port，鉴权 TODO（patchright 原生支持） |
| **camoufox 受控兜底（DataDome/最狠出版商）** | ⚠️ **未接入** | 实验期，仅 nodriver/patchright 都被挡时人工受控引入 |
| **强验证切 headed/Xvfb** | ⚠️ **须人工** | 默认 headless 够 SERP；遇 Google 强验证/无显示环境须人工切 headed 或 Xvfb |
| **机构订阅 SSO 登录 / 真订阅墙 403** | ⚠️ **须人工/救不了** | 需人工凭据（路线 A）；真付费墙任何 CF/JA3 手段都救不了，只能机构订阅/灰色兜底 |

---

## 4. 落地建议（一句话）

**把唯一干净本地 IP 当稀缺资源**：默认走「curl_cffi → nodriver（headed 时更稳）→ SerpApi 正门」，CF 站用仓内 nodriver shim 兜底、JA3 绑定站走浏览器内直下 PDF；**打码/住宅代理/camoufox/headed 一律默认关，仅偶发人工启用**；「不触发」永远比「过验证」划算。
