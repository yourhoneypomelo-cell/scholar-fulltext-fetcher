## 二、解决方案（三层·均经真实验证）

1. **谷歌学术可达 PDF（用户目标③·opt-a 首要）**：吸收 `paper-search-mcp` / `paper-download-mcp` 配方——DOI→Semantic Scholar + Unpaywall + OpenAlex 并发取首个 `%PDF` 有效 + fallback 链（→OpenAIRE/CORE/EuropePMC/PMC→Unpaywall）。**业界标准做法，方向正确**。无 DOI 长尾用 `scholarly2` + 住宅 IP。
2. **过 CF 挑战**：P1 nodriver→`zendriver`（75% pass 数据支撑充分；一步修 Chrome146 cookie + CDP 漂移 + Docker + cookie 持久）。
3. **过 Turnstile**：`EzSolver`（64★ MIT 本地 API·新发现更优）或 `nodriver-cf-verify`；`zendriver` 内置 `verify_cf()` 兜底。

## 三、新发现（值得评估，未在总评估）

- **Scrapling v0.4.8**（2026-05）：三合一 fetcher——`Fetcher`(TLS-impersonated HTTP) / `StealthyFetcher`(Camoufox + `solve_cloudflare=True` 自动过盾) / `DynamicFetcher`(Playwright)。**一库覆盖从 TLS 到全浏览器全 range**，可能简化过验证层技术栈，值得专项评估。
- **flaresolverr-rs**（Rust）：FlareSolverr drop-in，20/20 100% pass，avg 2.7s，`cf_clearance` 缓存复用，比 py 版快 1.4–3.5×。
- **Byparr**（Camoufox 后端 FastAPI，FlareSolverr API 兼容）：对 2026 Turnstile / Managed Challenge 成功率高于原版 FlareSolverr。
- **EzSolver**（见上）：本地 Turnstile 求解，零付费服务。

## 四、诚实边界（不变）

- 开源侧净增有限（个位数~低两位数 pp）；**最大净增来自 A5 机构订阅（非过认证墙开源工具，已按用户口径降级为收口后可选）**。
- 数据中心 IP 过 CF / RSC governor 坏 reCAPTCHA / 纯订阅付费墙——开源无解，见《回收实测结论-CF与免费路线到顶》。
