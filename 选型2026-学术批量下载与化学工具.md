## 一、一页结论(Top 采用建议)

1. **★ 首选采用/整合:`paperscraper`(v1.0.0,2026-06-16,Production/Stable)**——本批语料**最契合**:元数据 + 全文 PDF/XML,覆盖 **arXiv / bioRxiv / medRxiv / chemRxiv + PubMed**,并对**化学预印本 chemRxiv** 走 **Cambridge Open Engage 公共 API**;付费墙有 **BioC-PMC XML(PMC OA)/ eLife XML / 出版商 TDM API(Wiley/Elsevier,学术免费)/ bioRxiv S3** 多重回退。→ **本项目应新增 `sources/preprints.py`**(已有成员在建)复用其 **chemRxiv=Cambridge Open Engage 端点**与 **BioC-PMC XML** 思路(零依赖复刻或作可选依赖)。
2. **借鉴不整合:`PyPaperBot`(v1.4.1,2024-12)**——批量 UX(`--doi-file` / `--use-doi-as-filename` / `result.csv` / 按年·刊·被引过滤)与我们 CLI **已高度重合**;其 GS 直抓会被封、SciHub/SciDB/Anna's Archive 属**灰色**——**只借鉴、不接入**。
3. **可选补强:Zotero `translation-server` + `translators`**——把任意 URL/标识符**程序化**转 CSL-JSON 元数据(600+ 站点解析器),可作 `resolve.py` 对硬样本的兜底元数据源;但为 Node/Docker 服务、偏重,**可选**。
4. **化学专属数据面:Cambridge Open Engage(chemRxiv 官方 API)+ PubChem**——前者是 chemRxiv 全文正门(必用);后者是**化合物/物性**库(非论文全文,做化学实体富化时可选)。
5. **不用/慎用**:`scidownl` / `scihub.py`(停更、域名漂移、灰色);镜像站(角度7,灰色+篡改风险,不宜程序化批量)。

> 一句话:**批量下载器层面我们已够用**(自建 22 源 + CLI);真正的**增量在「化学预印本(chemRxiv)全文 + PMC/eLife XML 回退」**,而这正是 `paperscraper` 的强项——**吸收其端点与回退策略即可**,无需引入重框架。

## 三、Top1-2 源码要点

### 3.1 `paperscraper`(Top1,读源码要点)
- **PDF 检索主入口**:`from paperscraper.pdf import save_pdf; save_pdf({"doi": ...}, filepath=..., save_metadata=True)`;批量:`save_pdf_from_dump(jsonl, pdf_path=..., key_to_save="doi")`(`key_to_save` ∈ doi/title/date)。
- **化学预印本(关键)**:`save_pdf` 内对 `"chemrxiv" in doi.lower()` 走 **Cambridge Open Engage 公共 API**:
  `GET https://www.cambridge.org/engage/coe/public-api/v1/items/doi/{doi}` → `item["asset"]["original"]["url"]`(或 `asset.url`)即 PDF 直链;并有"chemRxiv 被 Cloudflare 挡时切 Cambridge 端点"的容错(commit #99)。
- **付费墙回退链(commit #72)**:直链失败 → **BioC-PMC XML**(PMC OA 全文)→ **eLife XML**(eLife 仓库)→ **出版商 TDM API**(Wiley `api.wiley.com/onlinelibrary/tdm/...`、Elsevier `api.elsevier.com/content/article/doi/...`,学术机构多免费,需 key/`.env`)→ **bioRxiv S3**(AWS requester-pays)。**明确声明:不能绕付费墙/robots**。
- **预印本本地 dump**:`from paperscraper.get_dumps import chemrxiv, biorxiv, medrxiv`——一次性拉全量 JSONL(chemRxiv <15min/+50K/~30MB),之后**本地可复现检索**(与我们 `snapshot.py` 零联网线理念一致)。
- **裁决**:化学语料下 chemRxiv + PMC-XML 回退是**真实增量**;**建议在 `sources/preprints.py` 复刻其 Cambridge Open Engage 端点 + BioC-PMC XML 回退**(零依赖、贴合本项目 `PdfCandidate` 契约),或把 paperscraper 作**可选依赖**调 `save_pdf`。

### 3.2 `PyPaperBot`(Top2,读源码要点)
- CLI:`python -m PyPaperBot --doi-file=dois.txt --dwn-dir=out [--use-doi-as-filename] [--min-year] [--max-dwn-cites] [--restrict 0/1] [--scihub-mirror] [--annas-archive]`;来源 GS→Crossref→SciHub→SciDB;产 `result.csv` + BibTeX。
- **与本项目对照**:批量文件输入、DOI 命名、状态 CSV、按年/被引过滤——**我们 CLI + report.py 已覆盖**;其增量仅 SciHub/SciDB/Anna's Archive(**灰色**)。→ **只借鉴 UX,不接入灰色源**。

## 四、对本项目的落地建议

| 增量点 | 建议 | 落点 |
|---|---|---|
| **chemRxiv 全文** | 复刻 `paperscraper` 的 Cambridge Open Engage 端点(`.../public-api/v1/items/doi/{doi}` → asset.original.url) | `sources/preprints.py`(在建),产 `PdfCandidate(kind='pdf', confidence≈78)` |
| **PMC OA 全文兜底** | 复刻 BioC-PMC XML / EuropePMC render(NIH 无反爬,近乎必得) | 已有 `europe_pmc` 源;可补 BioC-PMC XML 变体 |
| **eLife XML** | eLife 仓库 XML 兜底(gold OA) | `sources/preprints.py` 或 repositories |
| **预印本本地 dump** | chemRxiv/bioRxiv JSONL 一次性灌本地(零联网复现) | 与 `snapshot.py`/`ingest.py` 同理,可作 `--snapshot` 扩展 |
| **硬样本元数据** | 可选 Zotero translation-server(URL→CSL) | `resolve.py` 兜底(可选,重) |

> **接口对齐提醒(给 `sources/preprints.py` 负责人)**:chemRxiv 正门 = `GET https://www.cambridge.org/engage/coe/public-api/v1/items/doi/{doi}`,PDF 在 `item.asset.original.url`(回退 `item.asset.url`);建议产出 `PdfCandidate(url, source="chemrxiv", kind="pdf", confidence=78)`,与既有源体系一致,便于 `pipeline` 直接消费。

## 五、合规与边界

1. **正门优先**:chemRxiv/PMC/eLife/arXiv 等**开放预印本与 OA 全文**是合规正门;付费墙仍是物理边界(`paperscraper` 亦明言不绕付费墙/robots)。
2. **不接入灰色源**:SciHub/SciDB/Anna's Archive/镜像站默认不接(与本项目既有姿态一致,Sci-Hub 仅 `--enable-scihub` 显式开)。
3. **出版商 TDM API**:Wiley/Elsevier TDM 对**有机构订阅**的学术用户多免费,需自备 key——属"合法机构渠道",可作可选增强(对齐既有《机构订阅集成设计》)。
4. **可选依赖隔离**:若引入 `paperscraper` 作依赖,应函数内延迟导入、缺库优雅降级(沿用本项目既有可选依赖规范),不进强制依赖。
